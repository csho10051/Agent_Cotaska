## 3. 候補技術の比較

### 3.1 選択肢一覧

| # | 構成 | フロントエンド | バックエンド | DB接続 |
|---|------|--------------|-------------|--------|
| A | **Tauri** | HTML/CSS/JS（React/Vue/Svelte等） | Rust | rusqlite / sqlx |
| B | **Electron** | HTML/CSS/JS（React/Vue/Svelte等） | Node.js | better-sqlite3 |
| C | **Python GUI** | PyQt6 / PySide6 / Flet | Python | SQLAlchemy + sqlite3 |

### 3.2 評価軸と比較

| 評価軸 | Tauri (A) | Electron (B) | Python GUI (C) |
|--------|:---------:|:------------:|:--------------:|
| **4カラム UI 実現性** | ◯ Web技術で自由 | ◯ Web技術で自由 | △ PyQt可だが手作業 |
| **Markdown エディタ** | ◯ CodeMirror利用 | ◯ CodeMirror利用 | △ QScintilla利用 |
| **ドラッグ&ドロップ** | ◯ HTML5 D&D | ◯ HTML5 D&D | △ Qt D&D |
| **進捗バー/自動保存** | ◯ | ◯ | ◯ |
| **SQLite 連携** | ◯ rusqlite | ◯ better-sqlite3 | ◯ SQLAlchemy |
| **バンドルサイズ** | ◯ 50-80MB | ✕ 100MB以上 | ◯ 30-50MB |
| **メモリ使用量** | ◯ 軽量 | ✕ Chromium常駐 | ◯ 中程度 |
| **学習コスト** | △ Rust習得必要 | ◯ JS のみ | ◯ Python のみ |

| **インストール難度** | △ Rust+ビルドツール | ◯ Node.js のみ | ◯ Python のみ |

## 4. 決定理由

### 選択：**Electron（オプション B）**

**理由**

1. **Web 技術の自由度** - 4カラムレイアウト、ドラッグ&ドロップを容易に実装
2. **React 生態系** - 状態管理（Redux）、UI フレームワーク（Material-UI等）が充実
3. **Node.js 統一** - バックエンド処理も JavaScript 一言語で対応
4. **デスクトップアプリとしての成熟度** - VS Code、Discord など実績多数
5. **学習コスト** - チーム内メンバーが JavaScript に習熟
6. **ファイルシステムアクセス** - fs / fs.promises で直接操作（ローカルファースト要件に適合）

**トレードオフ**

- バンドルサイズが大（Chromium 同梱）
- メモリ使用量が相対的に多い
- Rust ベースの Tauri より軽量では無い

**対策**

- コード分割（Code Splitting）でバンドルサイズ削減
- 不要な依存を除外
- 定期的なパフォーマンス測定

## 5. セットアップと技術スタック

### 5.1 フロントエンド

- **Electron** 27.x 以上
- **React** 18.x 以上
- **CSS** - Plain CSS + CSS Modules
- **エディタ** - CodeMirror 6

### 5.2 バックエンド（Main プロセス）

- **Node.js** v22 以上
- **better-sqlite3** - 同期 DB 操作
- **fs.promises** - ファイル操作（UTF-8）
- **chokidar** - ファイル監視

### 5.3 ビルド・デプロイ

- **Vite** - フロントエンド ビルド（高速）
- **electron-builder** - Electron パッケージ化
- **GitHub Actions** - CI/CD（ビルド自動化）
