# 設定画面

## 1. 目的

Cotaska のアプリ全体に関わる情報表示と設定変更を、タスク管理画面から独立した設定画面として提供する。

設定画面は、初期スコープでは「アプリ情報」と「バージョン情報表示」を中心に整備し、将来的に表示設定、編集設定、バックアップ設定、AI連携設定などを追加できる構成とする。

## 2. 対象範囲

| 項目 | 内容 |
|------|------|
| 対象画面 | 設定画面 |
| 起動導線 | 左サイドバー下部の設定アイコン |
| 初期カテゴリ | アプリ情報 |
| 初期実装の主目的 | Cotaska の現在バージョンを確認できるようにする |
| 関連モック | `10_docs/10_設計/30_画面モック/settings_app_info_mock.html`, `10_docs/10_設計/30_画面モック/settings_display_edit_mock.html` |

## 3. 画面構成

設定画面は、以下の3領域で構成する。

| 領域 | 役割 |
|------|------|
| サイドバー | 既存アプリ共通の主要ナビゲーション。設定アイコンをアクティブ表示する |
| 設定カテゴリパネル | 設定カテゴリを一覧表示する左側パネル |
| 設定内容ペイン | 選択中カテゴリの設定内容を表示するメイン領域 |

デスクトップ表示では、サイドバー、設定カテゴリパネル、設定内容ペインを横並びにする。

狭い画面幅では、設定カテゴリパネルを非表示にし、設定内容ペインを優先して表示する。カテゴリ切り替えが必要になった段階で、ドロワーまたはメニュー形式の導線を追加する。

## 4. 外観方針

設定カテゴリパネルは黒基調とする。既存の左サイドバーも暗色であるため、設定画面全体の左側領域を落ち着いた管理画面らしい見た目に揃える。

| 要素 | 方針 |
|------|------|
| 設定カテゴリパネル背景 | 黒基調のダークカラー |
| 区切り線 | 背景より少し明るい境界色 |
| 通常カテゴリ文字 | 明るめのグレー |
| グループラベル | 低彩度のグレー |
| 選択中カテゴリ | 暗色の選択背景、白文字、アクセントライン |
| ホバー状態 | 通常背景より少し明るい暗色 |

色は実装時に CSS 変数またはテーマ定数として定義し、設定画面内で局所的に参照する。初期モックでは `--settings-panel-*` の CSS 変数を使用する。

## 5. 設定カテゴリ

初期カテゴリは以下とする。

| カテゴリ | 初期状態 | 内容 |
|----------|----------|------|
| アプリ情報 | 実装対象 | バージョン、配布種別、更新確認、実行環境情報 |
| 表示と編集 | 設計対象 | 表示密度、タスク本文編集、Markdown 表示、詳細ペイン動作など |
| バックアップ | 将来追加 | バックアップ作成、保存先、復元導線など |
| タスクデータ | 将来追加 | データ保存場所、インデックス再生成、データ検証など |
| 詳細設定 | 将来追加 | 開発・検証向けの詳細オプション |

将来カテゴリを追加する場合も、設定カテゴリパネルに項目を追加し、設定内容ペインにカテゴリ単位のコンポーネントを表示する構成を維持する。

## 6. アプリ情報

アプリ情報カテゴリでは、利用者と開発者が問い合わせ・検証時に必要な基本情報を確認できるようにする。

### 6.1 表示項目

| 項目 | 内容 |
|------|------|
| 製品名 | Cotaska |
| 現在バージョン | `Cotaska 0.1.0` のように表示 |
| バージョン正本 | `20_app/package.json` の `version` |
| 配布フォルダ | 例: `Cotaska-dist` |
| 利用状態 | 通常利用可能、更新確認失敗などを簡潔に表示 |
| 更新方式 | 初期対応では自動更新しない |
| 更新案内 | 利用者確認付きの手動ダウンロード案内 |
| 取得 API | `window.cotaskaAPI.app.getVersion()` |

### 6.2 操作

| 操作 | 動作 |
|------|------|
| 更新を確認 | 最新版取得元へ問い合わせ、現在バージョンと比較する |
| ダウンロードページを開く | 最新版がある場合のみ、外部ブラウザで配布ページを開く |
| 閉じる | 設定画面を閉じ、元のタスク画面へ戻る |

### 6.3 エラー時表示

| 項目 | 内容 |
|------|------|
| バージョン取得失敗 | `バージョンを取得できませんでした` のように表示し、タスク操作は妨げない |
| 更新確認失敗 | ネットワークエラー等を表示し、再試行できる状態にする |
| 外部ページ起動失敗 | ダウンロードページを開けない旨を通知する |

### 6.4 バージョン情報表示

現在バージョンは `20_app/package.json` の `version` を正本とする。

renderer は `package.json` や Node.js API に直接アクセスせず、preload bridge 経由でバージョン情報を取得する。

| 項目 | 内容 |
|------|------|
| API 例 | `window.cotaskaAPI.app.getVersion()` |
| 表示形式 | `Cotaska 0.1.0` |
| 取得失敗時 | 通常利用を妨げず、取得できない旨を控えめに表示する |

### 6.5 最新版確認と更新案内

初期対応では自動更新を行わない。Cotaska はローカルのタスクファイルを正本として扱うため、利用者確認なしの更新よりもデータ保全を優先する。

| 状態 | 表示・動作 |
|------|------------|
| 最新版あり | 最新版があることを表示し、ダウンロードページを開く導線を表示する |
| 最新版なし | 最新であることを控えめに表示する |
| 取得失敗 | ネットワークエラー等を表示しつつ、タスク操作は妨げない |

最新版取得元は、GitHub Releases または `latest.json` を候補とする。具体的な取得元は更新配布方式を決める段階で確定する。

## 7. 表示と編集

表示と編集カテゴリでは、タスク一覧、タスク詳細、Markdown 編集など、日常利用時の見た目と編集体験に関わる設定をまとめる。

初期設計では設定項目の候補を定義する。実装時は保存方式を確定した項目から段階的に有効化する。

### 7.1 設定項目

| 項目 | 初期値 | 内容 |
|------|--------|------|
| 表示密度 | 標準 | タスク一覧や詳細ペインの余白を `コンパクト`、`標準`、`ゆったり` から選択する |
| 完了タスク表示 | 表示する | 今日、明日、次の7日間などで完了タスクを表示するかを切り替える |
| タスクID表示 | 表示する | タスク行または詳細ペインで `T-XXXX` を表示する |
| Markdown 編集ハイライト | 有効 | 編集モードで Markdown 記法を視覚的に強調する |
| Markdown プレビュー初期表示 | 編集 | 詳細ペイン本文を開いたときの初期モードを選択する |
| 外部アプリで開く導線 | 表示する | 詳細ペインからタスク Markdown を OS 既定アプリで開く導線を表示する |
| 詳細ペイン拡大ボタン | 表示する | 詳細ペイン右上に拡大・解除ボタンを表示する |
| 日付入力ショートカット | 表示する | 今日、明日、1週間後などの日付入力ショートカットを表示する |

### 7.2 保存対象

表示と編集カテゴリの設定値は、ユーザーごとの UI 設定として保存する候補とする。

初期実装では未保存の画面状態として扱ってもよい。永続化する場合は、タスクデータとは別の設定ファイルまたは既存のユーザー設定保存領域を使い、タスク `.md` ファイルには保存しない。

| 項目 | 方針 |
|------|------|
| タスクファイル | 設定値を保存しない |
| `_index.yaml` | 設定値を保存しない |
| localStorage | 暫定保存先の候補 |
| ユーザー設定ファイル | 正式保存先の候補 |

### 7.3 初期値

初期値は既存の Cotaska の表示・編集挙動に近い状態とする。新しい表示を突然強制せず、設定画面追加後も既存操作の感覚を維持する。

| 項目 | 初期値 |
|------|--------|
| 表示密度 | 標準 |
| 完了タスク表示 | 表示する |
| タスクID表示 | 表示する |
| Markdown 編集ハイライト | 有効 |
| Markdown プレビュー初期表示 | 編集 |
| 外部アプリで開く導線 | 表示する |
| 詳細ペイン拡大ボタン | 表示する |
| 日付入力ショートカット | 表示する |

### 7.4 操作

| 操作 | 動作 |
|------|------|
| トグル切り替え | 対象設定の有効・無効を切り替える |
| セグメント選択 | 表示密度や初期表示モードなど、少数選択肢を切り替える |
| 初期値に戻す | 表示と編集カテゴリの設定を既定値に戻す |

設定変更は、保存方式が確定するまではモック上の表現に留める。実装時に即時反映、自動保存、保存ボタン方式のいずれにするかを決める。

## 8. バックアップ

バックアップカテゴリでは、タスクデータ保全に関わる設定と操作を扱う。

初期設計ではカテゴリ枠のみ定義し、詳細仕様はバックアップ機能の実装タスクで確定する。

## 9. タスクデータ

タスクデータカテゴリでは、データ保存場所、インデックス再生成、タスクファイル検証など、データ管理に関わる操作を扱う。

`data/tasks/*.md` はタスクの正本であり、設定画面から操作を追加する場合は、破壊的操作にならないよう確認導線とログ記録を設ける。

## 10. 詳細設定

詳細設定カテゴリでは、開発・検証向けのログ、デバッグ、実験的機能を扱う。

通常利用者向けの設定と混在させず、必要になった段階で個別に仕様化する。

## 11. 実装方針

設定画面は renderer 側の画面状態として扱い、初期実装では保存データ構造を増やさない。

| 項目 | 方針 |
|------|------|
| 画面表示状態 | React state で管理 |
| 設定カテゴリ | 定数配列で定義し、カテゴリIDで選択状態を管理 |
| アプリ情報取得 | `window.cotaskaAPI` 経由 |
| 自動保存 | 初期スコープ外 |
| タスクデータ変更 | 初期スコープでは行わない |

設定値を永続化するカテゴリを追加する場合は、保存先、初期値、移行、バックアップ対象を個別に設計する。

## 12. IPC/API

初期スコープで想定する API は以下とする。

```javascript
window.cotaskaAPI.app.getVersion()
```

将来的に更新確認を実装する場合は、以下のような API を追加候補とする。

```javascript
window.cotaskaAPI.app.checkForUpdates()
window.cotaskaAPI.app.openDownloadPage()
```

renderer は context isolation の前提を守り、main process の機能へ直接アクセスしない。

表示と編集カテゴリの永続化を実装する場合は、以下のような API を追加候補とする。

```javascript
window.cotaskaAPI.settings.get()
window.cotaskaAPI.settings.update(partialSettings)
window.cotaskaAPI.settings.reset(category)
```

## 13. テスト観点

- サイドバー下部の設定アイコンから設定画面を開けること
- 初期カテゴリとしてアプリ情報が表示されること
- 設定カテゴリを切り替えたとき、左側パネルのアクティブ表示と右側内容が一致すること
- 現在バージョンが `20_app/package.json` の `version` と一致すること
- renderer が `window.cotaskaAPI` 経由でバージョンを取得すること
- 表示と編集カテゴリのトグルやセグメントが、定義した初期値で表示されること
- 設定画面を開閉しても、既存のタスク一覧、詳細ペイン、保存処理に影響しないこと
- 更新確認に失敗しても、通常のタスク操作が妨げられないこと
