# 設定画面

## 1. 目的

Cotaska のアプリ全体に関わる情報表示と設定変更を、タスク管理画面から独立した設定画面として提供する。

設定画面は、左端サイドバーの設定アイコンから開く。画面内には設定カテゴリ用のサイドパネルを持ち、`アプリ情報`、`設定`、`バックアップと復元` を切り替えて表示する。

## 2. 対象範囲

| 項目 | 内容 |
|------|------|
| 対象画面 | 設定画面 |
| 起動導線 | 左端サイドバー下部の設定アイコン |
| 初期表示 | アプリ情報 |
| 実装済み範囲 | 設定画面の表示枠、アプリ情報の現在バージョン表示、設定タブの入力欄表示、バックアップタブの未実装表示 |
| 未実装範囲 | 設定値の永続化、各設定値の実機能への反映、更新確認、ダウンロードページ表示、バックアップ作成、復元 |

## 3. 画面構成

設定画面は以下の領域で構成する。

| 領域 | 役割 |
|------|------|
| 左端サイドバー | 既存アプリ共通のナビゲーション。設定アイコンをアクティブ表示する |
| 設定カテゴリサイドパネル | `アプリ情報`、`設定`、`バックアップと復元` を表示し、選択中カテゴリを切り替える |
| 設定内容ペイン | 選択中カテゴリの内容を表示する |

設定画面を表示している間は、タスク一覧、ナビパネル、詳細ペインは非表示とする。設定画面から別カテゴリへ切り替えても、タスクデータは変更しない。

## 4. カテゴリ

| カテゴリ | 状態 | 内容 |
|----------|------|------|
| アプリ情報 | 一部実装済み | 製品名、現在バージョン、更新確認、手動ダウンロード案内を表示する |
| 設定 | 表示のみ実装済み | 表示名、外部エディタ、通知時間、文字サイズ、外部AIAgent を表示する |
| バックアップと復元 | 実装済み | バックアップ作成、保存先選択、復元導線などを扱う |

## 5. アプリ情報

### 5.1 表示項目

| 項目 | 内容 | 実装状態 |
|------|------|----------|
| 製品名 | `Cotaska` | 実装済み |
| 現在バージョン | `Cotaska 0.1.1` のように表示する | 実装済み |
| 更新案内 | 利用者確認付きの手動ダウンロード案内 | 表示のみ実装済み |

アプリ情報には「実行環境」配下の状態、取得 API、配布フォルダ、更新方式などの詳細行は表示しない。必要になった場合は、詳細設定または診断情報カテゴリとして別途設計する。

### 5.2 バージョン情報取得

現在バージョンは `20_app/package.json` の `version` を正本とする。

renderer は Node.js API や `package.json` に直接アクセスしない。main process でアプリ情報を組み立て、preload bridge 経由で renderer に渡す。

| 項目 | 内容 |
|------|------|
| main IPC | `app:getInfo` |
| preload API | `window.cotaskaAPI.app.getInfo()` |
| 戻り値 | `{ productName, currentVersion, distributionFolder, updateGuidance }` |
| 表示形式 | `Cotaska ${package.version}` |

`distributionFolder` は内部情報として取得可能だが、現行画面では表示しない。

### 5.3 更新確認

CHG-067 の初期実装では自動更新を行わない。Cotaska は最新版の有無を確認し、必要な場合に利用者確認付きで手動ダウンロードページを開く導線だけを提供する。

CHG-069 では、Windows の NSIS インストール版を対象に確認付き自動更新を追加する。アプリ情報では、更新確認、更新ファイルのダウンロード、再起動して更新を段階的に操作できるようにする。ダウンロードと再起動更新はいずれも利用者操作を必要とし、無断でアプリを終了したり更新を適用したりしない。

現在バージョンは `20_app/package.json` の `version`、最新バージョンは公開済みの最新バージョン情報を返す配布元メタデータを正本とする。最新バージョン取得元は GitHub Releases または配布フォルダ側の `latest.json` 相当を候補とし、実装時に確定する。取得した最新バージョンは semantic version として比較し、比較不能な値は更新有無不明として扱う。

| 操作 | 方針 | 実装タスク |
|------|------|------------|
| 更新を確認 | 最新バージョン取得元へ問い合わせ、現在バージョンと比較する | T-0183 |
| ダウンロードページを開く | 利用者確認後に外部ブラウザで配布ページを開く | T-0183 |
| 取得失敗時 | 通常利用を妨げず、再試行可能なエラー表示にする | T-0183 |

| 状態 | 表示・動作 |
|------|------------|
| 未確認 | 現在バージョンと「更新を確認」操作を表示する |
| 確認中 | 操作ボタンを一時的に無効化し、確認中であることを表示する |
| 最新版なし | 現在のバージョンが最新である旨を表示する |
| 最新版あり | 最新バージョン番号と、手動ダウンロードページを開く操作を表示する |
| 取得失敗 | 失敗理由を簡潔に表示し、再試行できる状態に戻す |

ダウンロードページを開く前には確認ダイアログを表示する。外部ブラウザ起動に失敗した場合は、通常利用を妨げずエラーを表示する。更新ファイルの自動取得、差し替え、再起動、自動インストールは CHG-067 の対象外とする。

CHG-069 の自動更新では、`electron-updater` を利用し、更新確認結果、ダウンロード進捗、ダウンロード完了、エラーを renderer に通知する。portable 実行、開発実行、更新メタデータ不足などで自動更新が使えない場合は、アプリを終了せず、手動ダウンロード案内を表示する。

## 6. 設定

設定タブでは、以下の項目を扱う。設定値は `settings.yaml` を正本とし、renderer は直接ファイルを読み書きせず、preload bridge / IPC 経由で取得・保存する。

| 項目 | 内容 | 初期値 | 実装状態 | 実装タスク |
|------|------|--------|----------|------------|
| 表示名 | Cotaska の表示名 | `Cotaska` | CHG-063 で保存・反映 | T-0179 / T-0189 |
| 外部エディタ | タスクファイルを外部アプリで開く場合に使用する実行ファイルの絶対パス | 空欄 | CHG-064 で保存・反映 | T-0180 / T-0192 |
| 通知時間 | タスク日時が近い場合の事前通知時間 | `5` 分 | CHG-065 で保存・反映 | T-0181 / T-0195 |
| 文字サイズ | タスク詳細の文字サイズ | `14` px | CHG-066 で保存・反映 | T-0182 / T-0198 |
| 外部AIAgent | ローカル CLI 連携先 | `(未実装) ローカルCLI` | 表示のみ | CHG-058 系タスク |

画面上には「デフォルト値」列を表示しない。初期値は設計・実装上の値として扱う。

### 6.1 表示名

表示名は、利用者が Cotaska の表示名を変更したい場合に使う。

| 項目 | 方針 |
|------|------|
| 保存 | `settings.yaml` の `displayName` に保存する |
| IPC | `window.cotaskaAPI.settings.get()` で取得し、`window.cotaskaAPI.settings.update({ displayName })` で保存する |
| 反映先 | ウィンドウタイトルとアプリ情報の製品名表示へ反映する。OS ランチャー表示名や実行ファイル名は配布物メタデータのため、設定変更では変更しない |
| 入力制約 | 前後空白は保存前に除去する。空欄は無効として保存せず、既定名へ戻す操作はリセットまたは `Cotaska` の明示入力で行う。最大 40 文字を上限とし、改行や制御文字は不可とする |
| 反映タイミング | 保存成功後に現在の画面表示とウィンドウタイトルへ即時反映し、再起動後は `settings.yaml` から復元する |

### 6.2 外部エディタ

外部エディタは、タスク詳細から `.md` ファイルを外部アプリで開くときに使う。

| 項目 | 方針 |
|------|------|
| 保存 | `settings.yaml` の `externalEditorPath` に絶対パス文字列として保存する |
| IPC | `window.cotaskaAPI.settings.get()` / `update({ externalEditorPath })` で保存値を扱い、参照ボタンは `window.cotaskaAPI.settings.selectExternalEditor()` で main process のファイル選択ダイアログを開く |
| 参照ボタン | 実行ファイル選択ダイアログを開き、選択されたパスを入力欄へ反映する。保存は設定更新操作として行う |
| 起動 | 設定済みパスがある場合、その実行ファイルへ main process が解決した対象タスク `.md` パスを引数として渡す |
| 空欄時 | `.md` ファイルの既定のアプリで開く |
| 入力制約 | 空欄を許可する。入力する場合は絶対パスのみ許可し、相対パス、ディレクトリ、存在しないファイルは保存時または起動時にエラーとする |
| エラー | パス不存在、起動失敗、権限エラーを利用者へ表示する。エラー時もタスク本文や設定値を破損させない |

既存の `window.cotaskaAPI.shell.openTaskFile(taskId)` は、CHG-064 で外部エディタ設定を参照するように拡張する。renderer はタスクファイルパスや外部アプリ起動を直接扱わず、`taskId` を IPC へ渡す。

### 6.3 通知時間

通知時間は reminderService の通知判定へ反映する。

| 項目 | 方針 |
|------|------|
| 単位 | 分 |
| 初期値 | 5 分 |
| 保存 | `settings.yaml` の `notification.minutesBefore` に整数として保存する |
| IPC | `window.cotaskaAPI.settings.get()` / `update({ notification: { minutesBefore } })` で保存値を扱う |
| 最小値 | 0 分。0 分は事前通知なしではなく、タスク日時到達時の通知として扱う |
| 最大値 | 1440 分（24時間） |
| 未入力 | 保存しない。既存保存値がなければ初期値 5 分を使う |
| 入力制約 | 0 から 1440 までの整数のみ許可する。小数、負数、数値以外、過大値は保存前にエラー表示する |
| 反映先 | `reminderService` の通知判定 |
| 反映タイミング | 保存成功後に renderer 表示を更新し、`reminderService` は次回判定から保存済み設定を参照する。起動時は `settings.yaml` から読み込んだ値で判定する |

### 6.4 文字サイズ

文字サイズはタスク詳細の本文表示、本文編集、Markdownプレビューへ反映する。

| 項目 | 方針 |
|------|------|
| 単位 | px |
| 初期値 | 14px |
| 保存 | `settings.yaml` の `detailTextSize` に整数 px として保存する |
| IPC | `window.cotaskaAPI.settings.get()` / `update({ detailTextSize })` で保存値を扱う |
| 最小値 / 最大値 | 10px から 28px まで |
| 未入力 | 保存しない。既存保存値がなければ初期値 14px を使う |
| 入力制約 | 12 から 22 までの整数のみ許可する。小数、負数、数値以外、範囲外は保存前にエラー表示する |
| 反映先 | `--detail-content-font-size` を使うタスク詳細本文、本文編集、Markdownプレビュー、Markdown編集ハイライト |
| 既存操作 | `Ctrl` + `+` / `Ctrl` + `-` / `Ctrl` + マウスホイールによる変更は同じ設定値を更新し、範囲内で `settings.yaml` へ保存する。既存 localStorage 値がある場合は初回移行元として扱い、正式保存先は `settings.yaml` とする |
| 反映タイミング | 設定画面またはショートカットで保存成功後、選択中タスク詳細へ即時反映し、再起動後は `settings.yaml` から復元する |

### 6.5 外部AIAgent

外部AIAgent は CHG-058 系タスクで実装する。設定画面側では、CHG-058 の実装が進むまで `(未実装) ローカルCLI` の無効表示に留める。

設定画面と CHG-058 の接続が必要になった場合は、AI CLI プロバイダ選択、CLI パス、認証状態、疎通確認などを別途設計する。

## 7. バックアップと復元

バックアップと復元タブは、CHG-068 で手動バックアップ作成の初期機能を実装し、CHG-070 で復元機能を追加する。

| 項目 | 方針 |
|------|------|
| 対象 | `data/tasks/*.md`、`data/lists.yaml`、正式な設定保存ファイル `settings.yaml` |
| 除外 | `_index.yaml` は生成物のため、バックアップ対象外とし、復旧時は再生成対象として扱う |
| 操作 | 利用者が手動でバックアップ作成を実行する |
| 保存先 | 既定では Cotaska.exe と同じディレクトリの `backup` フォルダを作成して使用する。利用者は保存先選択ダイアログで変更できる |
| 復元 | 復元元バックアップzipを選択し、復元前バックアップを作成してからタスク正本、リストマスタ、設定ファイルを戻す |
| 安全策 | タスク正本を破損しないよう、読み取り専用コピー、実行前確認、結果ログを設ける |

### 7.1 バックアップ対象

バックアップ対象はタスク管理に必要な正本データと設定データに限定する。

| 種別 | 対象 | 備考 |
|------|------|------|
| タスク正本 | `data/tasks/*.md` | サブフォルダ配下のタスクファイルを含める |
| リストマスタ | `data/lists.yaml` | リスト・タグ定義を含む |
| 設定 | `settings.yaml` | 設定画面の正式保存先 |
| 生成物 | `data/tasks/_index.yaml` | 対象外。必要に応じて再生成する |
| 削除済みタスク | `data/archive/*.md` | 初期実装では対象外。必要性が出た場合に拡張する |

### 7.2 保存形式と保存先

バックアップ作成では、選択された保存先に日時を含む `Cotaska-backup-*.zip` を作成する。zip 内には `manifest.json` と `data/tasks`、`data/lists.yaml`、`data/settings.yaml` を格納する。

保存先の既定値は、Cotaska.exe と同じディレクトリに作成する `backup` フォルダとする。利用者が保存先を変更したい場合は保存先選択ダイアログで選択する。前回保存先の記憶は初期実装の必須要件にしないが、設定保存基盤が利用可能な場合は後続拡張候補とする。

### 7.3 実行状態と結果表示

| 状態 | 表示・動作 |
|------|------------|
| 初期表示 | Cotaska.exe と同じディレクトリの `backup` フォルダを保存先として表示する |
| 保存先選択済み | 選択した保存先を表示し、作成操作を有効化する |
| 作成中 | 操作を一時的に無効化し、処理中であることを表示する |
| 成功 | 作成先、作成日時、対象ファイル数を表示する |
| 失敗 | タスク正本を変更せず、失敗理由を表示する |
| 復元元選択済み | 選択したバックアップzipを表示し、復元操作を有効化する |
| 復元成功 | 復元元と復元前バックアップの保存先を表示する |

### 7.4 安全策

- バックアップ処理は対象ファイルを読み取り、バックアップ先へzipアーカイブするだけとし、タスク正本へ書き戻さない。
- 実行前に保存先と対象概要を確認できるようにする。
- 実行中は同じバックアップ操作の多重実行を防ぐ。
- 失敗時は途中成果物を成功扱いにしない。可能であれば一時名で作成後、完了時に正式名へ切り替える。
- バックアップ作成結果は画面表示し、必要に応じてログへ記録する。

### 7.5 復元機能の扱い

復元は既存タスク正本を上書きまたは差し替える可能性があるため、利用者確認を必須とする。復元前には現在のデータを既定の `backup` フォルダ配下へ退避し、復元元に `data/tasks` がない場合は実行しない。

初期復元対象は `data/tasks`、`data/lists.yaml`、`data/settings.yaml` とする。`data/tasks/_index.yaml` は生成物のため復元対象外とし、復元後にタスクキャッシュと `_index.yaml` を再構築する。

## 8. 設定保存モデル

設定値はタスクデータとは分離して保存する。

| 項目 | 方針 |
|------|------|
| タスク `.md` | 設定値を保存しない |
| `_index.yaml` | 設定値を保存しない。生成物として扱う |
| localStorage | 一時的または互換移行元として扱う。正式な設定保存先にはしない |
| 正式保存先 | `settings.yaml` |
| バックアップ対象 | 正式保存先を T-0184 のバックアップ対象に含める |

保存データの論理構造は以下とする。`settings.yaml` では日本語コメントを許可し、存在しないキーは既定値で補完する。

```yaml
# 表示名: 画面タイトルやアプリ情報に表示する名前
displayName: Cotaska

# 外部エディタ: タスク Markdown を開く実行ファイルの絶対パス。空欄なら OS 既定アプリ
externalEditorPath: ""

notification:
  # 予定時刻の何分前に通知するか。0 は時刻到達時
  minutesBefore: 5

# タスク詳細本文の文字サイズ(px)
detailTextSize: 14

externalAiAgent:
  provider: null
  cliPath: ""
```

## 9. IPC/API

renderer は context isolation の前提を守り、設定値の読み書きを main process へ直接依存しない。

### 9.1 実装済み API

```javascript
window.cotaskaAPI.app.getInfo()
```

### 9.2 追加予定 API

T-0178 で以下を追加候補とする。

```javascript
window.cotaskaAPI.settings.get()
window.cotaskaAPI.settings.update(partialSettings)
window.cotaskaAPI.settings.reset(keys)
```

設定保存 API は `settings.yaml` を main process 側で読み書きする。renderer は `settings.yaml` のパス、YAML パース、外部アプリ起動を直接扱わない。

外部エディタ参照用には、T-0180 で以下を追加候補とする。

```javascript
window.cotaskaAPI.settings.selectExternalEditor()
```

更新確認用には、T-0183 で以下を追加候補とする。

```javascript
window.cotaskaAPI.app.checkForUpdates()
window.cotaskaAPI.app.openDownloadPage()
window.cotaskaAPI.updates.check()
window.cotaskaAPI.updates.download()
window.cotaskaAPI.updates.install()
window.cotaskaAPI.updates.onStatus(callback)
```

バックアップ用には、T-0184 で以下を追加候補とする。

```javascript
window.cotaskaAPI.backup.create()
window.cotaskaAPI.backup.selectDestination()
window.cotaskaAPI.backup.restore()
window.cotaskaAPI.backup.selectRestoreSource()
```

復元用 API は初期実装では追加しない。

## 10. エラー表示

| ケース | 表示・動作 |
|--------|------------|
| アプリ情報取得失敗 | 設定画面は開いたまま、取得失敗メッセージを表示する |
| 設定保存失敗 | 保存できなかったことを表示し、入力値を失わない |
| 外部エディタ起動失敗 | 対象パスと失敗理由を利用者に分かる形で表示する |
| 更新確認失敗 | 通常利用を妨げず、再試行できる状態にする |
| バックアップ失敗 | タスク正本を変更せず、失敗理由を表示する |
| 復元操作 | 初期実装では提供しない |

## 11. テスト観点

- サイドバー下部の設定アイコンから設定画面を開けること。
- `アプリ情報` / `設定` / `バックアップと復元` を切り替えられること。
- 現在バージョンが `20_app/package.json` の `version` と一致すること。
- renderer が Node.js API に直接依存せず、preload 経由でアプリ情報と設定値を扱うこと。
- 設定画面を開閉しても、タスク一覧、検索、詳細編集、保存処理に影響しないこと。
- 設定値保存後、再起動しても値が復元されること。
- 表示名を変更すると、ウィンドウタイトルとアプリ情報表示に反映され、OS ランチャー名や実行ファイル名は変更されないこと。
- 表示名の空欄、40文字超、改行・制御文字、日本語名の入力制約が設計どおり動作すること。
- 外部エディタが空欄の場合、`.md` ファイルの既定アプリで開くこと。
- 有効な外部エディタ絶対パスでは対象 `.md` を指定エディタで開き、不正パスや起動失敗時はエラー表示されること。
- 通知時間は 0 / 5 / 1440 分、未入力、範囲外、数値以外で設計どおり保存・拒否・判定反映されること。
- 文字サイズは 10 / 14 / 28 px、未入力、範囲外、ショートカット操作で設計どおり保存・反映されること。
- バックアップ作成時にタスク正本が破損しないこと。
- バックアップ対象にタスク正本、リストマスタ、設定保存ファイルが含まれ、`_index.yaml` が対象外であること。
- 復元機能が初期実装の UI/API として提供されていないこと。
