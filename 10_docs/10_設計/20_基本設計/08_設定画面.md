# 設定画面

## 1. 目的

Cotaska のアプリ全体に関わる情報表示と設定変更を、タスク管理画面から独立した設定画面として提供する。

設定画面は、左端サイドバーの設定アイコンから開く。画面内には設定カテゴリ用のサイドパネルを持ち、`アプリ情報`、`設定`、`バックアップ` を切り替えて表示する。

## 2. 対象範囲

| 項目 | 内容 |
|------|------|
| 対象画面 | 設定画面 |
| 起動導線 | 左端サイドバー下部の設定アイコン |
| 初期表示 | アプリ情報 |
| 実装済み範囲 | 設定画面の表示枠、アプリ情報の現在バージョン表示、設定タブの入力欄表示、バックアップタブの未実装表示 |
| 未実装範囲 | 設定値の永続化、各設定値の実機能への反映、更新確認、ダウンロードページ表示、バックアップ作成 |

## 3. 画面構成

設定画面は以下の領域で構成する。

| 領域 | 役割 |
|------|------|
| 左端サイドバー | 既存アプリ共通のナビゲーション。設定アイコンをアクティブ表示する |
| 設定カテゴリサイドパネル | `アプリ情報`、`設定`、`バックアップ` を表示し、選択中カテゴリを切り替える |
| 設定内容ペイン | 選択中カテゴリの内容を表示する |

設定画面を表示している間は、タスク一覧、ナビパネル、詳細ペインは非表示とする。設定画面から別カテゴリへ切り替えても、タスクデータは変更しない。

## 4. カテゴリ

| カテゴリ | 状態 | 内容 |
|----------|------|------|
| アプリ情報 | 一部実装済み | 製品名、現在バージョン、更新案内を表示する |
| 設定 | 表示のみ実装済み | 表示名、外部エディタ、通知時間、文字サイズ、外部AIAgent を表示する |
| バックアップ | 未実装 | バックアップ作成、保存先選択、復元導線などを扱う |

## 5. アプリ情報

### 5.1 表示項目

| 項目 | 内容 | 実装状態 |
|------|------|----------|
| 製品名 | `Cotaska` | 実装済み |
| 現在バージョン | `Cotaska 0.1.1` のように表示する | 実装済み |
| 更新案内 | 利用者確認付きの手動ダウンロード案内 | 表示のみ実装済み |

アプリ情報には「実行環境」配下の状態、取得 API、配布フォルダ、更新方式などの詳細行は表示しない。必要になった場合は、詳細設定または診断情報カテゴリとして別途設計する。

### 5.2 バージョン情報取得

現在バージョンは `20_app/package.json` の `version` を正本とする。

renderer は Node.js API や `package.json` に直接アクセスしない。main process でアプリ情報を組み立て、preload bridge 経由で renderer に渡す。

| 項目 | 内容 |
|------|------|
| main IPC | `app:getInfo` |
| preload API | `window.cotaskaAPI.app.getInfo()` |
| 戻り値 | `{ productName, currentVersion, distributionFolder, updateGuidance }` |
| 表示形式 | `Cotaska ${package.version}` |

`distributionFolder` は内部情報として取得可能だが、現行画面では表示しない。

### 5.3 更新確認

初期実装では自動更新を行わない。更新確認とダウンロードページ表示は T-0183 で実装する。

| 操作 | 方針 | 実装タスク |
|------|------|------------|
| 更新を確認 | 最新バージョン取得元へ問い合わせ、現在バージョンと比較する | T-0183 |
| ダウンロードページを開く | 利用者確認後に外部ブラウザで配布ページを開く | T-0183 |
| 取得失敗時 | 通常利用を妨げず、再試行可能なエラー表示にする | T-0183 |

## 6. 設定

設定タブでは、以下の項目を扱う。現時点では入力欄の表示のみ実装済みで、保存と反映は T-0178 以降で実装する。

| 項目 | 内容 | 初期値 | 実装状態 | 実装タスク |
|------|------|--------|----------|------------|
| 表示名 | Cotaska のランチャー表示名の入力欄 | `Cotaska` | 表示のみ | T-0179 |
| 外部エディタ | タスクファイルを外部アプリで開く場合に使用する実行ファイルの絶対パス | 空欄または既定候補 | 表示のみ | T-0180 |
| 通知時間 | タスク日時が近い場合の事前通知時間 | `5` 分 | 表示のみ | T-0181 |
| 文字サイズ | タスク詳細の文字サイズ | `14` px | 表示のみ | T-0182 |
| 外部AIAgent | ローカル CLI 連携先 | `(未実装) ローカルCLI` | 表示のみ | CHG-058 系タスク |

画面上には「デフォルト値」列を表示しない。初期値は設計・実装上の値として扱う。

### 6.1 表示名

表示名は、利用者が Cotaska の表示名を変更したい場合に使う。

| 項目 | 方針 |
|------|------|
| 保存 | T-0178 の設定保存基盤に保存する |
| 反映先 | T-0179 でウィンドウタイトル、ランチャー表示名、アプリ情報表示のどこまで反映するか確定する |
| 入力制約 | 空欄、長すぎる名称、前後空白の扱いを T-0179 で定義する |

### 6.2 外部エディタ

外部エディタは、タスク詳細から `.md` ファイルを外部アプリで開くときに使う。

| 項目 | 方針 |
|------|------|
| 保存 | 絶対パス文字列として保存する |
| 参照ボタン | 実行ファイル選択ダイアログを開く |
| 起動 | 設定済みパスがある場合、その実行ファイルへ対象タスク `.md` パスを引数として渡す |
| 空欄時 | `.md` ファイルの既定のアプリで開く |
| エラー | パス不存在、起動失敗、権限エラーを利用者へ表示する |

既存の `window.cotaskaAPI.shell.openTaskFile(taskId)` は、T-0180 で外部エディタ設定を参照するように拡張する。

### 6.3 通知時間

通知時間は reminderService の通知判定へ反映する。

| 項目 | 方針 |
|------|------|
| 単位 | 分 |
| 初期値 | 5 分 |
| 最小値 | 0 分。0 分は事前通知なし、または時刻一致時通知として扱うか T-0181 で確定する |
| 最大値 | 過大な値で通知が破綻しない上限を T-0181 で定義する |
| 反映先 | `reminderService` の通知判定 |

### 6.4 文字サイズ

文字サイズはタスク詳細の本文表示、本文編集、Markdownプレビューへ反映する。

| 項目 | 方針 |
|------|------|
| 単位 | px |
| 初期値 | 14px |
| 反映先 | `--detail-content-font-size` を使う領域 |
| 既存操作 | `Ctrl` + `+` / `Ctrl` + `-` / `Ctrl` + マウスホイールによる変更と整合させる |
| 保存 | 現行の localStorage 保存を T-0178 の設定保存基盤へ移行するか、互換移行する |

### 6.5 外部AIAgent

外部AIAgent は CHG-058 系タスクで実装する。設定画面側では、CHG-058 の実装が進むまで `(未実装) ローカルCLI` の無効表示に留める。

設定画面と CHG-058 の接続が必要になった場合は、AI CLI プロバイダ選択、CLI パス、認証状態、疎通確認などを別途設計する。

## 7. バックアップ

バックアップタブは、初期実装では未実装表示のみとする。T-0184 で初期機能を実装する。

| 項目 | 方針 |
|------|------|
| 対象 | `data/tasks/*.md`、`data/lists.yaml`、設定保存ファイル |
| 除外 | `_index.yaml` は生成物のため、必要に応じて再生成対象として扱う |
| 操作 | 手動バックアップ作成を初期候補とする |
| 保存先 | 利用者が選択できるようにする |
| 復元 | 初期実装に含めるか、別タスクへ分離するか T-0184 で確定する |
| 安全策 | タスク正本を破損しないよう、実行前確認とログ記録を設ける |

## 8. 設定保存モデル

設定値はタスクデータとは分離して保存する。

| 項目 | 方針 |
|------|------|
| タスク `.md` | 設定値を保存しない |
| `_index.yaml` | 設定値を保存しない。生成物として扱う |
| localStorage | 一時的または互換移行元として扱う。正式な設定保存先にはしない |
| 正式保存先 | T-0178 で設定ファイルまたは Electron のユーザー設定領域として確定する |
| バックアップ対象 | 正式保存先を T-0184 のバックアップ対象に含める |

推奨する保存データの論理構造は以下とする。

```json
{
  "displayName": "Cotaska",
  "externalEditorPath": "",
  "notificationLeadMinutes": 5,
  "detailFontSizePx": 14,
  "externalAiAgent": {
    "provider": null,
    "cliPath": ""
  }
}
```

## 9. IPC/API

renderer は context isolation の前提を守り、設定値の読み書きを main process へ直接依存しない。

### 9.1 実装済み API

```javascript
window.cotaskaAPI.app.getInfo()
```

### 9.2 追加予定 API

T-0178 で以下を追加候補とする。

```javascript
window.cotaskaAPI.settings.get()
window.cotaskaAPI.settings.update(partialSettings)
window.cotaskaAPI.settings.reset(keys)
```

外部エディタ参照用には、T-0180 で以下を追加候補とする。

```javascript
window.cotaskaAPI.settings.selectExternalEditor()
```

更新確認用には、T-0183 で以下を追加候補とする。

```javascript
window.cotaskaAPI.app.checkForUpdates()
window.cotaskaAPI.app.openDownloadPage()
```

バックアップ用には、T-0184 で以下を追加候補とする。

```javascript
window.cotaskaAPI.backup.create()
window.cotaskaAPI.backup.selectDestination()
```

## 10. エラー表示

| ケース | 表示・動作 |
|--------|------------|
| アプリ情報取得失敗 | 設定画面は開いたまま、取得失敗メッセージを表示する |
| 設定保存失敗 | 保存できなかったことを表示し、入力値を失わない |
| 外部エディタ起動失敗 | 対象パスと失敗理由を利用者に分かる形で表示する |
| 更新確認失敗 | 通常利用を妨げず、再試行できる状態にする |
| バックアップ失敗 | タスク正本を変更せず、失敗理由を表示する |

## 11. テスト観点

- サイドバー下部の設定アイコンから設定画面を開けること。
- `アプリ情報` / `設定` / `バックアップ` を切り替えられること。
- 現在バージョンが `20_app/package.json` の `version` と一致すること。
- renderer が Node.js API に直接依存せず、preload 経由でアプリ情報と設定値を扱うこと。
- 設定画面を開閉しても、タスク一覧、検索、詳細編集、保存処理に影響しないこと。
- 設定値保存後、再起動しても値が復元されること。
- 外部エディタが空欄の場合、`.md` ファイルの既定アプリで開くこと。
- 通知時間、文字サイズが設計どおり反映されること。
- バックアップ作成時にタスク正本が破損しないこと。
