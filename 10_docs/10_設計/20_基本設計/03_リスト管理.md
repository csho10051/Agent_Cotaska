# 基本設計 - リスト管理

## 1. 概要

リストはタスクを分類・整理するためのグループ単位。
ユーザが自由に作成・編集・削除でき、タスクは1つのリストに所属する。

---

## 2. リストのデータ項目

| 項目 | 型 | 必須 | 説明 |
|------|----|:---:|------|
| id | INTEGER | ○ | 自動採番（PK） |
| name | TEXT | ○ | リスト名 |
| color | TEXT | - | 表示色（例: #FF5733） |
| sort_order | INTEGER | ○ | ナビパネルでの表示順 |
| created_at | TEXT | ○ | 作成日時 |
| updated_at | TEXT | ○ | 更新日時 |

---

## 3. 特殊リスト

以下はシステム固定のビューであり、listsテーブルには格納しない。

| 名称 | 条件 | 備考 |
|------|------|------|
| すべて | status ≠ done かつ delete_flag = 0 の全タスク | 一覧把握用 |
| リストなし | list_id = NULL のタスク | リスト未割当。左ナビのリストセクション最上部に固定表示 |
| 今日 | due_date = 今日 | 期限ベース |
| 明日 | due_date = 今日+1 | 期限ベース |
| 次の7日間 | due_date が7日以内 | 期限ベース |
| 保留 | progress_status = '保留' かつ status ≠ 'done' | 進捗ステータスベース |
| 完了 | status = 'done' | ステータスベース |
| ゴミ箱 | is_deleted = 1 | 削除フラグベース |

---

## 4. 機能一覧

### 4.1 リスト作成

| 項目 | 内容 |
|------|------|
| トリガー | ナビパネル「＋」ボタン |
| 入力 | リスト名（必須）、色（任意） |
| 入力表示 | 入力中・確定後ともに白文字で表示し、暗色背景上でも可読性を維持する |
| 保存先 | lists テーブル |
| 表示 | ナビパネルのリストセクションに追加 |

### 4.2 リスト編集

| 項目 | 内容 |
|------|------|
| トリガー | リスト名の右クリックまたはホバー時「…」メニュー → 「編集」 |
| 対象 | リスト名、色 |
| 入力表示 | 編集中も白文字表示を維持する |
| 保存 | listsテーブルに反映 |

### 4.3 リスト削除

| 項目 | 内容 |
|------|------|
| トリガー | リスト名の右クリックまたはホバー時「…」メニュー → 「削除」 |
| 確認 | 確認ダイアログ表示 |
| 処理 | リスト削除、所属タスクの list_id を NULL に変更（リストなしへ） |

### 4.4 リスト並び替え

| 項目 | 内容 |
|------|------|
| トリガー | ナビパネルでドラッグ&ドロップ |
| 処理 | sort_order を更新 |
| スコープ | 将来実装 |

---

## 5. タスクとリストの関係

```text
リスト (lists)  1 ──── * タスク (tasks)

- タスクは0または1つのリストに所属
- list_id = NULL → リストなし扱い
- リスト削除時、タスクは受信トレイに戻る（削除しない）
- タスクの所属リストは、詳細ペインだけでなくタスク右クリックメニューの「リスト設定」からも変更できる
```

---

## 6. バリデーション

| 項目 | ルール |
|------|--------|
| リスト名 | 1文字以上、100文字以内 |
| 色 | #RRGGBB形式、またはNULL |
| 重複 | 同名リストは許容する（IDで区別） |
