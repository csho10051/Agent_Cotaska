# 基本設計 - 補助機能

## 1. 概要

タスク管理を支える補助的な機能群。ログ記録、自動保存、並び替えなどを含む。

---

## 2. ログ記録

### 2.1 目的

- タスクの変更履歴を追跡可能にする
- 監査・デバッグ用途で参照可能にする
- 将来のAI連携で変更パターン分析に利用する

### 2.2 ログデータ項目

| 項目 | 型 | 必須 | 説明 |
|------|----|:---:|------|
| id | INTEGER | ○ | 自動採番（PK） |
| task_id | INTEGER | ○ | 対象タスクID（FK） |
| action | TEXT | ○ | 操作種別（後述） |
| details | TEXT | - | 変更内容の詳細 |
| created_at | TEXT | ○ | 記録日時 |

### 2.3 操作種別（action）

| action | 発生タイミング |
|--------|---------------|
| created | タスク新規作成 |
| updated | タスク項目の編集 |
| completed | タスク完了 |
| reopened | 完了取消 |
| trashed | ゴミ箱移動 |
| restored | ゴミ箱から復元 |
| moved | リスト変更 |
| progress_changed | 進捗率変更 |
| commented | コメント追加 |
| reordered | 並び順変更 |

### 2.4 details の記録例

```text
action: updated
details: "title: '旧タスク名' → '新タスク名'"

action: updated
details: "priority: 'medium' → 'high'"

action: moved
details: "list: '業務' → '個人'"

action: progress_changed
details: "progress: 30 → 70"
```

## 3. 自動保存

### 3.1 方針

- 詳細ペインでの編集は **フォーカスアウト時に自動保存**
- 保存ボタンは設けない（操作ステップ削減）
- 保存成功時はUIに控えめなフィードバック（例: 「保存済」表示）

### 3.2 保存タイミング

| イベント | 動作 |
|---------|------|
| テキストフィールドからフォーカスアウト | 変更があれば保存 |
| ドロップダウン変更 | 即時保存 |
| 日付ピッカー変更 | 即時保存 |
| チェックボックス変更 | 即時保存 |
| 進捗バー操作 | 即時保存 |

### 3.3 競合制御

- シングルユーザ前提のため、楽観ロックは不要
- 将来のマルチユーザ対応時に updated_at ベースの楽観ロックを検討

## 4. 並び替え

### 4.1 方針

- タスク一覧内の表示順は sort_order カラムで管理
- 初期実装ではUI上の並び替え操作は省略
- デフォルトソート（優先度順、日付順）で運用

### 4.2 将来実装

- ドラッグ&ドロップによる手動並び替え
- sort_order の再計算ロジック

## 7. タグ機能

### 7.1 背景

- タスクに対する柔軟なカテゴリ分けを可能にする
- リスト分類とは異なる、複数の属性軸でのグループ化
- AI が自動タグ付けする際の拡張性

### 7.2 仕様

**データ構造**：
- `lists.yaml` の `tags` セクションで タグマスタ管理
- 各タスクの frontmatter `tags: []` 配列で紐付け
- タグは文字列型（例：`["urgent", "client", "review"]`）

**操作**：
- タグ追加・削除（タスク単位）
- タグマスタの CRUD（リスト全体管理）
- タグ選択時のフィルタリング

### 7.3 UI フロー

| UI 要素 | 動作 |
|--------|------|
| 詳細ペイン | タグ表示・追加・解除 UI（チップ＋ドロップダウン） |
| 右クリックメニュー | 「タグ設定」で素早くタグ変更 |
| ナビパネル | タグ一覧と件数バッジ表示・タグ選択で絞り込み |

### 7.4 制限

- タグ名は英数字と `-` で構成（スペース・特殊文字なし）
- 1 タスク最大 10 タグまで
- `lists.yaml` 更新時に自動デデュプ（重複削除）

### 7.5 実装方針（frontmatter ベース）

**Service層**：
- `listService.js` に `addTag(name)` / `removeTag(name)` / `getAllTags()` 実装
- `taskService.js` に `updateTaskTags(taskId, tags)` 実装

**IPC**：
- `tags:getAll` / `tags:add` / `tags:delete`
- `taskTags:set(taskId, tags)`

**Renderer**：
- `preload.js` に `window.cotaskerAPI.tags.*` と `window.cotaskerAPI.taskTags.set()`
- `App.jsx` / `DetailPane.jsx` / `NavPanel.jsx` に UI 実装
