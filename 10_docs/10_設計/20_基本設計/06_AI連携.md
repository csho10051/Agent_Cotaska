# 基本設計 - AI連携

## 1. 概要

AIエージェント（Codex等）が本システムのタスクデータを参照・編集でき、UIがリアルタイムに同期される仕組みを提供する。

**アーキテクチャ（2026-03-18 改訂）**：
- タスクが個別Markdownファイルなため、AIが直接ファイルを読み書き可能
- `_index.yaml` で高速なタスク一覧取得を実現
- `fs.watch` + watcher.js でリアルタイム双方向同期
- Markdown本文をAIが直接編集するため、「Markdown出力」は達成済み

## 3. AI連携の中核：双方向同期

### 3.1 データフロー

```
┌─ AIエージェント（VSCode）────────┐
│                                    │
│ 30_data/tasks/ の .md ファイルを    │
│ 直接編集（frontmatter + 本文）      │
│                                    │
└────────────┬──────────────────────┘
             │ fs.writeFileSync()
             ▼
   30_data/tasks/T-XXX.md（更新）
             │
             ├─→ watcher.js 検知
             │    (chokidar)
             │
             ▼
   taskService.js → キャッシュ更新
             │
             ├─→ indexService.js
             │    (_index.yaml 再生成)
             │
             ▼
   win.webContents.send('tasks:changed')
             │
             ▼
┌─ Renderer プロセス（UI）───────────┐
│                                     │
│ tasks:changed イベント受信           │
│  → tasks:getAll 呼び出し            │
│  → リスト表示をリアルタイム更新       │
│                                     │
└─────────────────────────────────────┘
```

### 3.2 処理フロー（詳細）

1. **AIがファイルを編集**
   ```markdown
   # 30_data/tasks/T-001.md
   ---
   status: doing → blocked  # 変更
   progress_status: 未着 → 仕掛  # 変更
   ---
   タスク本文の編集...
   ```

2. **watcher が検知**（chokidar）
   - ファイル変更イベントを捕捉
   - taskService に通知

3. **キャッシュ更新**
   - taskService が新ファイルを読み込み
   - メモリキャッシュを更新

4. **インデックス再生成**
   - indexService が `_index.yaml` を上書き
   - タスク一覧のスナップショット保存

5. **UI通知**
   - `tasks:changed` を renderer に送信
   - renderer が自動的に `tasks:getAll` を呼び出し
   - 画面のタスク一覧がリアルタイム更新

## 4. アクセス方法

### 4.1 AIがタスク情報を読み取る

**高速読み取り（推奨）**：
```yaml
# 30_data/tasks/_index.yaml（自動生成・読み取り専用）
tasks:
  - id: T-001
    title: タスク名
    status: todo
    priority: high
    due_date: 2026-03-20
    list: リスト名
    created_at: 2026-03-18T10:00:00
  - id: T-002
    ...
last_updated: 2026-03-18T15:30:00
next_task_id: 1000
```

**詳細読み取り**：
```markdown
# 30_data/tasks/T-001.md（フルデータ）
---
id: T-001
title: ...
...（全frontmatter）
---
Markdown本文...
```

### 4.2 AIがタスク情報を書き込む

**推奨方法**：frontmatter + 本文を編集
```markdown
---
id: T-001
title: 修正後のタスク名
status: doing
progress_status: 仕掛
updated_at: 2026-03-18T16:00:00
---

修正後の本文内容

- 詳細な記述
- チェックリスト
```

**禁止事項**：
- `_index.yaml` の直接編集（自動上書きされる）
- `id` フィールドの変更（タスク識別ができなくなる）
- ファイル名を T-XXX.md 形式以外に変更

## 5. リスト・タグ管理

### 5.1 リスト

```yaml
# 30_data/lists.yaml
lists:
  - name: 仕事
    color: blue
    created_at: 2026-03-18T10:00:00
  - name: 個人
    color: green
    created_at: 2026-03-18T10:00:00
```

AIが新しいリストを追加するには、このファイルに直接追記。アプリが自動で同期する。

### 5.2 タグ

```yaml
# 30_data/lists.yaml の tags セクション（T-032以降）
tags:
  - name: urgent
    color: red
  - name: client
    color: orange
```

タスクフロントマターの `tags: [urgent, client]` と連動。

## 6. UIとAIの協働パターン

### パターン1：AIが複数タスクを作成

```
AI → 30_data/tasks/ に複数の .md ファイルを配置
  ↓
watcher 検知 → indexService で _index.yaml 更新
  ↓
UI は `tasks:changed` で自動リロード
```

### パターン2：AIがタスクを更新

```
AI → 30_data/tasks/T-XXX.md を編集（status, progress等）
  ↓
watcher 検知 → キャッシュ & インデックス更新
  ↓
UI は `tasks:changed` で自動リロード
  ↓
UIしたがって changed イベントハンドラで tasks:getAll を呼び出す
  ↓
画面に反映（ステータス変更、優先度更新等）
```

### パターン3：AIが分析・提案を UX に埋め込む

```
AI → _index.yaml を読み取り
  ↓
優先度・期限を分析
  ↓
CURRENT_SPRINT.md に「今日のフォーカスタスク」を記述
  ↓
開発者が CURRENT_SPRINT.md を確認
  ↓
参考情報として活用
```

## 7. 設計上の注意

| 項目 | ルール |
|------|--------|
| ファイル命名 | `T-XXXX.md` 形式固定（小文字は不可） |
| frontmatter | YAML形式、各フィールドの型を厳守 |
| 本文 | Markdown自由記述、外部リンク・画像も可 |
| 同期タイミング | fs.watch で検知（最大1秒程度の遅延） |
| 競合対策 | 同時更新は last-write-wins（上書き）で対応 |
| バージョン管理 | Git で差分追跡が容易 |

## 8. サービス層インターフェース

### watcher.js

```javascript
// Main プロセス
watcher.startWatcher(win)
// 30_data/tasks/ を監視開始
// ファイル追加 / 変更 / 削除を検知
// taskService のキャッシュ更新 → indexService で再生成
// win.webContents.send('tasks:changed') で notify
```

### taskService.js

```javascript
// AI がファイルを作成・編集した場合も対応
tasks = taskService.getAllTasks()
// watcher 経由で更新されたキャッシュから取得

// ただし、アプリが直接更新する場合は IPC 経由
taskService.updateTask(id, { status: 'doing', progress_status: '仕掛' })
// → ファイル上書き
// → watcher が検知（自ファイルの書き込みもイベント対象）
// → renderer に task:changed を送信
```
