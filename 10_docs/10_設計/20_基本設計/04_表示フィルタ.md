# 基本設計 - 表示・フィルタ

## 1. 概要

ナビパネルの選択に応じてメインペインのタスク一覧を切り替える機能。
ビュー（表示条件）ごとに異なる抽出ロジックを持つ。

---

## 2. ビュー定義

### 2.1 システムビュー（固定）

| ビュー名 | 抽出条件 | セクション分け | 初期スコープ |
|----------|----------|:---:|:---:|
| すべて | is_deleted=0 AND status≠done | なし | ○ |
| 今日 | due_date = 今日 OR （親due_date = 今日の直下サブタスク） AND is_deleted=0 AND status≠done | 遅延 / 今日 | ○ |
| 明日 | due_date = 今日+1 AND is_deleted=0 AND status≠done | なし | ○ |
| 次の7日間 | due_date ≤ 今日+7 AND is_deleted=0 AND status≠done | 遅延 / 今日 / 明日 / 以降 | ○ |
| リストなし | list_id IS NULL AND is_deleted=0 AND status≠done | なし | ○ |
| 完了 | status = 'done' AND is_deleted=0 | なし | ○ |
| ゴミ箱 | is_deleted = 1 | なし | ○ |

### 2.2 リストビュー（ユーザ作成）

| ビュー名 | 抽出条件 | セクション分け |
|----------|----------|:---:|
| ユーザ作成リスト | list_id = 指定ID AND is_deleted=0 AND status≠done | なし |

---

## 3. セクション分けロジック

「今日」「次の7日間」ビューでは、タスクをセクションに分割して表示する。

### 今日ビュー

| セクション | 条件 |
|-----------|------|
| 遅延 | due_date < 今日 AND status ≠ done |
| 今日 | due_date = 今日、または親タスクの due_date = 今日である直下サブタスク |

### 次の7日間ビュー

| セクション | 条件 |
|-----------|------|
| 遅延 | due_date < 今日 AND status ≠ done |
| 今日 | due_date = 今日 |
| 明日 | due_date = 今日+1 |
| 以降 | 今日+2 ≤ due_date ≤ 今日+7 |

---

## 4. ソート

### デフォルトソート

| ビュー | ソート順 |
|--------|---------|
| すべて | sort_order |
| 今日 / 次の7日間 | セクション順 → sort_order（未設定時は優先度 high→low → id） |
| 明日 | sort_order（未設定時は優先度 high→low → id） |
| リストなし / リスト | sort_order |
| 完了 | completed_at DESC |
| ゴミ箱 | deleted_at DESC |

### 4.1 ドラッグ並び替え時のソートルール（CHG-021）

| 条件 | 並び順ルール |
|------|-------------|
| ドラッグ有効ビュー（すべて / 今日 / 明日 / 次の7日間 / リスト / リストなし） | `sort_order` を最優先 |
| `sort_order` 未設定タスクを含む場合 | 表示時に末尾へ寄せ、保存時に連番再採番 |
| 完了 / ゴミ箱 / 検索ビュー | ドラッグ無効（既存ソート維持） |

---

## 4.2 セクション間ドラッグ移動時の自動更新（CHG-021）

### 日付セクション

| 移動元 → 移動先 | 自動更新 |
|----------------|----------|
| 今日 → 明日 | `due_date = 明日` |
| 明日 → 今日 | `due_date = 今日` |
| 期限未設定 → 今日 | `due_date = 今日` |
| 期限未設定 → 明日 | `due_date = 明日` |

### 進捗セクション

| 移動元 → 移動先 | 自動更新 |
|----------------|----------|
| 未着 → 仕掛 | `progress_status = 仕掛` |
| 仕掛 → 未着 | `progress_status = 未着` |

### 優先順位

1. 完了整合（`status = done` は未着/仕掛への移動禁止）
2. ドロップ先セクション意味の反映（`due_date` / `progress_status`）
3. 手動設定値の保持（1,2と矛盾しない範囲）

---

## 4.3 適用範囲と制約

- 親子構造は維持し、親子関係を変更するドラッグは初期スコープ外
- サブタスクは同一親配下での並び替えのみ許可
- 連続ドラッグ時は操作単位で原子的に保存し、部分反映を残さない

---

## 5. 検索（次段階）

| 項目 | 内容 |
|------|------|
| 対象 | タスクタイトル、メモ |
| 方式 | LIKE検索（部分一致） |
| 範囲 | 通常タスク（ゴミ箱除外） |
| UI | サイドバーの検索アイコンから検索ビューを開く |

---

## 6. フィルター（次段階）

| フィルタ項目 | 選択肢 |
|-------------|--------|
| ステータス | todo / doing / blocked / done |
| 優先度 | high / medium / low |
| 期限 | 期限あり / 期限なし / 期限超過 |
| タグ | タグ一覧から選択 |

フィルタは組み合わせ（AND条件）で適用する。

---

## 7. サービス層インターフェース

```text
FilterService
  ├ getAllActiveTasks()
  ├ getTodayTasks()
  ├ getTomorrowTasks()
  ├ getNext7DaysTasks()
  ├ getNoListTasks()
  ├ getCompletedTasks()
  ├ getTrashedTasks()
  ├ getTasksByList(list_id)
  ├ searchTasks(keyword)           ← 次段階
  └ filterTasks(conditions)        ← 次段階
```
