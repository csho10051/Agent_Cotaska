# テーブル定義 - タスクログ

## 基本情報

| 項目 | 値 |
|------|-----|
| 論理名 | タスクログ |
| 物理名 | log_task |
| 用途 | ログ |
| 概要 | タスクの変更履歴を記録し、操作追跡・AI分析に利用する |

---

## カラム定義

| # | 論理名 | 物理名 | 型 | NOT NULL | デフォルト | 備考 |
|---|--------|--------|-----|:---:|-----------|------|
| 1 | ID | id | INTEGER | ○ | 自動採番 | PK |
| 2 | タスクID | task_id | INTEGER | ○ | | FK → trx_task.id |
| 3 | 操作種別 | action | TEXT | ○ | | 後述の操作種別一覧 |
| 4 | 変更詳細 | details | TEXT | | | 変更内容の説明 |
| 5 | 記録日時 | created_at | TEXT | ○ | | ISO 8601 |

---

## 制約

| 制約名 | 種別 | 対象カラム | 備考 |
|--------|------|-----------|------|
| pk_log_task | PRIMARY KEY | id | |
| fk_log_task_task_id | FOREIGN KEY | task_id | → trx_task(id) ON DELETE CASCADE |

---

## インデックス

| インデックス名 | 対象カラム | 備考 |
|---------------|-----------|------|
| ix_log_task_task_id | task_id | タスク別のログ取得 |
| ix_log_task_created_at | created_at | 時系列でのログ取得 |

---

## 操作種別（action）一覧

| action | 発生タイミング |
|--------|---------------|
| created | タスク新規作成 |
| updated | タスク項目の編集 |
| completed | タスク完了 |
| reopened | 完了取消 |
| trashed | ゴミ箱移動 |
| restored | ゴミ箱から復元 |
| moved | リスト変更 |
| progress_changed | 進捗率変更 |
| commented | コメント追加 |
| reordered | 並び順変更 |

---

## details の記録例

```text
action: updated
details: "title: '旧タスク名' → '新タスク名'"

action: updated
details: "priority: 'medium' → 'high'"

action: moved
details: "list: '業務' → '個人'"

action: progress_changed
details: "progress: 30 → 70"
```

---

## 備考

- ログは追記のみ（更新・削除しない）
- タスクの物理削除時は CASCADE で自動削除
- 将来的に詳細ペインの「変更履歴」セクションに表示
- AI連携での変更パターン分析に利用予定
