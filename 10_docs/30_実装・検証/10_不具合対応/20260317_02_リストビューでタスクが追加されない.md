## 現象

ナビパネルでユーザー作成リストを選択した状態でクイック追加バーからタスクを追加しようとしても、タスクがリストに表示されない。

---

## 再現手順

1. アプリを起動する
2. ナビパネルでユーザー作成リスト（例：「マイリスト」）を選択する
3. クイック追加バーにタスク名を入力して Enter を押す
4. タスクがリストに表示されない

---

## 期待動作

Enter キー押下後、入力したタイトルで当該リストにタスクが登録され、リスト内に表示される。

---

## 調査メモ

### 原因（特定済み）

**`handleAddTask` がリストビュー時に `list_id` を設定していない。**

```js
// App.jsx の handleAddTask（現状）
await window.cotaskerAPI?.tasks?.add({
  title:    title.trim(),
  status:   "todo",
  priority: "normal",
  progress: 0,
  due_date,           // ← BUG-20260317-01 で追加済み
  // list_id が未設定 → DB に NULL で保存される
});
```

- タスクは DB に `list_id = NULL` で保存される（受信トレイ扱い）
- リストビューの `visibleTasks` は `list_id` でフィルタするため、追加したタスクが表示されない

### 関連する別問題（同時修正推奨）

`App.jsx` の `onAddTask` / `onToggleComplete` / `onTrashTask` の条件式に typo がある。

```js
// 現状（誤）
activeNav !== "ゴミ笹"

// 正しくは
activeNav !== "ゴミ箱"
```

この typo により、ゴミ箱ビューでも onAddTask などが意図せず有効になっている可能性がある。

---

## 修正方針

`handleAddTask` でアクティブなビューがリスト名の場合、対応する `list_id` を取得して設定する。

```js
// 修正案
const listObj = lists.find((l) => l.name === activeNav);
const list_id = listObj ? listObj.id : null;

await window.cotaskerAPI?.tasks?.add({
  title, status, priority, progress, due_date, list_id,
});
```

あわせて `"ゴミ笹"` → `"ゴミ箱"` の typo 修正も同時に行う。
