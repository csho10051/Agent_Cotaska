## 現象

詳細ペインのメモ本文（textarea）に文字を入力しても、アプリを再起動すると入力内容が消える。タイトル欄の変更は正常に保存される。

---

## 再現手順

1. アプリを起動する
2. 任意のタスクをクリックして詳細ペインを開く
3. メモ本文（textarea）に任意のテキストを入力する
4. 500ms 以上待つ（debounce 自動保存が走るタイミング）
5. アプリを再起動する
6. 同じタスクの詳細ペインを開く
7. → メモ本文が空欄になっている（入力前の状態に戻っている）

---

## 期待動作

メモ本文に入力した内容が 500ms 後（debounce）に自動保存され、再起動後も保持されること。

---

## 調査結果

### 原因

`20_app/src/main/taskService.js` の `updateTask()` 関数内に以下の除外条件が存在した。

```js
// 修正前（問題のあるコード）
Object.keys(updates).forEach(key => {
  if (key !== 'content') {   // ← content を除外していた
    task[key] = updates[key];
  }
});
```

`DetailPane.jsx` の `debouncedSave` は `tasks:update` IPC に `content` フィールドを含めて送信していたが、サービス層で無条件に無視されていたため、メモ本文への変更がキャッシュにもファイルにも反映されなかった。

タイトルは `title` キーで送られるため除外条件の対象外となり、正常に保存されていた。

### 影響範囲

- メモ本文（`content`）の変更のみ非保存
- タイトル・優先度・進捗・タグ等は正常に保存されていた

---

## 修正内容

`taskService.js` の `updateTask()` から `content` の除外条件を削除し、全フィールドを一律更新するよう変更した。

```js
// 修正後
Object.keys(updates).forEach(key => {
  task[key] = updates[key];
});
```

`writeTaskFile()` はもともと `task.content` を gray-matter の body として正しく書き出す設計になっているため、この変更により content もファイルに永続化される。
