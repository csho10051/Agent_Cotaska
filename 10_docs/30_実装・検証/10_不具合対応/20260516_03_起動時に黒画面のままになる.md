## 現象

`Cotaska/00_mgmt/Cotaska_タスク管理ツール/Cotaska.exe` を起動すると、ウィンドウが表示されるものの画面が黒いままになり、タスク管理画面が描画されない。

## 再現手順

1. `Cotaska/00_mgmt/Cotaska_タスク管理ツール/Cotaska.exe` を起動する。
2. 起動後のウィンドウ表示を確認する。

## 期待動作

Cotaska のメイン画面が正常に描画され、タスク一覧や詳細ペインを操作できる。

## 調査メモ

`CotaskaCore.exe --enable-logging --v=1` で起動して確認したところ、メインプロセスは起動し、IPC ping も成功していた。一方でレンダラー側に次の例外が出ていた。

```text
Uncaught ReferenceError: normalizeProgressStatusValue is not defined
```

`App.jsx` では `normalizeProgressStatusValue` を使用していたが、`taskViewModel.js` から import していなかった。React の描画中に例外が発生し、画面描画が止まったことが黒画面の直接原因と判断した。

## 修正方針

`20_app/src/renderer/App.jsx` の `taskViewModel` import に `normalizeProgressStatusValue` を追加する。

## 関連ファイル

- `20_app/src/renderer/App.jsx`
- `20_app/src/renderer/lib/taskViewModel.js`

## 検証結果

- `20_app/src/renderer/App.jsx` に `normalizeProgressStatusValue` の import を追加した。
- Node.js v24.14.0 環境で `vite build` が成功した。
- `release-all.ps1 -Version "0.2.1"` が成功し、修正版の配布物を生成した。
- `sync-task-master-release.ps1` が成功し、タスク管理ツール配下の `Cotaska.exe` / `_app` を修正版へ反映した。
- 修正版の `_app/CotaskaCore.exe` を起動し、DevTools Protocol 経由でメイン画面 DOM とスクリーンショット描画を確認した。
- 起動後ログで `normalizeProgressStatusValue is not defined` の再発がないことを確認した。
