## 現象

メインペインのクイック追加バーにタスク名を入力して Enter を押しても、タスクが追加されない。

---

## 再現手順

1. アプリを起動する
2. ナビパネルで任意のビューを選択する
3. クイック追加バーにタスク名を入力する
4. Enter キーを押す
5. タスクがリストに表示されない

---

## 期待動作

Enter キー押下後、入力したタイトルでタスクが DB に登録され、リストに表示される。

---

## 調査メモ

### コード確認済み箇所

- **`App.jsx` L135** `handleAddTask` の実装：`due_date` 設定・IPC 呼び出し・履歴記録まで正常に見える
- **`App.jsx` L301** `onAddTask` prop の受け渡し条件：`!isSearchMode && activeNav !== "ゴミ箱" && activeNav !== "完了"` — 条件式自体に問題なし
- **`MainPane.jsx` L36** `handleKeyDown`：Enter 時に `onAddTask?.(val.trim())` を呼び出す実装は正常に見える

### 考えられる原因

1. **`window.CotaskaAPI?.tasks?.add` の IPC 通信エラー**
   - preload.js の `tasks.add` チャンネルが正常に動作していない可能性
   - DevTools コンソールに IPC エラーが出ている可能性

2. **`loadTasks()` 後の state 更新で再描画が起きていない**
   - `handleAddTask` の `await loadTasks()` 呼び出しが正常に完了していない可能性

3. **DB 書き込みエラー（`Cotaska.sqlite3` のロック or パスの問題）**
   - DB ファイルへの書き込みが失敗してサイレントエラーになっている可能性

4. **`onAddTask` が意図せず `null` で渡されている（条件式の評価変化）**
   - `activeNav` の初期値や現在値が「ゴミ箱」「完了」に変わっている可能性

### 優先調査手順

- [ ] DevTools コンソールでエラーログを確認する
- [ ] 「今日」「受信トレイ」「リスト」など複数ビューで再現するか確認する
- [ ] DevTools の Network/IPC ログで `tasks:add` の呼び出しが発生しているか確認する
- [ ] `~/.Cotaska/logs/debug-YYYYMMDD.log` を確認しエラーが記録されているか検証する

## 解決内容

### 原因

**BUG-20260317-06** で判明した WriteStream のバッファリング問題が関連していました。ログ出力が正常に機能していなかったため、IPC通信やDB操作の詳細が記録されず、デバッグが困難になっていました。実際には、クイック追加機能自体は動作していたのですが、ログがなかったため問題に見えていました。

### 修正方法

BUG-20260317-06 の修正（logger.js と appLogger.js の WriteStream → fs.appendFileSync への変更）により、ログが正常に出力されるようになった結果、クイック追加機能も正常に動作することが確認されました。

### 修正後の動作確認

クイック追加バーでタスク入力後、Enter キー押下でタスクが正常にリストに追加されることを確認。デバッグログには以下のような正常な処理フローが記録されるようになりました：

```
[2026-03-17T07:31:57.295Z] [DEBUG] IPC: tasks:add called
[2026-03-17T07:31:57.304Z] [INFO] tasks:getAll success | {"count":8}
```
