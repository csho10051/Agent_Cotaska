# 20260516_01 バックアップ復元時EPERM

## 概要

バックアップ復元時に `data/tasks` ディレクトリ自体を削除しようとして、Windows 環境で `EPERM: permission denied` が発生した。

## 影響

- 復元処理が途中で停止する。
- 失敗タイミングによっては `data/tasks` 配下のタスク正本が一時的に空になる可能性がある。

## 原因

復元処理が `fs.rmSync(data/tasks, { recursive: true })` で監視対象ディレクトリ自体を削除していた。chokidar 監視や Windows のディレクトリハンドルにより、ディレクトリ削除が拒否される場合がある。

## 対応

- 復元時は watcher を一時停止する。
- `data/tasks` ディレクトリ自体は削除せず、配下のエントリだけを削除してから復元元をコピーする。
- 削除処理には retry を設定する。
- 復元後に watcher を再起動し、タスクキャッシュと `_index.yaml` を再構築する。
- 失敗で空になった `data/tasks` は復元前バックアップから復旧した。

## 検証

- Node 22.14.0 で `npm.cmd run build` 成功。
- `data/tasks` に 80 件のタスク Markdown が復旧していることを確認。
- デバッグ環境を再起動し、起動ログで `Services initialized | {"tasks":80,...}` を確認。
