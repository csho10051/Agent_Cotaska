## 現象

`Cotaska\00_mgmt\Cotaska_タスク管理ツール\Cotaska.exe` を起動すると、タスクが何も表示されない。

## 再現手順

1. `Cotaska\00_mgmt\Cotaska_タスク管理ツール\Cotaska.exe` を起動する
2. メイン画面のタスク一覧を確認する
3. タスクが0件表示になる

## 期待動作

`data/tasks/*.md` に存在するタスクが一覧に表示される。

## 調査メモ

- 2026-03-26 12:xx 時点で起票。
- 直前に `sync-task-master-release.ps1` による配布差し替えを実施しているため、差し替え手順の影響も含めて確認する。
- `launcher.log` 上は起動自体は成功（child process start success）。
- `logs/app-2026-03-26.log` に以下の致命エラーを確認。
	- `YAMLException: can not read a block mapping entry; a multiline key may not be an implicit key at line 22`
	- エラー先頭行: `### 処理概要`
- 該当ファイルを特定: `00_mgmt/Cotaska_タスク管理ツール/data/tasks/T-0028.md`
- `T-0028.md` のfrontmatter終了区切り `---` が欠落しており、gray-matter が本文見出しをYAMLとして解釈して例外化。

## 原因

`T-0028.md` のfrontmatter破損（終了デリミタ `---` 欠落）。
これにより起動時の全件ロードで例外が発生し、タスク一覧取得処理が失敗。

## 対応内容

- `T-0028.md` の `deleted_at: null` の直後にfrontmatter終了 `---` を追加。
- Node + gray-matter で `data/tasks/*.md` 全件パース検証を実施し、`OK 28 markdown files parsed` を確認。

## 検証結果

- パースエラーは解消済み。
- 次回 `Cotaska.exe` 起動時に一覧復旧する見込み。
- 実機UI表示確認はユーザ確認待ち。
