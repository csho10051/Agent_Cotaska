## 現象

左ナビゲーション（左上）の「明日」ビューで、親タスクの期限日が「今日」の場合、配下のサブタスクの期限日が明日ではないにもかかわらず、「明日」ビューに表示されている。

## 再現手順

1. アプリを起動する
2. 親タスク（期限日 = 今日）を作成する
3. その親タスク配下にサブタスク（期限日 = 別の日付、明日ではない）を作成する
4. 左ナビの「明日」ビューをクリックする
5. サブタスク（期限日が明日ではないもの）が表示される

## 期待動作

「明日」ビューでは、期限日が厳密に明日に設定されたタスク（親・子問わず）のみが表示される。

**ただし、CHG-018 の設計意図によっては：**
- 親タスクのみが今日期限で、配下のサブタスクが親の期限を継承表示されるべき場合もある
- その場合は、親タスク期限が今日でも、配下のサブタスクを「親の期限に応じて表示」する仕様をコード化する必要がある

## 実際の動作

- サブタスクの実期限日（明日以外）に関わらず、親期限の影響で「明日」ビューに表示される

## 調査メモ

### 原因の推測

- `App.jsx` の 「明日」ビューフィルタロジック（`buildSections()` 内の `次の7日間` ビュー処理）で、親タスク期限のみを判定している可能性
- 子タスク（サブタスク）の期限日が「明日」でない場合も、親タスク期限が「明日」である親タスク配下の子タスクが表示される可能性

### 参照タスク

- CHG-018: ビュー構成・フィルタリング関連の仕様変更（詳細は 仕様変更管理 を参照）

## 根本原因と修正内容

### 根本原因

**`App.jsx` の `buildSections()` 関数内の「次の7日間」ビュー処理で、「今日」セクションは親子関係（todayChildIds）を考慮しているが、「明日」セクションは考慮していなかった。**

以下のコードで、「今日」セクション（line 110）は：
```javascript
const todayChildIds = new Set(
  allTasks.filter((t) => t.parent && todayParentIds.has(t.parent) && t.status !== "done").map((t) => t.id)
);
const todayTasks = sortByPriority(
  filtered.filter((t) => todayChildIds.has(t.id) || t.due_date === today)
);
```

一方、「明日」セクション（line 137, 修正前）は：
```javascript
const tomorrows  = sortByPriority(filtered.filter((t) => t.due_date === day1));
```

つまり、「明日」セクションでは親タスク期限が「明日」の場合、その配下の子タスク（Due: 他の日付）が含まれていなかった。

### 修正内容

`App.jsx` の `buildSections()` 関数内で、「明日」セクション処理に親子関係ロジックを追加した。

**修正前：**
```javascript
if (view === "次の7日間") {
    const today7 = addDays(today, 7);
    const day1   = addDays(today, 1);
    const day2   = addDays(today, 2);
    const filtered = allTasks.filter(
      (t) => t.due_date && t.due_date <= today7 && t.status !== "done"
    );
    const sections  = [];
    const overdue    = sortByPriority(filtered.filter((t) => t.due_date <  today));
    const todayTasks = sortByPriority(filtered.filter((t) => t.due_date === today));
    const tomorrows  = sortByPriority(filtered.filter((t) => t.due_date === day1));  // ← 親子関係考慮なし
```

**修正後：**
```javascript
if (view === "次の7日間") {
    const today7 = addDays(today, 7);
    const day1   = addDays(today, 1);
    const day2   = addDays(today, 2);
    const filtered = allTasks.filter(
      (t) => t.due_date && t.due_date <= today7 && t.status !== "done"
    );
    
    // 明日期限の親タスクとその配下の子タスクを含める
    const day1ParentIds = new Set(
      allTasks.filter((t) => t.due_date === day1 && t.status !== "done").map((t) => t.id)
    );
    const day1ChildIds = new Set(
      allTasks.filter((t) => t.parent && day1ParentIds.has(t.parent) && t.status !== "done").map((t) => t.id)
    );
    
    const sections  = [];
    const overdue    = sortByPriority(filtered.filter((t) => t.due_date <  today));
    const todayTasks = sortByPriority(filtered.filter((t) => t.due_date === today));
    const tomorrows  = sortByPriority(filtered.filter((t) => day1ChildIds.has(t.id) || t.due_date === day1))); // ← 親子関係考慮
```

### 修正効果

- 「明日」ビューで、「明日」期限の親タスク配下の子タスク（status !== "done"）が正しく表示されるようになった
- 「明日」期限でない子タスクが誤って表示されることはなくなった

## 備考

CHG-018 の「考慮不足」は、「次の7日間」ビューにおいて「今日」セクションと「明日」セクションの親子関係処理に不均衡があったことを指していた可能性がある。修正後は両セクションが同じロジックに統一された。
