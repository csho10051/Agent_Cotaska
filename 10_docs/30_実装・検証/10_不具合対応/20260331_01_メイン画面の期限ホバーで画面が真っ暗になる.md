| 状態 | 調査中 |

## 現象

メイン画面のタスク行に表示される期限へマウスカーソルを当てると、画面が真っ暗になる。

## 再現手順

1. タスク正本の Cotaska を起動する
2. メイン画面で期限が表示されているタスク行を表示する
3. 期限表示の上にマウスカーソルを移動する
4. 画面が真っ暗になることを確認する

## 期待動作

期限表示へマウスカーソルを当てても画面は正常表示のまま維持され、少なくとも renderer crash や黒画面化は発生しない。

## 調査メモ

- 配布版ログ [Cotaska/00_mgmt/Cotaska_タスク管理ツール/logs/app-2026-03-31.log](Cotaska/00_mgmt/Cotaska_タスク管理ツール/logs/app-2026-03-31.log) に `Renderer process gone` が記録されている
- 2026-03-31 16:10 頃の起動では `exitCode=-1073741819`、同日 16:07 頃の起動では `exitCode=-36861` が記録されており、単なる React 描画崩れではなく renderer プロセス自体のクラッシュと判断できる
- [Cotaska/20_app/src/main/main.js](Cotaska/20_app/src/main/main.js#L32) で `app.disableHardwareAcceleration()` は既に有効で、一般的な GPU 無効化は実施済み
- タスク正本の `due_date` 実値を確認したところ、存在した値は `2026-03-26`、`2026-03-30`、`2026-03-31`、`2026-03-31T10:30`、`null` のみで、異常形式の混入は確認できなかった
- [Cotaska/20_app/src/renderer/components/MainPane.jsx](Cotaska/20_app/src/renderer/components/MainPane.jsx#L220) の期限表示自体には `onMouseEnter` や `title` はなく、hover だけで React state を更新する実装は見当たらない
- 一方で CHG-039 で [Cotaska/20_app/src/renderer/components/DueDatePopover.jsx](Cotaska/20_app/src/renderer/components/DueDatePopover.jsx) と期限表示 UI が導入されており、期限セル周辺の描画変更が回帰起点である可能性がある
- 開発モード (`http://localhost:5173/`) では、正本データ相当 62 件を dev 側へ複製しても再現しなかった
- launcher を介さない production 起動 (`20_app/dist/renderer/index.html`) でも現時点では再現しておらず、問題は `dist/renderer` 単体よりも配布版固有の条件に寄っている
- `release/win-unpacked/CotaskaCore.exe` を正本相当データで直接起動しても再現しなかった
- 一方で task-master 正本と最新 release 出力のハッシュ比較では、`Cotaska.exe` は一致するが、`_app/resources/app.asar` と `_app/CotaskaCore.exe` は不一致だった
- したがって不具合の主因は task-master 正本の `_app` が最新 release と未同期である可能性が高い
- `20_app/sync-task-master-release.ps1` による同期を試行したが、task-master 側 `app.asar` が別プロセスによりロックされており置換未完了
- 現時点では「hover そのものが直接 JavaScript 例外を起こしている」根拠は取れておらず、ネイティブクラッシュまたは Electron/Chromium 側を含む描画起因の可能性を残す

## 修正方針

- まず開発環境で再現させ、期限表示周辺を段階的に無効化してクラッシュ発生条件を切り分ける
- 次の対応は task-master 正本の `_app` を最新 release で同期し、再現有無を再確認する
- 優先的な切り分け対象は以下とする
  - task-master 側 `app.asar` ロック解除
  - 最新 release の `_app` 反映
  - 同期後の再現確認
- 切り分け結果に応じて、回帰箇所の縮退修正または Electron 側回避策を検討する
