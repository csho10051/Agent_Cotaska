## 現象

リリース版 Cotaska を起動した際、Windows のタスクマネージャ上の表示名が `Cotaska` ではなく `Electron` と表示される。

T-0075 では「タスクマネージャ上の表示名が Electron になっている。Cotaska にしてほしい」と報告されている。

## 再現手順

1. `Cotaska\00_mgmt\Cotaska_タスク管理ツール\Cotaska.exe` を起動する。
2. Windows タスクマネージャを開く。
3. 実行中の Cotaska 本体プロセスの表示名を確認する。
4. 表示名が `Cotaska` ではなく `Electron` になっていることを確認する。

## 期待動作

Windows タスクマネージャ上のアプリケーションまたはプロセス表示名が `Cotaska` として表示される。

## 調査メモ

2026-05-05 初回調査:

- 現在の実行プロセス名は `CotaskaCore` だが、`CotaskaCore.exe` のバージョン情報が Electron 由来のまま残っている。
- `Cotaska\00_mgmt\Cotaska_タスク管理ツール\_app\CotaskaCore.exe` の `VersionInfo` は以下の状態。
  - `FileDescription: Electron`
  - `ProductName: Electron`
  - `OriginalFilename: electron.exe`
  - `InternalName: electron.exe`
  - `CompanyName: GitHub, Inc.`
  - `ProductVersion: 41.0.2`
  - `FileVersion: 41.0.2`
- `Cotaska.exe` / `_app\CotaskaCore.exe` の関連アイコンは `setup\launcher\icon.ico` と一致しており、今回の主因はアイコン埋め込みではなく EXE のバージョンリソース表示名と推定する。
- `20_app\release-all.ps1` は `CotaskaCore.exe` に対して `Set-ExeIcon.ps1` を実行しているが、現状の `Set-ExeIcon.ps1` は `icon` のみを rcedit で更新しており、`FileDescription` / `ProductName` 等のバージョン文字列は更新していない。
- `20_app\package.json` の `build.productName` は `CotaskaCore` だが、出力済み `CotaskaCore.exe` のバージョン情報には反映されていない。
- `win-unpacked\CotaskaCore.exe` の時点でも `FileDescription` / `ProductName` は `Electron` のため、`organize-release.ps1` や task 正本配布同期での混入ではない。
- rcedit の型定義では `version-string`、`file-version`、`product-version` を更新できる。現在の一時スクリプト `run-rcedit.mjs` は `icon` のみ指定している。
- Go ランチャー `Cotaska.exe` の `versioninfo.json` は `ProductName: Cotaska` / `FileDescription: Cotaska Launcher` を持つが、現象対象はランチャーではなく実際にウィンドウを持つ `_app\CotaskaCore.exe` 側と見られる。

2026-05-06 追加調査・対応:

- rcedit に `version-string` / `file-version` / `product-version` を渡すことで、`CotaskaCore.exe` の表示名メタデータを後書きできることを一時コピーで確認した。
- `InternalFilename` だけでは PowerShell の `VersionInfo.InternalName` が `electron.exe` のまま残ったため、`InternalName` も明示的に `version-string` へ渡すようにした。
- 一時コピー `20_app\temp\BUG-20260505-03-CotaskaCore-test.exe` では以下を確認済み。
  - `FileDescription: Cotaska`
  - `ProductName: Cotaska`
  - `OriginalFilename: CotaskaCore.exe`
  - `InternalName: CotaskaCore`
  - `CompanyName: EbiSenbei`
  - `ProductVersion: 0.1.0`
  - `FileVersion: 0.1.0`
- `20_app\release\win-unpacked\CotaskaCore.exe` と `20_app\release\Cotaska-0.1.0-dist\_app\CotaskaCore.exe` には後処理を適用済み。
- `00_mgmt\Cotaska_タスク管理ツール\_app\CotaskaCore.exe` は実行中プロセスがロックしており、rcedit が `Fatal error: Unable to commit changes` で失敗したため未反映。アプリ終了後に `sync-task-master-release.ps1` で最新リリース成果物を反映する必要がある。

2026-05-06 リリース作業:

- 初回ビルドでは electron-builder が `winCodeSign` を展開する際、Windows のシンボリックリンク作成権限不足で失敗した。
- `20_app\package.json` の `build.win.signAndEditExecutable` に `false` を追加し、Electron Builder による署名・EXE編集を回避する構成へ戻した。
- `release-all.ps1` を再実行し、リリース生成と出荷前検証が完了した。
- Go が未インストールのため Go ランチャー再ビルドは警告でスキップされたが、既存の `setup\launcher\Cotaska.exe` を配布ルートへコピーし、後段の検証は通過した。
- `sync-task-master-release.ps1` により task 正本側 `00_mgmt\Cotaska_タスク管理ツール` へ最新リリース成果物を同期した。バックアップは `00_mgmt\10_task\backup\Cotaska-0.1.0-dist_20260506_001556`。

2026-05-06 再発調査:

- ユーザー確認で、同期済み `00_mgmt\Cotaska_タスク管理ツール` から起動してもタスクマネージャ上の名前が `Electron` のままであると判明した。
- EXE の `VersionInfo` は `Cotaska` 系に更新済みのため、Windows の表示名が EXE メタデータではなく AppUserModelID 単位のキャッシュ、または BrowserWindow / Electron app name 由来で決まっている可能性が高い。
- 既存の AppUserModelID は `com.cotaska.app.release.<instanceHash>` のままで、過去に `Electron` 表示としてキャッシュされた ID を再利用していた。

## 修正方針

対応方針:

- `Set-ExeIcon.ps1` を `Set-ExeMetadata.ps1` 相当に拡張し、`CotaskaCore.exe` へアイコンだけでなくバージョン情報も後書きする。
- `release-all.ps1` の出荷前検証に、`CotaskaCore.exe` の `FileDescription` / `ProductName` が `Cotaska` 系の値であることを確認するチェックを追加する。
- Electron Builder 側でバージョン情報を正しく埋め込める設定に寄せられるかを確認する。

暫定的には、既存の後処理に以下の値を追加する方針が最小変更と考えられる。

- `FileDescription: Cotaska`
- `ProductName: Cotaska`
- `OriginalFilename: CotaskaCore.exe`
- `InternalFilename: CotaskaCore`
- `CompanyName: EbiSenbei`
- `file-version: 0.1.0`
- `product-version: 0.1.0`

## 対応内容

2026-05-06:

- `20_app\setup\launcher\Set-ExeIcon.ps1` を更新し、アイコンに加えて以下の EXE メタデータを rcedit で後書きするようにした。

- `BrowserWindow` の `title` と作成直後の `win.setTitle("Cotaska")` を明示した。

## 2026-05-06 最終対応

状態: 解決済み

- 方針を「Task Manager 上で `Electron` と表示されなければよい。内側 EXE 名は `CotaskaCore.exe` を優先」に変更。
- `20_app/package.json` の `build.productName` は `CotaskaCore` に戻した。
- `_app\CotaskaCore.exe` の `ProductName` / `FileDescription` / `OriginalFilename` / `InternalName` を `CotaskaCore` 系へ統一するよう `Set-ExeIcon.ps1` と `release-all.ps1` を更新。
- AppUserModelID は Windows 側の旧表示キャッシュを避けるため `com.cotaska.app.release.v3.<instanceHash>` へ更新し、本番ログにも設定値を残すようにした。
- Codex 検証環境に `ELECTRON_RUN_AS_NODE=1` が残ると Electron が Node として即終了することを確認したため、Go / C# ランチャー双方で子プロセス起動時に同環境変数を除去するようにした。
- Go 未導入環境でもランチャーを再ビルドできるよう、`LauncherFallback.cs` と `build.ps1` の C# フォールバック経路を追加。
- `release-all.ps1` でビルド後、`sync-task-master-release.ps1` で `00_mgmt\Cotaska_タスク管理ツール` へ同期済み。バックアップは `00_mgmt\10_task\backup\Cotaska-0.1.0-dist_20260506_010032`。
