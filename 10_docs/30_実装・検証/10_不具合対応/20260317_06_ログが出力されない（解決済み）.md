## 現象

アプリを起動して操作しても、デバッグログファイルに何も出力されていない。

ログの出力先を `C:\WorkDevelop\Agent_TaskHub\workspace\logs\` に変更したが、ディレクトリは作成されるもの、ログが記録されていない状態。

---

## 原因分析

### WriteStream のバッファリング問題

DebugLogger と AppLogger が `fs.createWriteStream()` でファイルをオープンし、`write()` メソッドでログを書き込んでいた。しかし、WriteStream はバッファリング機構を持つため：

- `write()` だけではデータがディスクにフラッシュされない
- アプリケーション終了時に `end()` が呼ばれるとフラッシュされるはずだが、タイミングによってはデータが失われることがある
- アプリが強制的にクローズされた場合、バッファ内のデータは失われる

### package.json パスの誤り

main.js で `require("../package.json")` と記述されていたが、main.js は `src/main/main.js` にあるため、正しいパスは `../../package.json` である必要があった。

---

## 修正内容

### 1. logger.js の修正

**変更：WriteStream → fs.appendFileSync**

```javascript
// 変更前（バッファリング問題）：
this.logFile = fs.createWriteStream(filepath, { flags: 'a' });
...
this.logFile.write(msg + '\n');

// 変更後（確実に書き込まれる）：
this.logFilePath = path.join(this.logDir, filename);
...
fs.appendFileSync(this.logFilePath, msg + '\n');
```

**その他の変更：**
- 各ログメソッド（debug, info, warn, error）でファイルに同期的に書き込むように修正
- `destroy()` メソッドを簡略化（同期操作なので特別なクローズ処理不要）
- コンストラクタにデバッグ出力を追加し、logger の初期化状態を確認可能に

### 2. appLogger.js の修正

**logger.js と同じ変更を適用：**
- `fs.createWriteStream()` → `fs.appendFileSync()`
- `_write()` / `logError()` / `logWarning()` メソッドを同期的に動作するように修正
- `destroy()` メソッドを簡略化
- コンストラクタにデバッグ出力を追加

### 3. main.js の修正

**package.json のパス参照を修正：**
```javascript
// 変更前：
const pkg = require("../package.json");

// 変更後：
const pkg = require("../../package.json");
```
