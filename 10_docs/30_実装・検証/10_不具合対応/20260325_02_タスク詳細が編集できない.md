## 現象

右詳細ペイン（画面右の真ん中側）のタスク詳細項目（タスク名、メモなど）が編集できなくなっている。
テキスト入力欄が反応しない、または入力しても保存されない。

## 再現手順

1. アプリ起動後、左ペインからタスクを1つ選択する
2. 右ペインの詳細パネルが表示される
3. タスク名またはメモ欄をクリックして編集を試みる
4. テキストを入力する

## 期待動作

テキスト入力欄がフォーカス可能であり、入力内容がリアルタイムで表示される。
別のタスク選択時または手動保存時に、入力内容が保存される。

## 実際の動作

- テキスト入力欄が応答しない（入力不可）
- または入力してもUIに反映されない

## 根本原因と修正内容

### 根本原因

**タスク詳細の入力欄（textarea）の縦幅が 120px に固定されており、ペインの真ん中より下の領域がクリーム出していなかった。**

`App.css` の `.detail-content` スタイル：
```css
.detail-content {
  min-height: 120px;  /* ← 固定高さ */
  /* ...その他... */
}
```

この結果、タスク詳細の真ん中あたりをクリックしても入力欄の範囲外となり、内容は入力済みでもUIに反応が返されていなかった。見かけ上「編集できない」状態に見えていた。

### 修正内容

`App.css` の `detail-body` を flex コンテナに変更し、`detail-content` を拡大できるようにした。

**修正前：**
```css
.detail-body {
  flex: 1;
  overflow-y: auto;
  padding: 16px 20px;
}

.detail-content {
  font-size: 14px;
  min-height: 120px;  /* ← 最小高さ 120px で固定 */
  /* ...その他... */
}
```

**修正後：**
```css
.detail-body {
  flex: 1;
  display: flex;           /* ← flex コンテナに */
  flex-direction: column;  /* ← 縦方向レイアウト */
  overflow-y: auto;
  padding: 16px 20px;
}

.detail-title {
  /* ...その他... */
  flex-shrink: 0;          /* ← 固定高さで縮まない */
}

.detail-content {
  font-size: 14px;
  flex: 1;                 /* ← 残りスペースを埋める */
  min-height: 0;           /* ← flex 計算時の下地 */
  /* ...その他... */
}
```

### 修正効果

- タスク詳細の textarea が、「タグを追加してください」ラベルの上部分まで拡大される
- ペインの大部分がタスク詳細の入力欄となり、どこをクリックしても input/textarea にフォーカスできるようになった
- プレビューモード時も同様に `detail-preview` が拡大される

---

## 備考

入力自体は機能していたが、UIのレイアウト不具合により、ユーザーからは「編集できない」と見えていた。BUG-20260325-01 の修正とは無関係の UI レイアウト不具合。
