## 現象

右詳細ペインの「進捗ステータス」プルダウン（未着 / 仕掛 / 完了）を変更しても、変更内容が保存されない。
アプリを再起動すると元のステータスに戻っている。

## 再現手順

1. アプリを起動する
2. 任意のタスクをクリックして詳細ペインを開く
3. 「進捗ステータス」プルダウンを変更する（例: 未着 → 仕掛）
4. 他のタスクをクリックして詳細ペインを切り替える、またはアプリを再起動する
5. 元のタスクを再度クリックする

## 期待動作

手順3で変更した進捗ステータスが保持され、再選択時に変更後の値が表示される。

## 実際の動作

手順5で確認すると、手順3で変更する前のステータスに戻っている。

## 調査メモ

### 根本原因

**`DetailPane.jsx` の `persist` 関数内で `window.CotaskaAPI`（大文字 C）を参照していた。**

- `preload.js` では `contextBridge.exposeInMainWorld("cotaskaAPI", ...)` (小文字 c) で expose している
- `App.jsx` は正しく `window.cotaskaAPI` を使用している
- `DetailPane.jsx` の `persist` 関数のみ `window.CotaskaAPI`（大文字 C）を参照していた
- `undefined?.tasks?.update()` はオプショナルチェーンによりエラーなしにスキップされるため、保存処理全体が実行されていなかった

この問題は 進捗ステータス変更のみならず、詳細ペインでのタイトル・メモ・優先度変更など **全ての保存操作が機能していなかった**ことを意味する。

**`DetailPane.jsx` の `handleProgressStatusChange`（行141付近）:**

```javascript
const handleProgressStatusChange = async (e) => {
  const next = e.target.value;
  setProgressStatus(next);
  const nextStatus = next === "完了" ? "done" : "todo";
  const nextCompletedAt = nextStatus === "done" ? (completedAt || new Date().toISOString()) : null;
  setCompleted(nextStatus === "done");
  setCompletedAt(nextCompletedAt);
  setStatus(nextStatus);
  await persist({ progress_status: next, status: nextStatus });
};
```

**`persist` 関数（行95付近）:**

```javascript
const persist = async (patch) => {
  await window.CotaskaAPI?.tasks?.update({
    id: task.id,
    ...（省略）
    progress_status: progressStatus,  // ← クロージャで捕捉した旧値
    ...patch,                          // ← patch で上書きされるはずだが要確認
  });
  onSaved?.();
};
```

### 疑わしい原因（調査優先順）

1. **`onSaved()` 後の親コンポーネント再レンダリングによるリセット**
   - `persist` → `onSaved?.()` → `App.jsx` がタスク再読み込み → DetailPane に新しい `task` prop が渡される
   - `useState` の初期値はマウント時のみ有効なため、`task.progressStatus` が変わっていない場合に古い値が表示される可能性がある

2. **`taskService.js` への `progress_status` の書き込み漏れ**
   - `tasks?.update()` のバックエンド実装（`taskService.js` または `ipc-handlers`）で `progress_status` フィールドが Markdown ファイルに書き込まれていない可能性

3. **`_index.yaml` と実ファイルの不整合**
   - `_index.yaml` は更新されているが、タスク Markdown ファイル（`30_data/tasks/*.md`）の frontmatter が更新されていない可能性

## 修正方針

`DetailPane.jsx` の `persist` 関数内の `window.CotaskaAPI` を `window.cotaskaAPI` （小文字 c）に修正する。

```diff
- await window.CotaskaAPI?.tasks?.update({  // 大文字 C（誤記）
+ await window.cotaskaAPI?.tasks?.update({  // 小文字 c（正）
```

**一行修正のみで解決。**読み込み時に出るエラーなし（optional chaining のためサイレントに失敗）の奇妙なバグであったため、小文字タイポに警右する必要がある。
