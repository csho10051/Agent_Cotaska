# システム構成ルール

## 目的

CoTasker のシステム構成に関するルールと設計方針を明文化し、実装時の判断基準を統一する。

---

## 採用技術

| 項目 | 技術 | 備考 |
|------|------|------|
| アプリ基盤 | Electron | デスクトップアプリ |
| フロントエンド | HTML / CSS / JavaScript | フレームワークは詳細設計時に決定 |
| バックエンド | Node.js | Electron Main プロセス |
| データベース | SQLite | better-sqlite3 で接続 |

## Electron のプロセス構成

Electron は 2 つのプロセスで構成される。

```text
┌─────────────────────────────────────────────────┐
│  Main プロセス（Node.js）                         │
│    ├ ウィンドウ管理（BrowserWindow）               │
│    ├ DB 接続（better-sqlite3 → SQLite）           │
│    ├ ファイル操作                                  │
│    └ IPC ハンドラ（ipcMain）                      │
└────────────────┬────────────────────────────────┘
                 │ IPC（contextBridge / ipcRenderer）
┌────────────────┴────────────────────────────────┐
│  Renderer プロセス（Chromium）                     │
│    ├ 4 カラム UI                                  │
│    │   ├ サイドバー                                │
│    │   ├ ナビパネル                                │
│    │   ├ メインペイン                              │
│    │   └ 詳細ペイン                                │
│    ├ Markdown エディタ                             │
│    └ ドラッグ & ドロップ                            │
└─────────────────────────────────────────────────┘
```

### ルール

- **Main プロセス**は DB 操作・ファイル操作など Node.js API を必要とする処理を担当する。
- **Renderer プロセス**は UI の表示・操作のみを担当する。
- Renderer から直接 Node.js API や DB にアクセスしない。必ず **IPC 経由**で Main に依頼する。
- `contextBridge` を使い、Renderer に公開する API を最小限に制限する（セキュリティ確保）。

## レイヤー構成

```text
Renderer（UI 層）
  │  preload.js（contextBridge）
  │
Main（アプリケーション層）
  ├ サービス層（ビジネスロジック）
  │   ├ TaskService
  │   ├ ListService
  │   ├ TagService
  │   ├ CommentService
  │   └ LogService
  │
  ├ リポジトリ層（データアクセス）
  │   ├ TaskRepository
  │   ├ ListRepository
  │   ├ TagRepository
  │   ├ CommentRepository
  │   └ LogRepository
  │
  └ DB 層
      └ better-sqlite3 → SQLite ファイル
```

### ルール

- **UI 層**はサービス層を直接呼ばない。IPC チャンネル経由で呼び出す。
- **サービス層**はビジネスロジックを担当する。DB を直接操作せず、リポジトリ層を呼ぶ。
- **リポジトリ層**は SQL の発行とデータ変換のみを担当する。ビジネスロジックを含めない。
- **DB 層**は better-sqlite3 の接続管理のみを担当する。

---

## データの正本ルール

- タスクの正本は **SQLite データベース**とする。
- UI はデータベースの内容を表示・更新するビューである。
- Markdown ファイルは AI 連携や設計書用途で出力する副次データとする。
- データの作成・更新・削除は必ず DB を経由する。UI 側でローカルキャッシュを正本としない。

---

## 通信・セキュリティルール

- 外部ネットワーク通信は行わない（ローカル完結）。
- `nodeIntegration` は `false` に設定する。
- `contextIsolation` は `true` に設定する。
- Renderer に公開する API は `preload.js` の `contextBridge.exposeInMainWorld` で明示的に定義する。

## ファイル配置の方針

```text
CoTasker/
  20_app/
    main/           ← Main プロセス（エントリポイント、IPC ハンドラ）
      services/     ← サービス層
      repositories/ ← リポジトリ層
      database/     ← DB 接続管理
    renderer/       ← Renderer プロセス（HTML / CSS / JS）
    preload/        ← preload.js（contextBridge）
  30_data/
    init_schema.sql ← DDL
    cotasker.db     ← SQLite データベースファイル（実行時生成）
```

※ 詳細なディレクトリ構成は詳細設計時に確定する。
