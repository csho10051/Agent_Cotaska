## Electron のプロセス構成

Electron は 2 つのプロセスで構成される。

\\\	ext
┌─────────────────────────────────────────────────┐
│ Main プロセス（Node.js）                          │
│   ├─ ウィンドウ管理（BrowserWindow）               │
│   ├─ DB 接続（better-sqlite3 ↔ SQLite）          │
│   ├─ ファイル操作（fs / fs.promises）             │
│   └─ IPC ハンドラ（ipcMain）                      │
└────────────────┬────────────────────────────────┘
                 │ IPC（contextBridge / ipcRenderer）
┌────────────────┴────────────────────────────────┐
│ Renderer プロセス（Chromium）                     │
│   ├─ 4 カラム UI                                │
│   │   ├─ サイドバー                              │
│   │   ├─ ナビパネル                              │
│   │   ├─ メインペイン                            │
│   │   └─ 詳細ペイン                              │
│   ├─ Markdown エディタ                           │
│   └─ ドラッグ & ドロップ                          │
└──────────────────────────────────────────────────┘
\\\

### ルール

**Main プロセス**
- DB 操作、ファイル操作など Node.js API を必要とする処理を担当
- IPC を通じて Renderer プロセスと通信

**Renderer プロセス**
- UI レンダリングとユーザー操作を担当
- Main プロセスへの要求は IPC 経由のみ

## データ保存構成

### ディレクトリ構造

\\\	ext
data/
  ├─ tasks/
  │   ├─ T-0001.md
  │   ├─ T-0002.md
  │   ├─ ...
  │   └─ _index.yaml（自動生成）
  ├─ lists.yaml
  └─ archive/（アーカイブ済みタスク）
\\\

### ファイルフォーマット

**タスクファイル（T-XXX.md）**
- YAML frontmatter + Markdown 本文
- 正本データ
- エンコーディング：UTF-8（BOM なし）

**サマリーインデックス（_index.yaml）**
- 自動生成（手編集禁止）
- タスク一覧のメタデータ
- React UI の初期化に使用

**リストマスタ（lists.yaml）**
- リスト定義のマスタデータ
- 色、デフォルト順序など

## Core Services

### taskService

- タスク CRUD 操作
- ファイル読み書き
- frontmatter パース

### indexService

- _index.yaml の再生成
- タスク一覧の集約ビュー

### listService

- リスト定義管理
- lists.yaml の読み書き

### watcher

- ファイルシステム監視（chokidar）
- タスク更新の検知
- UI へのリアルタイム反映

## 開発環境セットアップ

### 前提条件

- Windows 10 以上（または macOS、Linux）
- Node.js v22 以上
- npm v10 以上

### インストール手順

1. リポジトリをクローン
\\\ash
git clone <repository>
cd Cotaska
\\\

2. 依存関係をインストール
\\\ash
cd 20_app
npm install
\\\

3. 開発サーバーを起動
\\\ash
npm run dev
\\\
