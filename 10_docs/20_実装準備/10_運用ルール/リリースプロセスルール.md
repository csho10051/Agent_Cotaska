# Cotaska リリースプロセスルール

## 1. 目的

本ルールは、Cotaska のリリース作業を再現可能にし、誰が実施しても同じ成果物を作れるようにすることを目的とする。

---

## 2. 配布構成（正本）

配布フォルダは以下を正本とする。

- `20_app/release/Cotaska-Portable/`

### 2.1 ディレクトリ構成

配布フォルダの構成は、ユーザーデータを`_app/`外に分離した新構成：

```
Cotaska-Portable/
├── _app/              (実行バイナリのみ)
│   ├── app.asar       (アプリケーションバンドル)
│   ├── locales/       (言語リソース)
│   └── [DLL群, pak ファイル群]
├── data/              ★ ユーザーデータ領域（直接編集可能）
│   ├── tasks/         (個別タスクファイル: *.md)
│   ├── archive/       (完了済みタスク)
│   └── lists.yaml     (リスト・タグ定義)
├── logs/              ★ アプリログ出力先
│   └── (app-YYYY-MM-DD.log など)
├── Cotaska.exe        (ランチャー実行ファイル)
├── launcher.log       (ランチャーログ)
├── Cotaska_AIエージェント運用ルール.md
└── README.md          (ユーザー向けガイド)
```

### 2.2 各フォルダの役割

| フォルダ | 内容 | ユーザー編集 |
|---------|------|-----------|
| `_app/` | Electron バイナリ・リソース | 不可（黒魔法） |
| `data/` | タスク・リスト定義 | 可能（YAML/Markdown 編集） |
| `logs/` | アプリ・ランチャーログ | 読み取り専用 |

### 2.3 旧構成からの変更点

従来は`_app/resources/30_data/`と`_app/workspace/logs/`にデータが埋め込まれていた。  
新構成では、ルート直下に`data/`と`logs/`を分離し、アップグレード時のデータ保全を簡潔化。

---

## 3. 前提条件

### 3.1 ツール

- Windows 10/11
- Node.js v22 系
- npm
- Go 1.21 以上（ランチャービルド用）

### 3.2 事前確認

- `00_mgmt/Cotaska_タスク管理ツール/data/tasks/*.md`（タスク正本）の状態が最新になっており、未完了・ブロック中のタスクがないことを確認する
- `DECISIONS.md` に必要な設計判断が反映されている
- 既知不具合の状態を確認している

## 4. リリース手順

### Step 1. Renderer ビルド

```powershell
cd "C:\WorkDevelop\Agent_Cotaska\Cotaska\20_app"
npm run build
```

確認:

- `dist/renderer/` が生成される
- ビルドエラーがない

### Step 2. Electron 配布物生成（dir）

```powershell
cd "C:\WorkDevelop\Agent_Cotaska\Cotaska\20_app"
npm run dist:dir
```

`dist:dir` は次のコマンドを実行すること（固定）:

```powershell
electron-builder --dir --win --x64 --publish never
```

確認:

- `release/win-unpacked/` が生成される
- `release/win-unpacked/Cotaska.exe` が存在する

### Step 3. Go ランチャーをビルド

```powershell
cd "C:\WorkDevelop\Agent_Cotaska\Cotaska\20_app\setup\launcher"
PowerShell -ExecutionPolicy Bypass -File build.ps1
```

確認:

- `20_app/setup/launcher/Cotaska.exe` が生成される

### Step 4. 配布フォルダを新構成に再構成

`npm run dist:dir` の出力には旧構成（`_app/resources/`, `_app/workspace/`）が含まれているため、  
以下のスクリプトで自動的に新構成に整理する。

```powershell
cd "C:\WorkDevelop\Agent_Cotaska\Cotaska\20_app"
.\organize-release.ps1 -BuildDir "release" -Version "0.1.0"
```

確認:

- `release/Cotaska-Portable/` 直下に `data/`, `logs/`, `_app/` が存在する
- `00_mgmt/Cotaska_タスク管理ツール/data/tasks/` に `_index.yaml` が存在する
- `data/lists.yaml` が存在する
- `_app/resources/` と `_app/workspace/` が削除されている

**スクリプト内容** (自動実行):

1. `data/`, `logs/` ディレクトリを作成
2. `_app/resources/30_data/` の contents を `data/` へ移動
3. `_app/workspace/logs/` の contents を `logs/` へ移動
4. 旧ディレクトリを削除

### Step 5. 起動確認

```powershell
& "C:\WorkDevelop\Agent_Cotaska\Cotaska\20_app\release\Cotaska-Portable\Cotaska.exe"
```

確認:

- Cotaska プロセスが起動する
- ウィンドウが表示される
- `data/` および `logs/` ディレクトリが自動作成される（初回起動時）
- アプリが正常に動作する（主要導線確認）

## GitHub Releases アップロード方針

GitHub Releases へアップロードする配布物は、次の2系統を正本とする。

### Portable版・自動更新 / 手動配布用

Portable版の自動更新および手動配布用として、以下をアップロードする。

- `20_app/release/Cotaska-Portable.zip`
- `20_app/release/Cotaska-Portable.zip.sha256`（生成できる場合は必須）

用途:

- Cotaska-Portable 版のアプリ内確認付き自動更新
- zipを展開して利用する Portable版
- 自動更新が利用できない環境での手動更新
- 利用者が明示的に差し替える配布物

運用ルール:

- `Cotaska-Portable.zip` は Portable版更新の主対象とする。
- zip のルートフォルダは `Cotaska-Portable/` とする。
- zip 内には `Cotaska.exe`, `_app/`, `tools/`, `README.md`, `Cotaska_AIエージェント運用ルール.md` を含める。
- Portable版 updater は利用者データを保持するため、配布 zip に含まれる `data/`, `logs/`, `backup/` は更新時に既存環境へ上書きしない。
- `.sha256` をアップロードする場合、内容は `Cotaska-Portable.zip` の SHA-256 値を検証できる形式とする。

### NSISインストーラ用・自動更新用

インストーラ配布および `electron-updater` による確認付き自動更新用として、以下を同一リリースへ必ずセットでアップロードする。

- `20_app/release/CotaskaCore-<version>-win-x64.exe`
- `20_app/release/CotaskaCore-<version>-win-x64.exe.blockmap`
- `20_app/release/latest.yml`

運用ルール:

- `.exe`, `.exe.blockmap`, `latest.yml` は同じビルド結果から生成された組み合わせだけを使用する。
- `.exe` だけ、または `latest.yml` だけを差し替えない。
- `latest.yml` は自動更新が最新版と取得対象を判断するための必須ファイルとする。
- `.blockmap` は差分ダウンロード用メタデータとして扱い、NSISインストーラ本体と同時にアップロードする。
- NSISインストーラ系は当面の互換配布物として維持するが、Portable版利用を原則とする運用では `Cotaska-Portable.zip` を自動更新の主対象とする。
