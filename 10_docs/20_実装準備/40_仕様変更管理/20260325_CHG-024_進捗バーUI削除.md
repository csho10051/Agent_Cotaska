# 仕様変更ファイル

## 変更ID

CHG-024

## 変更内容

右詳細ペインの「進捗バー」UIとその関連ロジックを削除する。

- 削除対象: バー本体（`progress-bar-track` / `progress-bar-fill`）、%ラベル行（`progress-labels`）、クリック操作ロジック（`handleProgressClick`）、進捗 state（`progress` / `setProgress`）
- 削除しないもの: 進捗ステータス（未着/仕掛/完了）は別機能のため残存。タスクデータの `progress` フィールド値もデータ互換性のため保持（表示しないだけ）

## 変更理由

- 3px の細いバーをクリックして %を設定する操作が直感的でなく、使いにくい
- 進捗ステータス（未着/仕掛/完了）で運用上の進捗管理は十分であるため、数値バーの追加価値が低い
- UIのシンプル化を図る

  - `20_app/src/renderer/App.css`
