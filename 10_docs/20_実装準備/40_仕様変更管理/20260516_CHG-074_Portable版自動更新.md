# CHG-074 Portable版自動更新

## 概要

Cotaska の主利用形態を Portable版とし、`Cotaska-Portable.zip` を使ってアプリ内から更新確認、ダウンロード、終了後の差し替え、再起動まで行える仕組みを追加する。

NSIS インストーラ前提の自動更新はいったん対象外とし、Portable版に最適化した更新方式として再設計する。

## 背景

現在の自動更新は NSIS インストール版を主対象としており、Portable版では `electron-updater` が `app-update.yml` を参照して失敗する、または自動更新対象外として案内する挙動になっている。

運用上は `Cotaska-Portable.zip` を配布正本として使う想定のため、Portable版でも利用者が手作業で zip を展開し直さずに更新できる導線が必要である。

## 基本方針

- 更新元は GitHub Releases の latest release とする。
- 更新対象アセットは `Cotaska-Portable.zip` とする。
- アプリ本体実行中に `_app` や `Cotaska.exe` を上書きせず、外部 updater スクリプトを生成してアプリ終了後に差し替える。
- ユーザーデータ領域は更新対象外とし、既存の `data/`, `logs/`, `backup/`, `settings.yaml` 相当の利用者データを保持する。
- 更新前に現行実行フォルダのバックアップを作成し、失敗時に復元できるようにする。
- 更新完了後は `Cotaska.exe` を再起動する。

## 更新フロー

1. 設定 > アプリ情報で「更新を確認」を押す。
2. GitHub Releases latest API を取得する。
3. 現在バージョンと latest release の tag/version を比較する。
4. 更新ありの場合、release assets から `Cotaska-Portable.zip` を選択する。
5. 利用者が「更新をダウンロード」を押す。
6. アプリが zip を一時フォルダへダウンロードする。
7. 利用者が「再起動して更新」を押す。
8. アプリが更新用 PowerShell スクリプトを一時フォルダへ生成する。
9. 更新用スクリプトを detached 起動し、アプリ本体は終了する。
10. 更新用スクリプトが Cotaska 関連プロセス終了を待つ。
11. 現行フォルダをバックアップする。
12. zip を展開し、`Cotaska-Portable/` 内の更新対象を現行フォルダへ反映する。
13. `data/`, `logs/`, `backup/` などの利用者データは上書きしない。
14. `Cotaska.exe` を再起動する。

## 更新対象

原則として、次を差し替える。

- `Cotaska.exe`
- `_app/`
- `tools/`
- `README.md`
- `Cotaska_AIエージェント運用ルール.md`

原則として、次は保持する。

- `data/`
- `logs/`
- `backup/`
- 利用者が配置した任意ファイル

## GitHub Releases 側の配布物

必須:

- `Cotaska-Portable.zip`

推奨:

- `Cotaska-Portable.zip.sha256`

`Cotaska-Portable.zip.sha256` が存在する場合は、ダウンロード後に SHA-256 を検証する。存在しない場合は、GitHub API の asset size と実ファイルサイズが一致することを最低限確認する。

## UI 方針

設定 > アプリ情報に Portable版向けの状態表示を追加する。

- 最新です
- 新しいバージョンがあります
- ダウンロード中
- ダウンロード完了
- 再起動して更新できます
- 更新に失敗しました
- 手動更新が必要です

操作ボタン:

- 更新を確認
- 更新をダウンロード
- 再起動して更新
- ダウンロードページを開く

## 安全設計

- 更新適用前に必ずバックアップを作成する。
- zip のルートが `Cotaska-Portable/` であることを検証する。
- 展開先が現在の runtime root の外へ逸脱しないように検証する。
- 更新中は `data/` を削除・上書きしない。
- 更新失敗時はエラー内容をログに残し、可能ならバックアップから復元する。
- クラウド同期フォルダ上での更新は警告を表示し、利用者確認を必須にする。

## 対象外

- NSIS インストーラ版の自動更新
- electron-updater による NSIS 更新
- アプリ起動中の直接上書き
- ユーザーデータの移行仕様変更

## 完了条件

- Portable版で GitHub Releases の `Cotaska-Portable.zip` を検出できる。
- Portable版で更新zipをダウンロードできる。
- アプリ終了後に外部 updater が現行フォルダを差し替えられる。
- `data/` が保持されることを確認できる。
- 更新後に Cotaska が再起動する。
- 失敗時のログとバックアップが残る。
