# CHG-076 Portable版のみのリリース運用

## 背景

Cotaska の配布・更新導線は Portable版を主対象として再設計済みであり、NSISインストーラ、`.blockmap`、`latest.yml`、`electron-updater` 経路を維持すると、リリース作業と利用者導線が二重化する。

## 変更内容

- GitHub Releases にアップロードする配布物を `Cotaska-Portable.zip` と `Cotaska-Portable.zip.sha256` のみにする。
- `release-all.ps1` は Portable フォルダ、Portable zip、SHA-256 ファイルの生成までを完了条件とする。
- `20_app/package.json` から NSIS ターゲットと NSIS 用スクリプトを削除する。
- main process の更新処理から `electron-updater` 経路を削除し、Portable 更新経路のみにする。
- リリースプロセス文書とリリースノートから NSIS 配布物を削除する。

## 対象外

- 既存の過去リリースノートの書き換え。
- 既存 GitHub Releases 上の過去アセット削除。

## 完了条件

- `npm run build` が成功する。
- `release-all.ps1 -Version "0.2.2"` が Portable zip と `.sha256` を生成する。
- v0.2.2 の GitHub Release に Portable zip と `.sha256` のみをアップロードする。
