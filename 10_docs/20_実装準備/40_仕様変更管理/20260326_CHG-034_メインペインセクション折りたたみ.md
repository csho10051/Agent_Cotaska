# 仕様変更ファイル

## 変更ID

CHG-034

## 変更内容

MainPane（メインペイン）のタスクリストにおけるセクション見出し（「未着・仕掛」「⚠️ 遅延」「☀️ 今日」「📅 明日」「📆 以降」等）にクリックで折りたたみ／展開するトグル機能を追加する。

### 現状の問題点

- 進捗ステータスビュー（「未着・仕掛」「完了」）や日付セクション（「⚠️ 遅延」「☀️ 今日」「📅 明日」「📆 以降」）にセクション折りたたみ機能がない
- タスク数が多い場合に目的のセクションまでスクロールが必要で、一覧性が悪い
- 完了セクションのみ折りたたみ機能があるが、他のセクションには未実装

### 変更前の動作

```
┌──────────────────────────┐
│ 未着・仕掛                │  ← クリックしても何も起きない
│  □ タスクA                │
│  □ タスクB                │
│  □ タスクC                │
│ ▾ ✅ 完了 (2)            │  ← この行だけ折りたたみ可能
│  ☑ タスクD                │
│  ☑ タスクE                │
└──────────────────────────┘
```

### 変更後の動作

```
┌──────────────────────────┐
│ ▾ 未着・仕掛 (3)         │  ← クリックで折りたたみ可能
│  □ タスクA                │
│  □ タスクB                │
│  □ タスクC                │
│ ▾ ✅ 完了 (2)            │  ← 既存動作維持
│  ☑ タスクD                │
│  ☑ タスクE                │
└──────────────────────────┘
```

---

## 変更理由

1. タスク数が増えると全セクションが展開されたままでスクロール量が増え、目的のタスクに到達しにくい
2. 完了セクションのみ折りたたみ可能な現状は一貫性に欠ける
3. セクション単位で表示を制御できることで、ユーザが作業中の領域に集中しやすくなる

---

## 変更詳細

### 1. 折りたたみ対象セクション

| ビュー種別 | セクション例 | 対象 |
|-----------|-------------|------|
| 進捗ステータスビュー | 未着・仕掛（1セクション） | ✅ 新規追加 |
| 日付セクションビュー | ⚠️ 遅延、☀️ 今日、📅 明日、📆 以降 | ✅ 新規追加 |
| 完了セクション | ✅ 完了 (N) | 既存動作を維持 |
| ゴミ箱ビュー | 🗑️ ゴミ箱 | 対象外（単一セクションのため不要） |
| フラットリスト | （セクションなし） | 対象外 |

### 2. UI 仕様

| 項目 | 仕様 |
|------|------|
| トグルアイコン | 展開時: `▾`、折りたたみ時: `▸` |
| 件数表示 | セクションラベルの後に ` (N)` 形式で件数を表示 |
| クリック領域 | セクション見出し行全体 |
| カーソル | `pointer` |
| アニメーション | なし（即時表示切替） |

### 3. 状態管理

| 項目 | 仕様 |
|------|------|
| 管理方法 | MainPane 内の `useState` でオブジェクト管理 |
| キー | セクション種別＋ラベル（例: `progress:未着・仕掛`、`date:☀️ 今日`） |
| 初期状態 | すべて **展開** |
| 永続化 | **不要**（画面遷移やビュー切替でリセット） |

### 4. ドラッグ＆ドロップとの併存

- 折りたたみ中のセクションへのドロップ: **許可する**（セクション見出し行へのドロップで対象セクションにタスクを移動）
- 折りたたみ中のセクションからのドラッグ: タスクが非表示のため **操作不可**（自然な動作）

---

## 影響範囲

- UI:
  - `MainPane.jsx`: セクション折りたたみ状態の useState 追加、`renderProgressSections` / `renderSections` の見出し行にトグルクリックハンドラと条件付き描画を追加
  - `App.css`: `.section-header` に `cursor: pointer` と `user-select: none` を追加
- DB / IPC / ファイル形式: **変更なし**
- 既存機能影響:
  - 完了セクションの折りたたみは既存の `completedSectionExpanded` で動作しており、**変更なし**
  - ドラッグ＆ドロップの `onDragOver` / `onDrop` はセクション見出し行のまま維持

---
