# 仕様変更ファイル

## 変更ID

CHG-022

## 変更内容

画面左側（サイドバー、左ナビ、メインペイン左領域）に表示される英語ラベルを日本語へ統一する。

- 対象はユーザー向け表示文字列（UIラベル、プレースホルダ、補助文言）
- 変数名・内部ID・保存データキー（例: `status`, `progress_status`）は変更しない

## 変更理由

現状の左側UIには日本語と英語が混在しており、操作時の理解コストが高い。
Cotaska の運用前提（日本語UI）に合わせ、表示文言を日本語へ統一して可読性と学習コストを改善する。

## 対象範囲（本変更）

- サイドバー（50px）
- 左ナビパネル（固定ビュー、リスト、タグ、完了、ゴミ箱）
- メインペイン左領域（ヘッダー、クイック追加、セクション見出し、空表示メッセージ）

## 対象外（本変更では実施しない）

- 画面右の詳細ペイン文言
- Markdown本文、ユーザー入力値
- データ構造・API・保存形式のキー名変更

## 文言方針

- 既存の日本語語彙に合わせる（例: 未着 / 仕掛 / 完了、今日 / 明日 / 次の7日間）
- 新規訳語は短く統一する（全角記号の揺れを避ける）
- 単純置換で意味が崩れる文言は、文脈に合わせた自然な日本語へ変更する

## 想定翻訳例（左側領域）

- `Saving...` -> `保存中...`
- `Saved` -> `保存済み`
- `No tasks` -> `タスクがありません`
- `No results` -> `一致するタスクがありません`
- `Select a task` -> `タスクを選択してください`

※最終訳語は実装確認時に画面文脈で確定する。
