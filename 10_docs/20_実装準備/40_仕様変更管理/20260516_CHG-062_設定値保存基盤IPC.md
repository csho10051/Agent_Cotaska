# CHG-062 設定値保存基盤 IPC

## 変更ID

CHG-062

## 変更内容

設定画面の各入力値を保存・取得するための共通基盤を追加する。main process 側に設定サービスを置き、preload 経由で `window.cotaskaAPI.settings.get()` / `update()` 相当の API を提供する。

設定保存ファイルは `settings.yaml` とし、日本語コメントを記載できる人間可読な設定ファイルとして扱う。

## 変更理由

設定画面は表示枠まで実装済みだが、保存・取得の共通基盤がないため、表示名、外部エディタ、通知時間、文字サイズなどの設定値を永続化できない。タスク正本や `_index.yaml` を設定保存先にしない方針を明確にし、設定値はタスクデータから分離して扱う必要がある。

標準 JSON はコメントを書けないため、利用者や保守者が「どの設定か」を日本語コメントで確認できる `settings.yaml` を採用する。

## 影響範囲

- UI: 設定画面の初期表示、保存操作、保存失敗時のエラー表示
- DB: 既存DB変更なし
- API/IPC: settings get/update 系 IPC と preload bridge の追加
- 設定ファイル: `settings.yaml` を追加し、YAMLコメントで設定項目の説明を保持する
- 既存機能影響: タスク `.md`、`_index.yaml`、既存タスクIPCには設定保存処理を混在させない

## 設定ファイル方針

- ファイル名は `settings.yaml` とする。
- 保存先はタスク正本とは分離する。実装時にアプリ設定用ディレクトリを確定する。
- 文字コードは UTF-8 とし、日本語コメントを許可する。
- renderer は設定ファイルを直接読み書きせず、必ず preload / IPC 経由で取得・保存する。
- 初回起動時にファイルが存在しない場合は既定値で動作し、保存時に `settings.yaml` を作成する。

例:

```yaml
# 表示名: アプリ画面やタイトルに表示する名前
displayName: Cotaska

# 外部エディタ: タスクファイルを開くときに使うエディタ
externalEditorPath: ""

notification:
  # 通知を有効にするか
  enabled: true

  # 予定時刻の何分前に通知するか
  minutesBefore: 10

# タスク詳細本文の文字サイズ
detailTextSize: 16
```
