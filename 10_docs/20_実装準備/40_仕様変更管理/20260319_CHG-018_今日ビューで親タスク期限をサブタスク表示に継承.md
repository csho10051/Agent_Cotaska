# 仕様変更ファイル

## 変更ID

CHG-018

## 変更内容

左ナビの「今日」ビューで、親タスクの期限日が今日の場合は、配下のサブタスクを期限日に関係なく「今日」ビューに表示する。

## 変更理由

現行仕様では、サブタスクの `due_date` が今日でない場合に「今日」ビューへ表示されない。
親タスク起点で今日対応するべき作業全体を把握しにくいため、親タスクが今日なら配下サブタスクも表示対象に含める。

## 影響範囲

- **UI**:
  - 「今日」ビューの表示対象判定ロジック
- **表示フィルタ**:
  - 親子関係を考慮したフィルタ条件の追加
- **既存機能影響**:
  - 「次の7日間」「リスト」「タグ」等の他ビュー条件には影響させない

## 設計反映先

- `10_docs/10_設計/20_基本設計/04_表示フィルタ.md`
- `10_docs/10_設計/20_基本設計/02_タスク管理.md`
- `00_mgmt/CURRENT_SPRINT.md`

## 実装タスク分解

1. 設計書修正（今日ビューの親子表示ルール追記）
2. `App.jsx` の今日ビュー判定ロジック更新
3. 必要に応じて `MainPane.jsx` の表示順・重複表示防止を調整
4. 動作確認（親=今日 / 子!=今日 のケース、既存ケース回帰）

## 完了条件

- [ ] 親タスクが今日なら、配下サブタスクが今日ビューに表示される
- [ ] サブタスクが二重表示されない
- [ ] 他ビューの表示条件に回帰がない
- [ ] `npx vite build` が成功する

## テスト観点

- [ ] 親 `due_date!=今日` の場合は従来どおり
- [ ] 子のみ今日の場合も従来どおり表示される
