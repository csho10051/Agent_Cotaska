# 仕様変更

## 変更ID

CHG-009

## 変更内容

親タスクをチェックボックスで完了にした場合、配下のサブタスクも自動的に完了状態にする（カスケード完了）。

完了取消（done → todo）の場合はサブタスクへの波及は行わない。

## 変更理由

現状の設計では「最悪値ストラテジ（ボトムアップ）」のみ実装されており、親→子への完了波及（トップダウン）が未設計だった。
ユーザーが親タスクを完了にした際、子タスクが残留しかつ親子関係が視覚的に消えてしまうUX上の問題が発覚したため追加する。

## 影響範囲

- UI: `App.jsx` の `handleToggleComplete` に子タスク更新ループを追加
- DB/ファイル: 各サブタスクの `status: done`、`progress_status: 完了` をファイルに書き込む
- API/IPC: 既存の `tasks:update` を繰り返し呼ぶのみ（新規チャンネル不要）
- 既存機能影響: 完了取消（done → todo）は対象外。子タスクは変更しない

## 設計反映先

- `10_docs/10_設計/20_基本設計/02_タスク管理.md` — § 4.3 タスク完了 にカスケード仕様を追記

## 実装タスク分解

1. `App.jsx` の `handleToggleComplete` を修正
   - `newStatus === "done"` かつ `task.parent === null`（親タスク）のとき
   - `tasks` state から `parent === task.id` の子タスクを取得
   - 未完了の子タスクを `tasks:update` で `status: done` / `progress_status: 完了` に更新
2. 基本設計書 § 4.3 にカスケード完了の仕様を追記

## テスト観点

- [ ] サブタスクが1件ある親を完了にする → 子も完了

- [ ] サブタスクなし親を完了にする → 正常完了（エラーなし）
- [ ] 既に完了済みのサブタスクが含まれる → 重複更新なし
- [ ] 親の完了取消 → サブタスクのステータスは変わらない
