# 仕様変更ファイル

## 変更ID

CHG-031

## 変更内容

タスクID（`T-0001` 形式）を UI に表示する。

- 変更前: タスクIDは frontmatter に保持されているが、UI のどこにも表示されない
- 変更後: 詳細ペイン（右ペイン）フッターにタスクIDを常時表示する

### 表示仕様

| 項目 | 内容 |
|------|------|
| 表示場所 | 詳細ペイン（`DetailPane`）フッター最下部 |
| 表示形式 | `ID: T-0028` のようなラベル＋ID テキスト |
| 操作 | 表示のみ（コピー機能なし） |
| 表示タイミング | 常時表示 |
| スタイル | 小さめ・グレー系・控えめな表示（主要操作の邪魔をしない） |

---

## 変更理由

- AI エージェントが作業ログや会話中でタスクを参照する際、ID が UI に見えないため確認コストが高い
- タスクファイル（`T-0028.md` 等）との対応をユーザーが目視確認できない
- 「T-0028 を直して」のような ID 指示型コミュニケーションが自然にできるよう環境を整備する

---

## 影響範囲

- UI:
  - `DetailPane.jsx`: フッター（`.detail-footer`）にタスクID表示ブロックを追加
   - `App.css` の詳細ペイン用スタイルに `.d-task-id` 関連セレクタを追加
  - `layout_mock.html`: 詳細ペインフッターにID表示を追記
- DB / IPC / ファイル形式: **変更なし**（`task.id` は既に渡されている）
- 既存機能影響: **なし**（表示のみ追加）

---

## 設計反映先

- `10_docs/10_設計/20_基本設計/02_タスク管理.md`（詳細ペイン表示仕様に追記）

## 進め方

1. 画面モック（`layout_mock.html`）で表示位置と見え方を先に確定する
2. 基本設計へ表示仕様を反映する
3. 実装（`DetailPane.jsx` / `App.css`）に着手する
4. 動作確認を行う

---

## 実装タスク分解

1. `layout_mock.html` の詳細ペインフッターにID表示を追加し、配置を確定する
   - `ID: T-0028` 形式の静的表示を追加
2. `02_タスク管理.md` に詳細ペイン下部のID表示仕様を追記する
3. `DetailPane.jsx` のフッター（`.detail-footer`）にIDブロックを追加する
4. `App.css` に `.d-task-id` スタイル追加
   - フォントサイズ小・グレー系カラー・他操作より弱い視覚強度に調整
5. 動作確認（表示位置、完了/未完了での見え方、詳細ペイン空状態の安全性）
