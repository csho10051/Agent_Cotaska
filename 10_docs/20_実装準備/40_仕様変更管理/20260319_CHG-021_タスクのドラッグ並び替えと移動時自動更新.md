# 仕様変更ファイル

## 変更ID

CHG-021

## 変更内容

メインパネルでタスクをドラッグして並び順を変更できるようにする。
あわせて、セクション間へドラッグ移動した際に以下の業務ルールを自動適用する。

- 今日セクションから明日セクションへ移動した場合: `due_date` を明日へ更新する

※ CHG-028 により「未着」と「仕掛」は「未着・仕掛」セクションに統合済み。同一セクション内の移動となるため `progress_status` の自動変更は発生しない。

## 変更理由

現状は優先度ソート中心の固定表示であり、運用上の優先順位を手で並び替えて保持できない。
また、ビュー/進捗セクションを跨ぐ移動時に、ユーザーの意図（日付変更・着手状態変更）を手作業で二重入力する必要があり、操作負荷と更新漏れのリスクがある。

ドラッグ操作で「表示順の調整」と「移動意図に応じた属性更新」を一体化し、日次運用を高速化する。

## 影響範囲

- UI:
  - `MainPane.jsx` のタスク行ドラッグ開始/移動/ドロップ
  - ドロップ可能位置の可視化（挿入位置インジケータ）
  - 誤操作防止（ドラッグ無効ビュー、無効ドロップ先の明示）
- 表示ロジック:
  - 現行の優先度ソート基準に加えて、手動並び順（`sort_order`）を優先するルール
  - セクション内並び替えとセクション間移動のハンドリング
- データ/永続化:
  - `sort_order` の更新と保存
  - ドロップ時に `due_date` / `progress_status` の自動更新
  - `_index.yaml` 再生成時の順序整合
- 既存機能影響:
  - 完了/ゴミ箱/検索ビューでのドラッグ可否を明確化する必要あり
  - 親子タスク表示（展開/折りたたみ）との整合が必要

## 精査ポイント（重たい変更としての論点）

1. 並び順の単位
- 全体で1本の `sort_order` を持つか、ビュー/セクションごとに持つか
- 初期実装では「タスク全体で単一の `sort_order`」を採用し、表示時にセクション分割する方針を第一候補とする

2. 優先度ソートとの競合
- 現行は優先度順ソートが強く効いているため、ドラッグ結果が見えなくなる可能性がある
- ドラッグ有効ビューでは `sort_order` 優先へ切り替える設計が必要

3. セクション移動時の自動更新優先順位
- 本仕様では「移動先セクションの意味に値を合わせる」を基本原則とする
- 片方向だけでなく逆方向遷移も同じ原則で扱う

  a) 日付セクション間移動（今日/明日）
  - 今日→明日: `due_date=明日` に更新
  - 明日→今日: `due_date=今日` に更新
  - 期限未設定タスクを今日/明日へドロップ: ドロップ先に応じて `due_date` を設定
  - 今日/明日から「日付を持たない領域」へ移動する操作は初期スコープ外（自動で `due_date=null` にはしない）

  b) 進捗セクション（CHG-028 で統合済み）
  - 「未着・仕掛」セクションは統合済みのため、セクション間移動による `progress_status` 自動変更は発生しない
  - 完了セクションを将来対象に含める場合のみ、未着・仕掛→完了で `status=done` 連動を評価する

  c) 競合時の優先順位
  - 1位: 完了整合ルール（`status=done` のタスクは未着/仕掛への移動を禁止、または先に完了解除を要求）
  - 2位: ドロップ先セクション意味の反映（`due_date` / `progress_status` 自動更新）
  - 3位: 手動値の保持（上記2つと矛盾しない場合のみ保持）

  d) 失敗時の扱い
  - `sort_order` 更新と属性更新（`due_date` / `progress_status`）は同一操作として保存する
  - どちらかが失敗した場合は操作全体を不成立として再読込し、部分反映を残さない

4. 親子タスク整合
- 親子を跨ぐドラッグで親子関係を変更するか（今回スコープ外にするか）
- 第一段階では「親子関係は維持、親子間移動は不可」を基本方針とする

5. 更新トランザクション
- 並び順更新 + 属性更新（`due_date` / `progress_status`）を同時に保存する必要がある
- 部分失敗時のロールバック方針（最低でも再読込で整合回復）を定義する

## 実装タスク分解

1. 設計書修正（ドラッグ操作ルール、移動時自動更新ルール、適用範囲）
2. `sort_order` 運用ルール確定（再採番戦略、既存データ互換）
3. `MainPane.jsx` にドラッグ&ドロップUI実装（同一セクション内並び替え）
4. セクション間ドロップ処理実装（今日→明日 / 未着→仕掛の自動更新）
5. Main側の保存処理実装（順序と属性更新の整合保存）
6. 回帰確認（完了/ゴミ箱/検索/親子表示）

## 完了条件

- [ ] メインパネルでドラッグによる並び替えができる

- [ ] `npx vite build` が成功する

## テスト観点

- [ ] 同一セクション内で先頭/中間/末尾への並び替えが正しく保存される
- [ ] 今日→明日移動時、画面表示と保存値の両方で日付更新が確認できる
- [ ] 明日→今日移動時、画面表示と保存値の両方で日付更新が確認できる
- [ ] 未着→仕掛移動時、バッジ表示と保存値の両方で進捗更新が確認できる
- [ ] 仕掛→未着移動時、バッジ表示と保存値の両方で進捗更新が確認できる
- [ ] 完了タスクやゴミ箱タスクは誤ってドラッグ変更されない
- [ ] 親子表示中でも不正な親子崩れが発生しない
- [ ] 連続ドラッグ操作でインデックス再生成やwatcher通知が破綻しない
