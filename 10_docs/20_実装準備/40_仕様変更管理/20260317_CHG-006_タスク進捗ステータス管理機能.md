# 仕様変更ファイル

## 変更ID

CHG-006

## 変更内容

タスクごとに「未着 / 仕掛 / 完了」の3段階の進捗ステータスを管理できる機能を追加する。
既存の todo/doing/blocked/done（タスク状態）とは独立した別の管理軸として実装する。

## 変更理由

既存のステータス（todo/doing/blocked/done）は細かい作業状態を表すが、プロジェクト全体における「どこまで進んだか」を直感的に把握しにくい。「未着 / 仕掛 / 完了」という粗い粒度の進捗ステータスを追加することで、タスクの進行状況を一目で確認できるようにする。

## 影響範囲

- **UI**:
  - 詳細ペイン：`progress_status` のドロップダウン（未着 / 仕掛 / 完了）を追加
  - ナビパネル：3セクション（未着 / 仕掛中 / 完了）を ToggleButton 形式で表示、件数バッジ付き
  - メインペイン：ステータス別セクションで区切り表示、各タスク行に `progress_status` 表示カラム追加
- **DB**:
  - `trx_task` テーブルに `progress_status TEXT` カラム追加
  - 値：`'未着'` / `'仕掛'` / `'完了'`（文字列型）
  - デフォルト：`'未着'`（新規作成時）
- **API/IPC**:
  - 既存 `tasks:update` で `progress_status` を更新（新規 IPC 不要）
  - `tasks:getAll` の結果に `progress_status` を含める
- **既存機能影響**:
  - 既存の完了チェック（`status = 'done'`）と `progress_status = '完了'` は連動する
  - チェックボックス ✓ ↔ `progress_status = '完了'` が双方向で同期
  - 監査ログ（task_logs）への記録は不要

## 実装タスク分解

1. **T-015-01**: DB スキーマ拡張（progress_status カラム追加）
   - `db.js` の `ENSURE_DDL` 内 `trx_task` テーブルに `progress_status TEXT NOT NULL DEFAULT '未着'` を追加
   - `addTask()` の INSERT 文と引数に `progress_status` を追加
   - `getAllTasks()` の SELECT に `progress_status` を追加
   - `App.jsx` の `mapDbTask()` に `progressStatus` フィールドを追加
2. **T-015-02**: 完了チェック連動ロジック実装（App.jsx）
   - `handleCompleteTask(task)` — チェックボックス ✓ 時に `progress_status = '完了'` も更新
   - `handleReopenTask(task)` — 完了取消時に `progress_status = '仕掛'` に戻す
   - 詳細ペインで `progress_status = '完了'` に変更した際にチェックボックスも ✓ に更新
3. **T-015-03**: 親タスクの自動推定ロジック実装（App.jsx）
   - 最悪値ストラテジ：サブタスク群の最も低い progress_status を親に適用
     - `'未着'` が1つでもあれば → 親は `'未着'`
     - `'仕掛'` が1つでもあれば（未着なし）→ 親は `'仕掛'`
     - 全て `'完了'` → 親も `'完了'`
   - ユーザーが親を手動設定した場合は手動値を優先（自動推定を上書きしない）
   - サブタスクの追加・削除・変更後に親を自動再計算
4. **T-015-04**: 詳細ペイン UI 実装（DetailPane.jsx）
   - `progress_status` のドロップダウン（未着 / 仕掛 / 完了）を追加
   - 変更時は `tasks:update` で即時保存（`persist()` 経由）
5. **T-015-05**: ナビパネル UI 実装（NavPanel.jsx）
   - 「進捗ステータス」セクション追加（未着 / 仕掛中 / 完了）
   - ToggleButton 形式（任意の組み合わせで表示/非表示を切り替え）
   - 各セクションに件数バッジ（「未着 (12)」「仕掛中 (5)」「完了 (3)」）
6. **T-015-06**: メインペイン UI 更新（MainPane.jsx）
   - ナビパネルで選択されたステータスフィルターに応じてタスクを絞り込む
   - ステータス別セクション（未着 / 仕掛中 / 完了）で区切り表示
   - 各タスク行に `progress_status` のバッジ・アイコンを表示
7. **T-015-07**: スタイル追加（App.css）
   - ToggleButton スタイル（アクティブ/非アクティブ）
   - `progress_status` バッジのカラーコーディング（未着: グレー、仕掛: 青、完了: 緑）
8. **T-015-08**: 動作確認・デバッグ
