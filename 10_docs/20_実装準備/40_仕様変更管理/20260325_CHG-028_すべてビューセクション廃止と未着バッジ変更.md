# 仕様変更ファイル

## 変更ID

CHG-028

## 変更内容

以下の1点を変更する。

### 変更1: 進捗セクション「未着」と「仕掛」を「未着・仕掛」に統合

現行の進捗セクション表示では、未着タスクが「未着」セクション、進行中タスクが「仕掛」セクションに分かれている。
これを 1 つの「未着・仕掛」セクションに統合し、未着と仕掛を同一グループとして表示する。

※ `progress_status` の内部値およびタスク行の進捗ステータスバッジ表示は変更しない。

## 変更理由

### 変更1
- 現場運用上は `未着` と `仕掛` を厳密に分けずに見たい場面がある
- `未着` と `仕掛` を単一セクションに統合することで、一覧の見通しを保ちながら運用認識に合わせる

## 影響範囲

### 変更1: 進捗セクション `未着` / `仕掛` の統合

- **UI**:
  - 進捗セクションのうち、`未着` と `仕掛` が 1 つの `未着・仕掛` ヘッダーにまとまる
- **実装**:
  - `20_app/src/renderer/App.jsx`
    - `progressSections` の生成ロジックを修正し、`progress_status = 未着` と `progress_status = 仕掛` を同一セクションへ集約する
  - `20_app/src/renderer/components/MainPane.jsx`
    - 統合セクション表示とドラッグ移動時の内部値の扱いを整理する
- **変更しない**:
  - `progress_status` フィールドの内部値（`未着` のまま）
  - タスク行の進捗ステータスバッジ（`未着` のまま）

## 現状と変更後

### 変更1: 進捗セクション表示

| | 表示イメージ |
|---|---|
| 変更前 | セクション：**未着**（タスク一覧）/ **仕掛**（タスク一覧）/ **完了**（タスク一覧） |
| 変更後 | セクション：**未着・仕掛**（未着 + 仕掛のタスク一覧）/ **完了**（タスク一覧） |
