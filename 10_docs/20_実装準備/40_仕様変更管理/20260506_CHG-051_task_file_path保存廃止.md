## 変更ID

CHG-051

## 変更内容

タスクファイル正本および `_index.yaml` から `task_file_path` を保存項目として廃止する。
Obsidian のように、ファイルパスは Markdown/frontmatter に永続化せず、アプリが読み込み時に実行時メタデータとして解決する方式へ変更する。

## 変更理由

`task_file_path` はタスクファイル自身の保存先を表す自己参照情報であり、配布先・バックアップ・別PC・フォルダ移動で古くなりやすい。
タスクファイルを正本とする Cotaska では、frontmatter にはタスクの意味情報だけを残し、保存場所はアプリ内部で管理する方がデータの可搬性と可読性が高い。

## 影響範囲

- UI: 詳細ペインの対象ファイル表示、外部アプリ起動導線
- DB: なし（Markdown/frontmatter と `_index.yaml` のスキーマ変更）
- API/IPC: 外部アプリ起動やファイル解決を `task_file_path` ではなく `taskId` 起点に変更する可能性あり
- 既存機能影響: タスク読み込み、保存、削除、復元、インデックス再生成、検証スクリプト、AI運用ルール

## 実装タスク分解

1. 設計書とAI運用ルールに、タスクファイル正本へ保存する項目と実行時メタデータの分離方針を反映する。
2. `taskService` / `indexService` / renderer の `task_file_path` 依存を洗い出し、実行時 `_filePath` または `taskId` 解決へ置き換える。
3. 既存タスクファイルと `_index.yaml` から `task_file_path` を除去する移行を行う。
4. タスク読み込み、更新、削除、復元、外部アプリ起動、インデックス再生成の動作を確認する。
