# 仕様変更ファイル

## 変更ID

CHG-042

## 変更内容

タスクの期限日時に時刻が設定されている場合、期限の5分前に Windows 通知を表示する機能を追加する。

- 通知対象: 未完了タスク（status が done 以外）
- 通知条件: due_date が日時形式で、現在時刻が「期限5分前以上〜期限未満」の範囲に入ったとき
- 通知内容: タスク名と通知時刻（期限時刻）
- 重複抑止: 同一タスク・同一期限に対して複数回通知しない

---

## 変更理由

- 期限直前の見落としを防ぎ、タスク実行漏れを減らすため
- タスク管理ツールとして、期限管理の実効性を高めるため
- 既存の期限入力機能（日時）を運用価値の高い形で活用するため

---

## 影響範囲

- UI: OS 通知（Windows 通知トースト）
- DB: なし（既存 due_date を利用）
- API/IPC: なし（main プロセス内で定期チェック）
- 既存機能影響: 起動中のタスク監視処理、アプリ終了時のサービス停止処理

## 完了条件

- [ ] due_date に時刻がある未完了タスクで、期限5分前に通知される
- [ ] due_date が日付のみのタスクでは通知されない
- [ ] 完了済みタスクでは通知されない
- [ ] 同一タスク・同一期限で重複通知されない
- [ ] アプリ終了時に通知監視が停止する
- [ ] 既存のタスク編集・監視機能に回帰がない

---

## テスト観点

- [ ] 期限時刻を現在+6分で設定したタスクが、約1分後に通知される
- [ ] 期限時刻を現在+3分で設定したタスクが、直後に通知される
- [ ] 通知後に同じタスクで再通知されない
- [ ] status を done にしたタスクでは通知されない
- [ ] due_date が null または日付のみの場合は通知されない
- [ ] アプリ再起動後も対象条件で通知される
