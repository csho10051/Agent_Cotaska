# DECISIONS

## 2026-05-15 Cotaska 起動時の画面表示高速化

- 起動時にタスク/リストサービス初期化、全タスク読み込み、`_index.yaml` 再生成を完了してから `BrowserWindow` を作る構成は、画面表示までの体感時間をI/O量に依存させるため見直した。
- `BrowserWindow` はデータディレクトリ確保と必要な開発サーバ準備後に先行作成し、サービス初期化は `servicesReady` Promise として起動後に進める。
- IPC ハンドラは引き続き `servicesReady` を await するため、画面は早く出しつつ、タスク/リスト操作は初期化完了後に実行する方針とした。
- ファイル監視とリマインダーはタスクキャッシュ準備後に開始し、未初期化状態の検索ルートや空キャッシュを参照しないようにした。
- `_index.yaml` 再生成は既存内容との差分がない場合に書き込みを省略し、クラウド同期フォルダ配下での起動時I/Oを増やさない方針とした。

## 2026-05-15 設定画面基本設計の分離

- 設定画面はタスク管理固有機能ではなくアプリ全体設定を扱うため、基本設計を `10_docs/10_設計/20_基本設計/08_設定画面.md` として独立させる。
- 設定カテゴリパネルは黒基調とし、初期カテゴリは「アプリ情報」とする。
- 「アプリ情報」「表示と編集」は同一設計書内に詳細仕様を持たせ、画面モックはカテゴリ別に分ける。

## 2026-05-15 CHG-060 タスク詳細 Markdown 編集ハイライト

- タスク詳細本文は Markdown テキストのまま保存し、編集モードの表示だけを Markdown 記法に応じて強調する。
- VSCode の Markdown 編集画面に近い体験を目標とし、textarea 疑似ハイライトではなく CodeMirror 系の Markdown 対応エディタ導入を第一候補とする。
- 既存のプレビューモード、リンク起動、自動保存、外部アプリ起動は維持する。
- 2026-05-15 実装時点では Google Drive 配下の `node_modules` 書き込みで npm install が失敗したため、新規依存を追加せず textarea と背面ハイライト表示の組み合わせで初期対応する。

## 2026-05-15 CHG-059 タスク詳細ペイン拡大ボタン

- 詳細ペイン拡大は保存データや IPC を増やさず、renderer の一時 UI 状態として扱う。
- 拡大中はナビパネル、メインペイン、詳細リサイズハンドルを隠し、サイドバー以外の表示領域を詳細ペインに割り当てる。
- 通常幅は既存の `cotaska.detailPaneWidth` 保存値を維持し、拡大解除後に元の幅へ戻す。

- 表示場所は設定画面の「アプリ情報」セクションを第一候補とし、初期対応では自動更新ではなくユーザー確認付きの手動ダウンロード案内を優先する。

- 既存の `progress: 100` は `status: done` / `progress_status: 完了` で意味を維持できるため、データ移行では `progress:` 行を削除する。

## 2026-05-10 CHG-054 完了時日付自動設定

- タスク完了時に `due_date` が空欄の場合のみ、完了日のローカル日付を自動設定する。
- 既存の `due_date` は上書きせず、親タスク完了による子孫カスケード完了にも同じ補正を適用する。

## 2026-05-06 CHG-051 task_file_path保存廃止

- Obsidian を参考に、タスクファイル正本と `_index.yaml` へ `task_file_path` を保存しない方針として仕様変更を起票した。
- ファイルパスはタスクの意味情報ではなく、アプリが読み込み時に解決する実行時メタデータとして扱う。
- 外部アプリ起動などファイルパスが必要な処理は、renderer が保存済みパスを直接使うのではなく、`taskId` 起点で main process が現在の実ファイルを解決する方針とする。

## 2026-05-05 BUG-20260505-03 タスクマネージャ表示名

- タスクマネージャ上の `Electron` 表示はアイコンではなく、`CotaskaCore.exe` の `FileDescription` / `ProductName` などのバージョン情報が Electron 由来のまま残っていることを主因候補とする。
- 既存の rcedit 後処理はアイコンだけを更新しているため、修正時は EXE メタデータ更新と出荷前検証を追加する方針で検討する。
- Electron Builder の `win.signAndEditExecutable` は Windows のシンボリックリンク作成権限に依存するため `false` とし、CotaskaCore.exe のアイコンと表示名メタデータは `Set-ExeIcon.ps1` の rcedit 後処理で一元更新する。

## 2026-05-05 BUG-20260505-02 親子進捗連動と保留の優先

- 親タスクの進捗自動推定は、親タスクが `未着` の場合だけ実行する。
- 親タスクを `保留` にした場合は未完了の `未着` / `仕掛` 子孫タスクを `保留` へ同期し、完了済み子孫タスクは変更しない。
- 親タスクを `保留` から `未着` / `仕掛` に戻す場合は、`保留` の子孫タスクだけ同方向へ戻す。

## 2026-05-05 BUG-20260505-01 親タスク削除時のサブタスク扱い

- サブタスクを持つ親タスクをゴミ箱へ移動する場合は、アプリ内ダイアログで削除前に確認する。
- 確認ダイアログは「全て削除」「親だけ削除」「キャンセル」の3択とし、キャンセル時は親タスク・配下タスクとも変更しない。
- サブタスクも削除する選択では配下タスクをまとめてゴミ箱へ移動し、親だけ削除する選択では直下の子タスクの `parent` を `null` にして通常一覧の親不明警告を避ける。

## 2026-05-04 CHG-050 進捗ステータスに保留を追加

- `progress_status` に「保留」を追加し、`status` は完了以外 `todo` として扱う。
- メインペインでは「未着・仕掛」「保留」「完了」の順に進捗セクションを表示し、保留タスクがない場合はセクションを非表示にする。

- 詳細ペイン、日付ポップオーバー、期限通知タイトルのユーザー表示を同じ方針で統一する。

## 2026-05-04 CHG-048 サブタスクインデント幅調整

- 子タスクの初段インデントが右に寄りすぎる課題を、階層間のインデント差分を同程度に揃えるUI仕様変更として起票した。
- DB/API/IPC は変更せず、既存の `parent` frontmatter と5階層表示仕様を前提に、メインペイン表示と画面モックの調整で追跡する。
- 実装では階層ごとの固定インデントステップを `--task-depth` で算出し、子タスク初段だけ余分に入っていた空スペーサーを表示しない方針とした。

- DB 変更は行わず、5階層上限仕様との整合を設計・実装・ユーザ確認タスクで追跡する。

## 2026-05-10 CHG-053 リスト表示並び替え

- リストモードの並び替えは検索結果並び替えと同じ ID・日付・名前の昇順/降順を採用する。

- 保存済みの `sort_order` は変更せず、ドラッグ並び替えとは独立した一時表示順として扱う。

- 初期表示は ID 昇順とし、日付ソートでは日付未設定タスクを昇順・降順どちらでも末尾に寄せる。

## 2026-05-10 BUG-20260510-01 検索モードの検索対象拡張

- 検索モードの空白区切り入力は AND 条件として扱い、各キーワードがタスクの検索対象情報のいずれかに含まれる場合に一致とする。
- タグは `タグ名` と `#タグ名` の両方を検索対象に含め、日付は `due_date` の保存値・日付部分・画面表示形式を検索対象に含める。
## 2026-05-14 CHG-058 Codex CLIチャット連携
- CotaskaからCodexとやり取りする初期方式は、OpenAI APIキーをCotaskaへ保持する方式ではなく、ローカルのCodex CLIをmain processから呼び出す方式として起票する。
- Codexの応答はチャット表示用メッセージとタスク反映用の構造化アクションに分け、タスク反映はユーザー承認後のみ既存の `taskService` 経由で行う。
- rendererからCLIを直接起動せず、`window.cotaskaAPI` / IPCを通じてmain process側で実行・検証する方針とする。
- 2026-05-14追記: Codex CLI専用ではなく、GitHub Copilot CLI、Claude Code、Gemini CLIも追加可能なローカルAI CLIプロバイダ方式として扱う。初期実装は共通基盤とCodex CLI、他CLIは段階タスクで追加する。

## 2026-05-15 CHG-061 タスク詳細文字サイズ変更

- T-0171 の要望を CHG-061 として起票し、タスク詳細ペインの本文表示・編集領域を対象に文字サイズ変更を扱う。
- 操作は `Ctrl` + `+` / `Ctrl` + `-` と `Ctrl` + マウスホイールを第一候補とし、Electron全体のページズームではなく詳細本文領域の表示調整として設計する。
- CHG-059 の詳細ペイン拡大表示、CHG-060 のMarkdown編集ハイライトと競合しないことを実装・確認タスクの条件に含める。
- 2026-05-15追記: デバッグ環境で詳細ペインのローカルイベントではショートカットが届かないケースがあったため、詳細ペイン内に対象を限定したうえで `window` の capture フェーズでキー/ホイール操作を捕捉する。
- 2026-05-15追記2: renderer のイベントだけでは Electron デバッグ環境でキー入力が届かないケースが残ったため、main process の `before-input-event` から preload 経由で詳細本文サイズ変更イベントを通知する経路を追加する。
- 2026-05-15追記3: 日本語キーボードの `Ctrl` + `+` は `Shift` 併用キーになり得るため `Shift` を除外しない。さらに `Ctrl` + ホイール等が Chromium のページズームに流れる場合に備え、`zoom-changed` を抑止して詳細本文サイズ変更へ変換する。
- 2026-05-15追記4: ボタン操作で表示反映は確認できたため、ショートカット入力は詳細ペイン位置判定に依存させず、renderer の `keydown` / `wheel` capture でもアプリ全体から詳細本文サイズ変更へ割り当てる。
- 2026-05-15追記5: ユーザー確認後、画面上の文字サイズ変更ボタンは削除し、仕様上の操作は `Ctrl` + `+` / `Ctrl` + `-` / `Ctrl` + マウスホイールに限定する。

## 2026-05-16 CHG-062 設定値保存基盤 IPC
