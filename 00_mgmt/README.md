# README.md

## 概要

本プロジェクトは、**AIエージェント（例: Codex）と人間が共同で利用するローカルタスク管理システム**です。

企業ネットワークの制約やセキュリティ要件を考慮し、**クラウドに依存しないローカル完結型アーキテクチャ**を採用しています。

主な特徴：

* ローカルファースト設計
* Markdownタスクファイル（`30_data/tasks/`）
* デスクトップアプリケーション（Electron + React）
* AIエージェントと共有可能なタスク管理
* Markdownベースの運用ドキュメント

---

## システム構成

システムは以下の構成で動作します。

```text
デスクトップアプリ（Electron + React）
      │  IPC
      ▼
アプリケーションロジック（taskService / listService / indexService）
      │  fs read/write
      ▼
30_data/tasks/T-XXX.md  ← タスクファイル（正本）
30_data/tasks/_index.yaml  ← 自動生成サマリー
30_data/lists.yaml      ← リストマスタ
```

補助ファイル：

```text
AGENTS.md          AIエージェントの作業ルール
CURRENT_SPRINT.md  現在のタスク一覧
DECISIONS.md       設計判断の記録
90_作業ログ/       作業メモ・一時的な備忘録
```

## 設計方針

本システムは以下の原則で設計されています。

### 1. ローカルファースト

すべてのデータはローカル環境で管理します。

理由：

* 社内ネットワーク制約
* セキュリティ要件
* シンプルな運用

---

### 2. タスクファイルを正本とする

タスク情報の正本は **`30_data/tasks/` 配下の個別Markdownファイル**です。

各タスクは `T-001.md`, `T-002.md` のような形式で保存され、YAMLフロントマター + Markdown本文で構成されます。

YAMLインデックス（`_index.yaml`）はアプリが自動生成するサマリーです。AI・人間ともに直接読み書き可能ですが、上書きはアプリが行います。

Markdownファイルは以下の用途で使用します。

* タスク情報の永続化（正本）
* AIエージェントによる直接編集
* バージョン管理・差分確認
* 人間が素読みできる形式

---

### 3. AIと人間の協働

AIエージェントは以下のドキュメントを参照して作業します。

* `AGENTS.md`
  AIの作業ルール

* `CURRENT_SPRINT.md`
  現在取り組むタスク

* `DECISIONS.md`
  設計判断の履歴

仕様変更（CHG）を依頼する場合は、以下の運用ルールに従います。

* `10_docs/20_実装準備/10_運用ルール/仕様変更管理ルール.md`

特に `CURRENT_SPRINT.md` には次の3点を必須で分解登録します。

1. 設計書反映の必要可否確認（必要な場合は設計反映タスクを実施）
2. 実装
3. ユーザ確認

## 00_mgmt 備忘録ルール

`00_mgmt/90_作業ログ/` は、設計判断前のメモ、一時的な整理メモ、作業中の論点メモを保存するために使用する。

ファイル名は以下の形式とする。

```text
YYYYMMDD_連番_ファイル概要.md
```

例：

```text
20260318_01_ファイルファースト移行メモ.md
20260318_02_タスクfrontmatter設計案.md
```
