アプリの保存モデルと `_index.yaml` 出力から `progress` を削除する。

## 対象

- `20_app/src/main/taskService.js`
- `20_app/src/main/indexService.js`
- `20_app/src/renderer/App.jsx`

## 対応結果

- `taskService.js` は読み込み・更新入力・frontmatter書き込み時に `progress` を除去する。
- `indexService.js` は `_index.yaml` へ `progress` を出力しない。
- `App.jsx` は renderer payload から `progress` を送信しない。
- `validate-tasks.ps1` は `progress` を既知frontmatterキーとして扱わない。

## 検証

- 同梱Node v22.14.0で `npm.cmd run build` OK。
- `progress` が入力されても `taskService.js` で無視し、既存ファイルに残っている場合は読み込み後の正本書き戻しで除去する。
