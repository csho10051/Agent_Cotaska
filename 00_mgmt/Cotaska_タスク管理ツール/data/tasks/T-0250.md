# CHG-077-01 設計：起動ロード画面の表示仕様整理

## 目的

起動直後の黒い空白ウィンドウをなくすため、ロード画面の表示条件、終了条件、責務分担を整理する。

## 作業内容

- 起動時の BrowserWindow 表示タイミングと renderer 初期化タイミングを確認する。
- ロード画面を main process 側で出すか、renderer 側の初期表示として出すかを比較する。
- 初回 `tasks:getAll` 待機中の表示と、初回描画完了後の遷移条件を整理する。
- 必要な設計ドキュメントへ、起動ロード画面の仕様を反映する。

## 完了条件

- ロード画面の表示開始、表示終了、エラー時の扱いが決まっている。
- 実装対象ファイルと検証観点が明確になっている。
- 必要な設計ドキュメント更新が完了している。

## 実施結果

- `BrowserWindow` の背景色、`index.html` の静的ロード画面、`App.jsx` の初回タスク取得中ロード画面で分担する方針に決定。
- main process が `startup:progress` と `startup:getProgress` で起動進捗を renderer へ渡す方針に見直した。
- 進捗率は起動段階を示す目安とし、厳密なファイル読み込み割合ではなく現在処理中の内容と合わせて表示する。
- 初回タスク取得と初期 view model 構築完了を通常画面への切替条件とした。
- 初回タスク取得失敗時は通常画面へ進まず、起動失敗メッセージを表示する方針とした。
- `10_docs/10_設計/10_システム設計/システム全体設計.md` に CHG-077 の起動時初期表示仕様を追記した。
