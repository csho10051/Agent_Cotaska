## 概要

CHG-044「破損タスクファイルの検知と警告表示」について、実装前に破損タスクの扱いを詳細化する。

## 作業内容

1. 破損タスクとして扱う条件を整理する
2. `is_invalid` / `validation_error` / `task_file_path` などの内部項目を確定する
3. 正常タスクと破損タスクの表示・操作可否を整理する
4. ファイル修正後に通常タスクへ戻る条件を整理する

## 完了条件

- CHG-044 の破損タスク仕様が実装可能な粒度で整理されている
- UI 表示、詳細ペイン表示、ファイル修正後の復旧動作が明確になっている

## 対応内容

CHG-044 仕様変更票へ以下の詳細仕様を追記した。

1. 破損タスクとして扱う条件
2. `is_invalid` / `validation_error` / `validation_error_line` / `validation_error_column` などの内部項目
3. 一覧表示、詳細ペイン表示、通常操作の無効化方針
4. ファイル修正後に通常タスクへ復旧する条件

## 判定

破損タスクは「ユーザが原因ファイルを発見して修正するための診断用タスク」として扱う。
そのため、一覧表示・詳細確認・対象ファイル起動は可能とし、完了・編集・削除・並び替えなどの通常タスク更新操作は無効化する。
