# 目的

`task_file_path` を保存しない方式に変更した後、ユーザー視点で主要操作が問題なく動作することを確認する。

# 確認観点

- タスク一覧が正常に表示される。
- タスク詳細を開いて編集・保存できる。
- 新規タスクを作成しても frontmatter に `task_file_path` が増えない。
- `_index.yaml` に `task_file_path` が出力されない。
- タスクファイルを外部アプリで開ける。
- ゴミ箱移動・復元・完全削除が正常に動作する。
