# 目的

`task_file_path` をタスク `.md` と `_index.yaml` に保存しない実装へ変更する。

# 実装観点

- `taskService` はファイル走査時に実ファイルパスを `_filePath` などの非保存情報として保持する。
- 新規タスク作成・更新・復元・削除時に frontmatter へ `task_file_path` を書き込まない。
- `indexService` は `_index.yaml` に `task_file_path` を出力しない。
- renderer は `task_file_path` を前提にせず、必要な場合は main process へ `taskId` を渡してファイルを開く。
- 既存タスクファイルから `task_file_path` を削除する移行を行う。

# 完了条件

- 新規作成・更新後のタスク `.md` に `task_file_path` が出力されない。
- `_index.yaml` 再生成後に `task_file_path` が出力されない。
- 既存タスクの読み込み・保存・削除・復元が正常に動作する。
- タスクファイルを外部アプリで開く導線が維持される。

# 実施結果

- `taskService` は `_filePath` を内部メタデータとして保持し、frontmatter へ `task_file_path` を書き戻さない形に変更。
- `indexService` は `_index.yaml` のタスク要素へ `task_file_path` を出力しない形に変更。
- 詳細ペインの外部アプリ起動は `taskId` を main process に渡す IPC に変更。
- 既存タスクファイルと `_index.yaml` から `task_file_path` を削除。
