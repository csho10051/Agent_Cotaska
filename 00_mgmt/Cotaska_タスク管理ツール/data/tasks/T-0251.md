# CHG-077-02 実装：起動ロード画面の追加

## 目的

Cotaska.exe 起動後、メイン画面表示までの待機時間にロード画面を表示する。

## 作業内容

- 起動直後に黒い空白ウィンドウが出ないよう、初期背景またはロード UI を追加する。
- renderer の初期化中、初回タスク取得中、初回描画前の状態をロード画面として扱う。
- 初期化完了後に通常のメイン画面へ遷移する。
- 既存の `window.cotaskaAPI` 経由の IPC とタスク表示を壊さない。

## 完了条件

- 起動直後にロード画面が表示される。
- 初回タスク取得後、通常画面へ遷移する。
- `npm run build` が成功する。
- 手元の起動確認で黒い空白画面が改善している。

## 実施結果

- `20_app/src/main/main.js` のウィンドウ背景色をロード画面に近い明色へ変更。
- `20_app/src/main/main.js` に起動進捗状態と `startup:progress` 通知を追加。
- `20_app/src/main/preload.js` に `startup.getProgress()` と `startup.onProgress()` を追加。
- `20_app/src/renderer/index.html` に React 起動前の静的ロード画面を追加。
- `20_app/src/renderer/App.jsx` に初回 `tasks:getAll` 完了まで表示する renderer ロード画面と進捗率表示を追加。
- `20_app/src/renderer/App.css` にロード画面と進捗バーのスタイルを追加。
- `npm run build` は既定 Node 20.18.1 では Vite 要件により失敗したが、Node 24.14.0 を PATH 先頭に置いて成功した。
- `npm start` による Electron 起動スモーク確認で `did-finish-load`、`ready-to-show`、`Startup self-check` まで到達したことをログで確認した。
