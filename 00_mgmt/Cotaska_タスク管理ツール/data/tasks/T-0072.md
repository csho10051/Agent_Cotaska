不具合：タスクバーのロゴがElectionのものになっている。
ウィンドウの左上のアイコンは以下のパスのものになっていない
Cotaska\20_app\setup\launcher\icon.ico

本アプリはCotaska.exeとCotaskaCore.exeの二つがある。
この両方にアイコンファイルが適用されていないのでは？

## 現象

リリース版 Cotaska 起動時、タスクバーのアイコンが Cotaska のロゴではなく Electron の既定アイコンに見える。

T-0072 では「Election」と記載されているが、文脈上は Electron の既定アイコンを指していると判断する。

また、ウィンドウ左上のアイコンも `Cotaska\20_app\setup\launcher\icon.ico` のものになっていない可能性がある。

## 再現手順

1. `Cotaska\00_mgmt\Cotaska_タスク管理ツール\Cotaska.exe` を起動する。
2. Windows タスクバーに表示されるアプリアイコンを確認する。
3. Cotaska ロゴではなく Electron 既定アイコンに見えることを確認する。
4. ウィンドウ左上のアイコンも併せて確認する。

## 期待動作

- タスクバーのアイコンが Cotaska ロゴになること。
- ウィンドウ左上のアイコンが `Cotaska\20_app\setup\launcher\icon.ico` と同じ Cotaska ロゴになること。
- リリース版の `Cotaska.exe` と `_app\CotaskaCore.exe` のどちらを Windows が参照しても、同じ Cotaska ロゴが表示されること。

## 調査メモ

### 1. 関連ファイル

- `Cotaska\20_app\setup\launcher\icon.ico`
  - 正本と考えられる Cotaska アイコン
- `Cotaska\20_app\setup\launcher\Cotaska.exe`
  - Go 製ランチャー
- `Cotaska\20_app\release\Cotaska-0.1.0-dist\Cotaska.exe`
  - リリース配布ルートのランチャー
- `Cotaska\20_app\release\Cotaska-0.1.0-dist\_app\CotaskaCore.exe`
  - Electron 本体
- `Cotaska\00_mgmt\Cotaska_タスク管理ツール\Cotaska.exe`
  - タスク正本側に同期済みのランチャー
- `Cotaska\00_mgmt\Cotaska_タスク管理ツール\_app\CotaskaCore.exe`
  - タスク正本側に同期済みの Electron 本体

### 2. アイコン抽出結果

PowerShell の `System.Drawing.Icon.ExtractAssociatedIcon()` で各 EXE の関連アイコンを抽出し、PNG 化した SHA-256 先頭 16 桁を比較した。

| 対象 | アイコンSHA先頭16桁 | 判定 |
|------|---------------------|------|
| `20_app\setup\launcher\icon.ico` | `64FBCD79C5F0459C` | 正本アイコン |
| `20_app\setup\launcher\Cotaska.exe` | `64FBCD79C5F0459C` | 正本アイコンと一致 |
| `20_app\release\Cotaska-0.1.0-dist\Cotaska.exe` | `64FBCD79C5F0459C` | 正本アイコンと一致 |
| `00_mgmt\Cotaska_タスク管理ツール\Cotaska.exe` | `64FBCD79C5F0459C` | 正本アイコンと一致 |
| `20_app\release\Cotaska-0.1.0-dist\_app\CotaskaCore.exe` | `74D479EEB7E2C326` | 正本アイコンと不一致 |
| `00_mgmt\Cotaska_タスク管理ツール\_app\CotaskaCore.exe` | `74D479EEB7E2C326` | 正本アイコンと不一致 |

この結果から、ランチャー `Cotaska.exe` には Cotaska アイコンが埋め込まれているが、実際に起動後に常駐する `CotaskaCore.exe` の埋め込みアイコンが Cotaska アイコンになっていない可能性が高い。

### 3. ビルド / リリーススクリプト上の懸念

`20_app\package.json` では Electron Builder の `build.win.icon` に `setup/launcher/icon.ico` が指定されている。

```json
"win": {
  "icon": "setup/launcher/icon.ico",
  "signAndEditExecutable": false
}
```

ただし、現在のリリースフローでは以下の懸念がある。

- `signAndEditExecutable: false` により、Electron Builder が EXE アイコンを編集しない可能性がある。
- `release-all.ps1` には `CotaskaCore.exe` にアイコンを後書きする処理があるが、直近のリリース作業では Go 未導入により Step 2 で停止し、この後書き処理まで到達していない。
- `setup\launcher\Set-ExeIcon.ps1` は `C:\WorkDevelop\Agent_Cotaska\v22.14.0\npm.cmd` / `node.exe` を直接参照しており、移行先環境ではパス不一致により失敗する可能性が高い。

## 推定原因

Windows タスクバーは、ランチャー `Cotaska.exe` ではなく、実際にウィンドウを持つ `_app\CotaskaCore.exe` 側のアイコンまたは AppUserModelID に紐づくアイコンを表示している。

現状では `CotaskaCore.exe` の埋め込みアイコンが `setup\launcher\icon.ico` と一致していないため、タスクバー上で Electron 既定アイコンが表示されている可能性が高い。

## 修正方針案

1. `CotaskaCore.exe` にも `setup\launcher\icon.ico` を確実に埋め込む。
2. `20_app\setup\launcher\Set-ExeIcon.ps1` の Node / npm 参照を固定パスではなく、リポジトリ相対パスに修正する。
3. Go ランチャービルドに失敗しても、既存ランチャーを使って `organize-release.ps1` と `CotaskaCore.exe` アイコン後書き処理が継続できるようにする。
4. `release-all.ps1` の出荷前検証に、以下を追加する。
   - `Cotaska.exe` の関連アイコンが `setup\launcher\icon.ico` と一致すること
   - `_app\CotaskaCore.exe` の関連アイコンが `setup\launcher\icon.ico` と一致すること

## 暫定回避策

なし。

タスクバー表示は Windows 側のアイコンキャッシュにも影響されるため、修正後は以下も確認する。

- 新しい配布フォルダ名で起動する
- 必要に応じて Windows のアイコンキャッシュ更新後に確認する
