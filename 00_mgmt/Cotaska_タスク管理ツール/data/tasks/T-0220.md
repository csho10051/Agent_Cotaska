## 目的

`20_app/src` 配下の肥大化したファイルを段階的に整理し、今後の仕様変更で触る範囲を小さくする。

## 方針

- 挙動変更ではなく、責務分離を主目的にする。
- まず renderer 側の低リスクなロジック分離から進める。
- `taskService.js` はタスク正本に触るため、テスト観点を整理してから慎重に分割する。

## 仕様変更票

- `10_docs/20_実装準備/40_仕様変更管理/20260516_CHG-071_src構成整理.md`


## 対応内容

- renderer のタスク表示ロジックを `lib/taskViewModel.js` へ分離した。
- MainPane のツリー構築ロジックを `lib/taskTree.js` へ分離した。
- DetailPane の Markdown 編集補助を `lib/markdownEditor.js` と `hooks/useDebounce.js` へ分離した。
- main process のバックアップ/復元処理を `backupService.js` へ分離した。
- `taskService.js` はデータ破損リスクを避け、今回は検証観点整理に留めた。
- `npm run build` でビルド成功を確認した。
