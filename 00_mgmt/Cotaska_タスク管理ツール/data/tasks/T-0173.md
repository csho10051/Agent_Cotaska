## 目的

タスク詳細ペインの本文表示・編集領域で、ショートカットキーおよび Ctrl + マウスホイールによる文字サイズ変更を実装する。

## 完了条件

- `Ctrl` + `+` で本文文字サイズを拡大できる
- `Ctrl` + `-` で本文文字サイズを縮小できる
- `Ctrl` + マウスホイールで本文文字サイズを拡大・縮小できる
- 最小・最大サイズが設定され、範囲外へ変化しない
- Markdown編集ハイライト、プレビュー、本文保存、詳細ペイン拡大表示が維持される

## 対応結果

- 詳細ペイン本文の文字サイズを 12px から 22px の範囲で変更できるようにした。
- `Ctrl` + `+` / `Ctrl` + `-` と `Ctrl` + マウスホイールに対応した。
- Markdown編集ハイライト、textarea、Markdownプレビューに同じ文字サイズを適用した。
- 設定は `cotaska.detailContentFontSize` として renderer の localStorage に保持する。
- 2026-05-15 追記: デバッグ環境でショートカットが効かないケースがあったため、詳細ペイン内のイベントを `window` capture フェーズで捕捉し、`Ctrl` + マウスホイールは既定ズームを抑止して本文サイズ変更へ割り当てるよう修正した。
- 2026-05-15 追記2: renderer のイベントだけでは Electron デバッグ環境でキー入力が届かないケースが残ったため、main process の before-input-event から preload 経由で詳細本文サイズ変更イベントを通知する経路を追加した。
- 2026-05-15 追記3: 日本語キーボードの `Ctrl` + `+` が `Shift` 併用キーになる点と、Chromium のページズーム経路に流れる点を考慮し、`zoom-changed` イベントからも詳細本文サイズ変更へ変換するよう修正した。
- 2026-05-15 追記4: ボタン操作は正常に動作したため、ショートカット入力の捕捉に絞り、renderer 側でも `keydown` と `wheel` をアプリ全体で capture して詳細本文サイズ変更へ割り当てるよう修正した。
- 2026-05-15 追記5: ユーザー確認により画面上の文字サイズ変更ボタンは不要と判断し、ショートカット操作のみ残す形で削除した。
- 2026-05-16 追記6: 文字サイズ変更時に詳細ペイン上へ現在値を一時表示するインジケータを追加した。
