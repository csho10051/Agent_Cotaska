## 概要

破損タスクを選択したとき、詳細ペインに YAML 検証エラーと対象ファイルパスを表示し、ファイルを開ける導線を追加する。

## 作業内容

1. `validation_error` を詳細ペインに表示する
2. `task_file_path` を確認できるようにする
3. 既存の `shell:openPath` または `shell:openTarget` を使って対象ファイルを開けるようにする
4. 破損タスクでは編集できない項目がある場合、その扱いを明示する

## 完了条件

- 詳細ペインで破損理由を確認できる
- 対象の Markdown ファイルを詳細ペインから開ける

## 対応内容

- `DetailPane.jsx` に破損タスク用の検証エラーパネルを追加した
- エラー種別、エラー位置、エラー内容、対象ファイルパスを表示するようにした
- 既存の外部アプリで開くボタンから対象 Markdown を開けるようにした
- 破損タスクのタイトル、本文、進捗、期限、優先度、リスト、タグの編集操作を読み取り専用にした
