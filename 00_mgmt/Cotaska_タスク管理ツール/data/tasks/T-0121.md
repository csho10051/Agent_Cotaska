# 目的

`task_file_path` を保存データから廃止し、実行時メタデータとして扱う方針を設計書へ反映する。

# 対象候補

- `10_docs/10_設計/20_基本設計/02_タスク管理.md`
- `10_docs/10_設計/10_システム設計/システム全体設計.md`
- `00_mgmt/Cotaska_タスク管理ツール/Cotaska_AIエージェント運用ルール.md`

# 完了条件

- タスクファイル正本に保存する項目と、アプリ内部の実行時メタデータの扱いが分離して記載されている。
- `_index.yaml` に `task_file_path` を保存しない方針が明記されている。
- 外部アプリ起動は `taskId` 起点でファイルを解決する方針が記載されている。

# 実施結果

- `10_docs/10_設計/20_基本設計/02_タスク管理.md` に CHG-051 のパス非保存方針を反映。
- `10_docs/10_設計/10_システム設計/システム全体設計.md` に `_index.yaml` と実ファイルパス管理ルールを反映。
- `Cotaska_AIエージェント運用ルール.md` の `_index.yaml` 同期手順から `task_file_path` を除去。
