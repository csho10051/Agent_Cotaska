どこかキリが良いところでバージョン管理をし始めたい。
バージョン情報を設定画面か、アプリのタイトル部分か、どちらかに持たせて、わかるようにしたい。

## 検討メモ 2026-05-03

- バージョン管理は、まず「アプリ自身が現在バージョンを正しく表示できる状態」から始める。
- 現在バージョンの正本は `20_app/package.json` の `version` とするのが自然。
- 表示場所はアプリタイトル部分よりも、T-0021「設定画面の実装」で作る設定画面内の「アプリ情報」セクションが第一候補。
- そのため、基本的な実施順は T-0021 を先に進め、その後 T-0097 でバージョン表示・更新確認を載せる。
- 例外として、設定画面の実装が大きくなりそうな場合は、先に main/preload 側へ `cotaskaAPI.app.getVersion()` のような現在バージョン取得APIだけ作るのはあり。

想定ステップ:

1. T-0021 で設定画面の入口と基本レイアウトを作る。
2. T-0097 前半で `package.json` の `version` を main process で読み取り、preload 経由で renderer へ渡す。
3. 設定画面に `Cotaska 0.1.0` のような現在バージョンを表示する。
4. T-0097 後半で GitHub Releases または `latest.json` から最新バージョン情報を取得する。
5. 現在バージョンと最新バージョンを比較し、最新版がある場合にダウンロード案内を出す。

初期方針:

- いきなり自動更新までは行わない。
- まずは「最新版があります。ダウンロードページを開きますか？」という手動ダウンロード案内に留める。
- Cotaska はローカルタスクデータが重要なので、勝手な上書き更新よりも、ユーザー確認を挟む安全な流れを優先する。
