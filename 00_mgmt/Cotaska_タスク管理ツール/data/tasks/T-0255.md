# CHG-078-02 実装：今日/明日の保留タスクを保留セッションへ移動

## 目的

今日/明日セクションに残っている保留タスクを、保留セッション側へ分類して表示する。

## 作業内容

- renderer 側のタスク分類ロジックを確認する。
- 今日/明日セクションより保留セッションの分類を優先する。
- 保留ではない今日/明日タスクの表示を維持する。
- 保留解除時に期限日ベースの分類へ戻るようにする。
- 既存の IPC とタスク保存形式を変更せずに実装する。

## 完了条件

- 今日または明日期限の保留タスクが保留セッションに表示される。
- 今日/明日セクションには保留ではないタスクだけが表示される。
- 保留解除後、期限に応じて今日/明日セクションへ戻る。
- `npm run build` が成功する。

## 実施結果

- `20_app/src/renderer/lib/taskViewModel.js` に `isOnHoldTask()` と `isActiveDateTask()` を追加し、日付ビューの共通除外条件を整理した。
- 今日 / 次の7日間のセクション生成では、保留タスクを日付セクションから除外するようにした。
- 明日ビューと、今日 / 明日 / 次の7日間のナビ件数バッジでも、保留タスクを除外するようにした。
- 左ナビの「保留」ビューは `isOnHoldTask()` を使い、今日/明日期限の保留タスクも集約表示できるようにした。
- 既存の IPC とタスク保存形式は変更していない。
- Node 20.18.1 では Vite 要件により `npm run build` が失敗したが、Node 24.14.0 を PATH 先頭に置いて `npm run build` が成功した。
