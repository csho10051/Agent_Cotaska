設定画面の各入力値を保存・取得するための共通基盤を実装する。

## 実装方針

- main process 側に設定サービスを追加する。
- preload に `window.cotaskaAPI.settings.get()` / `update()` 相当の API を追加する。
- renderer は preload 経由で設定値を取得・保存する。
- 設定保存ファイルは `settings.yaml` とし、日本語コメントを記載できる形式にする。
- 保存ファイルはタスク正本とは分離し、タスクファイルや `_index.yaml` を設定保存先にしない。

## 完了条件

- 設定画面を開いたとき保存済み設定が表示される。
- 値を変更して再起動しても設定が保持される。
- 保存失敗時にユーザーへ分かる形でエラー表示される。
- `settings.yaml` に日本語コメントを含めても読み込み・保存できる。
