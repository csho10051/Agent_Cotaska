T-0211およびCHG-069について、
設定ー＞アプリ情報画面で「再起動して更新」をクリックして、アプリは落ちるが再起動はしない
アプリもバージョンアップされていない。

## バグレポート

- `10_docs/30_実装・検証/10_不具合対応/20260516_04_再起動して更新後にバージョンアップされない.md`

## 調査メモ

- GitHub Releases latest は `v0.2.2` で、`Cotaska-Portable.zip` と `Cotaska-Portable.zip.sha256` が存在する。
- `%TEMP%\Cotaska-updates\0.2.2\` に `Cotaska-Portable.zip` と `apply-portable-update.ps1` が残っており、ダウンロードと updater スクリプト生成までは到達している。
- 対象 Portable 環境の `logs/` に `portable-update-*.log` は作成されていない。
- 生成スクリプトは UTF-8 BOMなしで保存され、`powershell.exe` で実行される。Windows PowerShell 5.1 では日本語パスが文字化けすることを最小再現で確認した。
- 推定原因は、Portable updater スクリプトの文字コードと Windows PowerShell 5.1 の解釈不一致により、`Cotaska_タスク管理ツール` を含む更新先パスが壊れること。

## 対応メモ

- `20_app/src/main/main.js` で updater script を UTF-8 BOM付き生成に変更した。
- updater ログを temp 側と Portable 側の両方へ出すようにした。
- updater 起動ログ、起動失敗ログ、早期終了ログをアプリログへ出すようにした。
- 待機対象プロセスを、同じ PortableRoot 配下の `Cotaska.exe` / `CotaskaCore.exe` に限定した。
- `$PortableRoot` / `$ZipPath` の存在確認と、待機中プロセスIDのログ出力を追加した。
- `node --check` と Node.js v22.14.0 での `npm.cmd run build` は成功した。
- 実更新の再試験は未実施。

## 2026-05-19 再調査メモ

- 0.2.3 更新操作後、`%TEMP%\Cotaska-updates\0.2.3\` に `Cotaska-Portable.zip` と `apply-portable-update.ps1` が生成されていた。
- アプリログには `Portable updater process started` の直後に `code:0` で終了した記録がある一方、temp 側にも Portable 側にも `portable-update-*.log` が生成されていなかった。
- 同じ `.ps1` を手動で `powershell.exe -File` 実行すると、更新ログを出し、実行中 Cotaska プロセスの待機に入ることを確認した。
- Node/Electron から `spawn(..., { detached: true, stdio: "ignore" })` で `powershell.exe -File` を直接起動すると、PowerShell が即終了し、スクリプト本体が実行されないことを最小再現した。
- 生成スクリプト内の `$portableRootPrefix.EndsWith("\\")` は JS template のエスケープにより `EndsWith("")` へ崩れていた。主因ではないが、プロセス判定を不安定にするため修正対象とした。

## 2026-05-19 追加対応メモ

- updater 起動方式を、Electron から `detached: true` で `.ps1` を直起動する方式から、短命の PowerShell wrapper が `Start-Process` で実 updater を起動する方式へ変更した。
- wrapper 起動には Windows PowerShell の絶対パスと `-EncodedCommand` を使い、日本語パスや PATH 解決に依存しないようにした。
- updater スクリプト内の PortableRoot prefix 末尾区切りは `[System.IO.Path]::DirectorySeparatorChar` を使うよう変更した。
- 修正後の最小再現で、wrapper 終了後に別 PowerShell の updater が残り、`portable-update-*.log` を出して Cotaska プロセス待機に入ることを確認した。
- `node --check 20_app/src/main/main.js` と `npm run build` は成功した。

## 2026-05-19 Go updater 方針

- 利用者向け仕様は「再起動して更新で Portable 版を差し替える」のまま変えないため、仕様変更票は起票せず BUG-20260516-04 の追加対応として扱う。
- `20_app/setup/updater/` に Go 製 `CotaskaUpdater.exe` のソースとビルドスクリプトを追加した。
- リリース時に Go が利用できる環境では `20_app/scripts/CotaskaUpdater.exe` を生成し、配布物の `tools/` に同梱する。
- アプリ本体は `tools/CotaskaUpdater.exe` が存在する場合はこれを temp 更新作業フォルダへコピーして起動し、存在しない場合のみ PowerShell updater へフォールバックする。
- この環境では Go が未導入のため、Go updater exe の実ビルドは未実施。`node --check` と `npm run build` は成功し、updater build script は Go 未導入時に警告終了することを確認した。
