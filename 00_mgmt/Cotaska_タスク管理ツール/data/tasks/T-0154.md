## 仕様変更概要

CotaskaにローカルAI CLIとやり取りするチャット機能を追加する。

初期対応はCodex CLIを第一候補とするが、設計上はGitHub Copilot CLI、Claude Code、Gemini CLIなどの他CLIエージェントも後から追加できるプロバイダ方式にする。

Cotaskaの画面から相談内容を入力し、Electron main processから選択中のローカルAI CLIを起動して応答を取得する。CLIの標準出力はCotaskaのチャット欄へ表示し、必要に応じてログまたは履歴として保存する。

タスクへの反映は、AI CLIの自然文を直接適用せず、Cotaskaが検証できる構造化アクションとして扱う。ユーザーがプレビューを確認して承認した場合のみ、既存の `taskService` 経由で `data/tasks/*.md` を更新する。

## 対象CLI候補

- Codex CLI
- GitHub Copilot CLI
- Claude Code
- Gemini CLI

## 段階実装方針

1. 共通CLIプロバイダ設計とチャットUIを作る。
2. Codex CLIを初期プロバイダとして実装する。
3. タスク反映用の構造化アクションと承認制を実装する。
4. GitHub Copilot CLI、Claude Code、Gemini CLIは個別タスクで順次追加する。

## 方針

- Cotaska内に各サービスのAPIキーを直接保持しない。各CLI側の既存ログイン/認証状態を利用する。
- rendererはCLIを直接起動せず、`window.cotaskaAPI` からmain processへ依頼する。
- CLIごとの差異はプロバイダ設定で吸収する。
- CLI応答はチャット表示用メッセージとタスク反映用アクションに分ける。
- タスク反映は必ずユーザー承認制にする。
- `_index.yaml` は生成物として扱い、タスク本体は `data/tasks/*.md` を正とする。
