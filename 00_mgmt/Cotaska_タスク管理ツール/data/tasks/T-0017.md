## バグ管理情報
- 番号: BUG-20260326-02
- バグレポート: `10_docs/30_実装・検証/10_不具合対応/20260326_02_子タスク連動で親が完了にならない.md`

以下の構造のタスクがあります。
「T-0005」（仕様変更：期限未設定）
└「T-0014」（CHG-030-04 画面表示のユーザ確認）

この状態で、子タスク「T-0014」を完了にしても、親タスク「T-0005」が完了になりません。ただし、似た確認をした時に、以下のデータでは再現しませんでした。
「T-0015」（タスク（A））
└「T-0016」（サブタスク（A））

この「T-0014」はAIが追加したタスクですが
「T-0016」はアプリで追加したタスクです。
原因調査をお願いします。

## 原因
- 親進捗の自動連動ロジックで、`is_manual_progress = 1` の親タスクを除外していた。
- AI が追加した `T-0005` は `is_manual_progress: 1` だったため、子タスク完了時の親自動更新が動作しなかった。
- 一方、アプリ追加の親タスクは `is_manual_progress: 0` となるため、同事象が再現しなかった。

## 対応
### 短期対応（完了）
- `20_app/src/renderer/App.jsx` の親進捗自動推定処理を修正。
- 子タスクを持つ親については `is_manual_progress` の値に関係なく、子タスク連動を優先するよう変更。
- `npm run build` 成功を確認。

### 中期対応（完了）
この不具合調査を通じて、`is_manual_progress` フィールドが実質的に機能していないことが判明。廃止方針に基づき、以下サブタスクを完了：

- **T-0022**: App.jsx / DetailPane.jsx コード削除
- **T-0023**: DB スキーマ・frontmatter 削除 + マイグレーション
- **T-0024**: 設計ドキュメント更新
- **T-0025**: AIエージェント運用ルール更新
