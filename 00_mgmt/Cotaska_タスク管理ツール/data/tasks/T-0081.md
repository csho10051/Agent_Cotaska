## 概要

`taskService.js` の Markdown 読み込み処理で YAML frontmatter のパースエラーを捕捉し、例外で停止せず破損タスクとして返す。

## 作業内容

1. `loadTaskFromFile()` の `matter(content)` を try/catch で保護する
2. パース失敗時に `is_invalid: true` のタスクオブジェクトを生成する
3. エラー内容、対象ファイルパス、可能なら行番号・列番号を保持する
4. 破損タスクの ID が通常タスク ID と衝突しないようにする

## 完了条件

- YAML が壊れたタスクファイルを読み込んでも main process が例外停止しない
- 破損タスクとして UI に渡せるデータが生成される

## 対応内容

- `taskService.js` に `createInvalidTask()` を追加し、YAML パース失敗時に `is_invalid: true` の診断用タスクを生成するようにした
- `validation_error` / `validation_error_name` / `validation_error_line` / `validation_error_column` / `task_file_path` を保持するようにした
- `id` 欠落や重複 ID も破損タスクとして扱う方針を実装した
- 破損タスク更新を防ぐ `assertMutableTask()` を追加した
