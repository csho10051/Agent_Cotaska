# 現象
現状、子タスクが完了or仕掛となると、
親タスクも仕掛にする仕様がある。
そのため、子タスクが完了or仕掛だと
親タスクを仕掛⇒保留にできない。

# 修正案
子タスクが完了or仕掛となると、
親タスクも仕掛にする仕様は、
親タスクが未着の場合だけ、
未着⇒仕掛とし、仕掛になった後は動作しないようする。
また、親タスクが保留となった場合、
紐づく子タスクが未着or仕掛のものも保留に
子タスクが完了のものはそのままにしてほしい。
また親タスクが‘保留‘から進捗ステータスを更新した場合、
以下の分岐とする。
- 親タスクが保留⇒未着とした場合、子タスクも保留⇒未着（子タスクが完了なら処理なし）
- 親タスクが保留⇒仕掛とした場合、子タスクも保留⇒仕掛（子タスクが完了なら処理なし）
- 親タスクが保留⇒完了とした場合、子タスクも保留⇒完了
