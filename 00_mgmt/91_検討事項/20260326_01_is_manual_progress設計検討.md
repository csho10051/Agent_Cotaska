# is_manual_progress 設計検討

> 作成日: 2026-03-26  
> 関連: BUG-20260326-02（子タスク連動で親が完了にならない）

## 1. 概要

`is_manual_progress` フィールドの現在の仕様と、将来的に手動カスタマイズ機能を入れる場合のシーン・制約を検討する。

---

## 2. 現在の実装

### 2.1 仕様

| 値 | 意味 | 対象 |
|---|------|------|
| **0** | 自動推定 | 親タスクの進捗をサブタスク群から**自動計算**する |
| **1** | 手動設定 | ユーザー（またはAI）が進捗を**手動で設定**した値を保持する |

### 2.2 新規作成時の初期値

```javascript
// App.jsx L294
is_manual_progress: task.parent == null ? 1 : (task.isManualProgress ? 1 : 0)
```

- 親タスク（`parent == null`）→ デフォルト `1`（手動設定）
- サブタスク → 手動設定値を引き継ぎ

### 2.3 BUG-20260326-02 での修正

**問題点**：
- `is_manual_progress = 1` の親タスクは、子タスク完了時に親の進捗が自動更新されなかった

**原因コード（修正前）**：
```javascript
// 旧版：問題あり
if (children.length === 0 || parent.isManualProgress) continue;
// サブタスク有無に関わらず、手動フラグで進捗更新をスキップ
```

**修正内容**：
```javascript
// 新版：サブタスクを持つ親は手動フラグに関わらず自動推定を優用
if (children.length === 0) continue;
// サブタスク有無のみで判定。is_manual_progress は無視
```

## 3. 手動カスタマイズ機能が活躍するシーン

### シーン 1：「段階的な親タスク管理」

**状況**：
- 親タスク「プロジェクトA」（`is_manual_progress = 1`）
- 最初は子なし → 「準備中」と手動マーク
- AI が子タスク 5 件を追加

**現在の挙動**：
- 子追加後、親は子から自動再計算 → 「未着」に変更
- 想定と矛盾なし（問題なし）

---

### シーン 2：「メンテナンスモード親タスク」

**状況**：
- 親タスク「v1.0 サポート終了準備」（`is_manual_progress = 1`）
- サブタスク群：「既存Issues棚卸し」「ドキュメント転記」「ユーザー通知」...
- **すべてのサブタスクが「完了」** → 親も自動で「完了」に更新される

**問題**：
- 管理上の理由で親を「対応中」のまま保持したい
- 例：本当の終了日が別にあり、まだ終了報告していない

**改善ニーズ**：
- 子がすべて完了でも、親を「対応中」に手動設定のまま保持したい

---

### シーン 3：「予定と実績のズレ」

**状況**：
- 親タスク「Q2 機能開発」（`is_manual_progress = 1`）
- 当初計画：サブタスク 10 件（見積：10 日）
- 実績：実装 2 件、設計変更で 3 件追加削除
- 現状：サブタスク 7 件、うち 5 件完了 → 進捗 71%

**改善ニーズ**：
- PO が「これ以上追加しない」と決定
- 親を「ほぼ完了」と手動マークして、最終審査待ちの状態を正確に伝えたい

---

### シーン 4：「子タスク追加時の判定」

**状況**：
```
親：v0.9.5 リテスト（手動で「完了」に設定）
  ├子1: RC テスト（完了）
  ├子2: 不具合修正（完了）
  └子3: リリース確認（完了）

→ 新しい問題発見 → AI が「子4: 緊急パッチ適用」を追加
→ 親が自動的に「仕掛」に戻される
```

**評価**：
- ✅ ユーザーに「注目してね」という効果あり（有用）
- ⚠ ただしユーザー機点をシステムが自動判定すると、混乱の可能性

## 4. 推奨される将来仕様

### 4.1 パターンA：厳密版（現在）

```
是非：是
- 手動設定値をサブタスク変更があっても保持

懸念：懸念有
- 新しい子が追加されても進捗が戻らない
- 親が「完了」のまま、新しい未完了子が見えなくなる可能性
```

### 4.2 パターンB：推奨版（設計改善案）

```
是非：推奨
- 「新しい子が追加された」ときだけ、手動設定をリセット

利点：
  ✅ 既存操作では親進捗は手動値を尊重
  ✅ 子の大幅変化があれば自動再計算に切り替わる
  ✅ ユーザーの「意図と現実のズレ」に自動対応

実装イメージ：
  1. 子タスク追加時に「親の is_manual_progress を 0 にリセット」
  2. または「手動設定値をクリア、再計算モードに戻す」
```

### 4.3 UI/UX イメージ（パターンB）

```
【親タスク詳細ペイン】

進捗表示：⬜️⬜️⬜️⬜️ 75% 仕掛

[ラジオボタン]
◎ 自動（サブタスクから自動計算）
○ 手動設定：[ドロップダウン: 未着/仕掛/完了]

注記：
「手動設定」を選ぶと、サブタスク変更による再計算は「一時的に」無視
新規子タスク追加で自動的に「自動モード」に戻る
```

## 5. AIエージェント運用ルールとの関連

### 5.1 現在の制約（BUG-20260326-02 修正後）

- サブタスク有無で親進捗の自動再計算を判定
- `is_manual_progress` の値は影響しない（子がある親はすべて自動再計算）

### 5.2 将来検討時の運用ルール追加案

セクション 4.4「is_manual_progress の運用上の制限」として以下を追記：

```markdown
#### 現在の仕様

- `is_manual_progress: 1` = 親タスクの進捗をユーザーが手動設定
- 子タスクの完了/変更があっても、親の手動設定値は維持される

#### ルール

- AI がサブタスク構造を大幅に変更する場合は、親の `is_manual_progress` をリセット（`0` に）してから操作する
- 理由：手動設定値が陳腐化し、ユーザーに見えない新規タスクが発生する可能性を回避

#### 将来検討

- 親タスクに UI で「自動 / 手動」を切り替え可能にする機能
- 手動設定時に「子追加で自動リセット」する動作オプション
```
