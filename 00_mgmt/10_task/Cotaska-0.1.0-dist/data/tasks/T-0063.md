メイン画面の期限表示へマウスカーソルを当てると画面が真っ暗になる不具合を調査する。

## バグ管理情報
- 番号: BUG-20260331-01
- バグレポート: `10_docs/30_実装・検証/10_不具合対応/20260331_01_メイン画面の期限ホバーで画面が真っ暗になる.md`

## 現時点の調査結果
- 配布版ログに `Renderer process gone` が出ており、renderer プロセスがクラッシュしている
- `due_date` の実値は ISO 日付または `YYYY-MM-DDTHH:MM`、`null` のみで、異常フォーマット混入は確認できなかった
- [Cotaska/20_app/src/renderer/components/MainPane.jsx](Cotaska/20_app/src/renderer/components/MainPane.jsx) の期限表示には hover 専用イベントがなく、直接の React state 更新は見当たらない
- CHG-039 で導入した期限表示 UI と [Cotaska/20_app/src/renderer/components/DueDatePopover.jsx](Cotaska/20_app/src/renderer/components/DueDatePopover.jsx) が回帰候補

## 次の調査
- 開発環境で再現確認
- 期限表示 DOM/CSS と期限ポップオーバーの切り分け
- 異常な `due_date` データ有無の確認
