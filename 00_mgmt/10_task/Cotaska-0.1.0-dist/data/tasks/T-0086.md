## 概要

CHG-044 の実装後に、破損タスク検知・一覧警告表示・詳細ペイン表示・復旧動作を手動確認する。

## 確認内容

1. 不正 YAML のタスクファイルを用意して Cotaska を起動する
2. 正常タスクが通常通り表示されることを確認する
3. 破損タスクが赤系表示・警告アイコン付きで表示されることを確認する
4. 詳細ペインでエラー内容とファイルパスを確認する
5. 対象ファイルを修正後、再読み込みまたは再起動で通常タスクに戻ることを確認する
6. 既存ビューと主要操作に回帰がないことを確認する

## 完了条件

- CHG-044 の完了条件とテスト観点を満たしている
- ユーザ確認で問題なしと判断されている

## 確認結果

一時データフォルダを使い、実データを壊さずに以下を確認した。

1. 不正 YAML の `T-0002.md` を配置し、`taskService.openTaskService()` を実行
2. 正常タスク `T-0001` と破損タスク `__INVALID__T-0002.md` が同時に取得できることを確認
3. 破損タスクに `is_invalid: true`、`validation_error_line: 3`、`validation_error_column: 18` が設定されることを確認
4. 対象ファイルを正しい YAML に修正後、`rebuildCache()` で通常タスク `T-0002` として復旧することを確認
5. `npm.cmd run build` を Node v22.14.0 で実行し、ビルド成功を確認
6. `tools/validate-tasks.ps1` をタスク正本配布フォルダとリリースフォルダの両方で実行し、現行データに検証エラーがないことを確認

## 補足

画面の赤系表示・警告マーク・詳細ペイン表示は、実装済みコードとビルド成功で確認した。
実際の目視確認は、次回 Cotaska 起動時に一時的な破損タスクを作って確認できる。
