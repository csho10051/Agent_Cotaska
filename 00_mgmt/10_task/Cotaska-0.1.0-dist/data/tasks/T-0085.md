## 概要

AI が `data/tasks/*.md` を追加・編集した後に、YAML frontmatter の破損を事前検知できる検証スクリプトの追加を検討する。

## 作業内容

1. `validate-tasks.ps1` などの検証スクリプト要否を判断する
2. 追加する場合、全タスク Markdown をパースしてエラー一覧を出す
3. エラー時にファイル名、行番号、理由を表示する
4. AI 作業後の運用ルールに組み込めるか確認する

## 完了条件

- 検証スクリプトを追加するか判断されている
- 追加する場合、AI 作業後に実行できる形になっている

## 対応内容

- `20_app/scripts/validate-tasks.ps1` を追加した
- `data/tasks/T-*.md` の frontmatter 構造、`id` 欠落、`id` 重複、未クォートの `:`、`tags` 行の結合ミスを検出できるようにした
- 引数なし実行時は、配置場所から近い `data/tasks` を自動検出するようにした
- リリース配布用に `Cotaska-0.1.0-dist/tools/validate-tasks.ps1` へ配置する方針にした
- `release-all.ps1` に tools 配布処理と出荷前チェックを追加した
- `sync-task-master-release.ps1` に release から task-master への tools 同期処理を追加した
- 現在の `20_app/release/Cotaska-0.1.0-dist/tools/validate-tasks.ps1` と `00_mgmt/10_task/Cotaska-0.1.0-dist/tools/validate-tasks.ps1` にも配置済み

## 検証結果

- タスク正本 `data/tasks`: OK
- リリース出力 `data/tasks`: OK
- 意図的に壊した一時タスク: NG を検出できることを確認
